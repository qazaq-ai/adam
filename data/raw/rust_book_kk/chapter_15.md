# 15-тарау. Ақылды сілтемелер

**Сілтеме** (`&T`) — `Rust`-тың негізгі мән-сілтеу механизмі. Бірақ кейде
сізге одан да күшті құрылым керек болады: автоматты тазалау,
есептеуіш-есеп, бір мәнге бірнеше иеленуші, орындалу кезіндегі
қарыз-тексеру. Соған **ақылды сілтемелер** (smart pointers) арналған.

«Ақылды» сөзі: ол жай көрсеткіш қана емес — оған қосымша мінез-құлық
бекітілген. Мысалы, иесі аумақтан шыққанда жадты автоматты тазарту,
сілтейтін мәнге қол жеткізу синтаксисін қарапайымдау, сілтеулер санын
есептеу.

`Rust`-тың стандартты кітапханасында бірнеше ақылды сілтеме түрі бар.
Бұл тарауда жиі кездесетіндерін қарастырамыз:

- **`Box<T>`** — мәнді үймеге орналастырады.
- **`Rc<T>`** — бір мәнге бірнеше иеленуші болуға мүмкіндік береді
  (тек бір ағында).
- **`Ref<T>`** мен **`RefMut<T>`** (қол жеткізу `RefCell<T>` арқылы) —
  орындалу кезінде қарыз ережелерін тексереді.
- **`Weak<T>`** — иеленуші емес, әлсіз сілтеме (циклдерді болдырмау
  үшін).

Бұл тарау сізге стандартты сілтемелерден шектеу нәрсе керек болғанда
қандай таңдау жасау керектігін, әр құралдың айырмашылықтарын
көрсетеді.

## 15.1. `Box<T>` — мәнге үймеде сілтеу

`Box<T>` — `Rust`-тың ең қарапайым ақылды сілтемесі. Ол мәнді стектен
үймеге шығарып, оған сілтеме сақтайды.

```rust
fn main() {
    let b = Box::new(5);
    println!("b = {}", b);
}
```

Әдеттегі айнымалымен айырмашылық: `b` стекте тұрған сілтеме (Box-тың
өзі), ал нақты `5` саны — үймеде. Бұл жалғыз ғана көрінбейтін
қосым — сипат бейтарап сезілмейді.

`Box<T>`-тың пайдасы келесі жағдайларда көрінеді:

- Сізге компиляция кезінде мөлшері белгісіз тип қажет, бірақ оны нақты
  мөлшері белгілі контексте қолданғыңыз келеді.
- Сізге өте үлкен мәнді көшіруге емес, иелікті ауыстыруға қажет болғанда.
- Сізге белгілі бір мәнді ие болу қажет, бірақ оның қандай нақты типке
  жататынын маңызсыз — тек қандай трейтті бекіткені.

### Рекурсивті типтер

Қарапайым мысал: cons-тізім (LISP-тен келген рекурсивті тізім). Әр
элемент келесі элементті ұстайды, тізім соңы — `Nil` нұсқасы.

```rust
enum List {
    Cons(i32, List),
    Nil,
}
```

Бұл құрастырылмайды:

```text
error[E0072]: recursive type `List` has infinite size
```

`Rust` `List`-тің мөлшерін есептей алмайды. `Cons(i32, List)` — `i32`
плюс тағы бір `List`. Ал `List`-тің мөлшері = `i32` + тағы бір `List`-тің
мөлшері. Тізбек шексіз — мөлшерсіз.

Шешімі — `Box`:

```rust
enum List {
    Cons(i32, Box<List>),
    Nil,
}
```

Енді `Cons` ішіндегі `List` — `Box<List>`-ке (фиксацияланған мөлшерлі
сілтеме) сай. Компилятор `List`-тің мөлшерін есептей алады:
`i32` + `Box`-тың мөлшері (көрсеткіш — әдетте 8 байт).

Cons-тізімді жасау:

```rust
use crate::List::{Cons, Nil};

fn main() {
    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
}
```

Бұл `1 → 2 → 3 → Nil` тізбегі. `Box` рекурсивті типті жұмыстайды.

## 15.2. `Deref` трейті — ақылды сілтемені қарапайым сілтеме сияқты пайдалану

`*` операторы (дереф-оператор) кәдімгі сілтемені оның ішіндегі мәнге
түрлендіреді. `*&5` — 5 саны. `Box<T>` те солай: `*Box::new(5)` — 5
саны.

Бұл жұмысты `Deref` трейті бекітеді. Сіз өзіңіздің ақылды сілтемені
жасап, оған `Deref` бекітсеңіз — `*` операторы соған да арнап жұмыс
істейді.

### Өзіндік ақылды сілтеме жасау

`MyBox<T>` атты қарапайым нұсқаны жазайық:

```rust
struct MyBox<T>(T);

impl<T> MyBox<T> {
    fn new(x: T) -> MyBox<T> {
        MyBox(x)
    }
}
```

Бұл жай ғана структ, ішінде бір мән бар. Қазірге `MyBox`-та `*` оператор
жұмыс істемейді:

```rust
let y = MyBox::new(5);
println!("{}", *y); // ҚАТЕ: type `MyBox<{integer}>` cannot be dereferenced
```

`Deref` бекітсек:

```rust
use std::ops::Deref;

impl<T> Deref for MyBox<T> {
    type Target = T;

    fn deref(&self) -> &T {
        &self.0
    }
}
```

`Deref::deref` ішкі мәнге сілтеме қайтарады. Енді `*y` жұмыс істейді
— компилятор оны `*(y.deref())` деп қарайды.

### Деref-итеру (Deref Coercion)

`Rust`-тың тағы бір ыңғайлы мүмкіндігі — **деref-итеру**: `&T` керек жерде
сіз `&Smart<T>` берсеңіз, компилятор автоматты түрде `Deref`-ті қолданып
түрлендіреді.

Мысалы, `&str` күтетін функция бар:

```rust
fn hello(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    let m = MyBox::new(String::from("Айнұр"));
    hello(&m);
}
```

`&m` — `&MyBox<String>`. Бірақ `hello` `&str` күтеді. Деref-итеру:

1. `&MyBox<String>` → `MyBox::deref` → `&String`
2. `&String` → `String::deref` → `&str`

Екі қадамды компилятор автоматты жасайды. Сіз кодта `&m` жазсаңыз да —
функция `&str` алады.

Бұл — өте ыңғайлы. `&String`-ті алатын жерлерде `&Box<String>`,
`&Rc<String>`, `&MyBox<String>` — бәрі сай.

## 15.3. `Drop` трейті — тазалау логикасын орындау

`Drop` трейті — мән аумақтан шыққанда орындалатын кодты бекітеді.
`Box`, `String`, `Vec` — бәрі `Drop` арқылы өздерінің жадын тазартады.

Мысал:

```rust
struct CustomSmartPointer {
    data: String,
}

impl Drop for CustomSmartPointer {
    fn drop(&mut self) {
        println!("CustomSmartPointer ({}) тазаланды.", self.data);
    }
}

fn main() {
    let c = CustomSmartPointer { data: String::from("c-нің мәні") };
    let d = CustomSmartPointer { data: String::from("басқа") };
    println!("CustomSmartPointer-лер жасалды.");
}
```

Шығаратыны:

```text
CustomSmartPointer-лер жасалды.
CustomSmartPointer (басқа) тазаланды.
CustomSmartPointer (c-нің мәні) тазаланды.
```

Соңғы екі жол — `main` бітпей `drop` функциялары шакырылды. Тазалау —
кері тәртіппен (соңғы жасалғандай — алдында тазаланады).

### Мерзімінен бұрын `drop` шакыру

`drop` әдісін қолмен шакыру тыйым салынған — өйткені сонда мән екі
рет тазалану қаупі туады. Бірақ `std::mem::drop` функциясы
сізге мәнді мерзімінен бұрын тазалатуға мүмкіндік береді:

```rust
fn main() {
    let c = CustomSmartPointer { data: String::from("кейбір дерек") };
    println!("CustomSmartPointer жасалды.");
    drop(c);
    println!("CustomSmartPointer mainнің соңында емес жерде тазаланды.");
}
```

Шығатыны:

```text
CustomSmartPointer жасалды.
CustomSmartPointer (кейбір дерек) тазаланды.
CustomSmartPointer mainнің соңында емес жерде тазаланды.
```

`std::mem::drop` иелікті алады, `drop` функциясын шакырады, мәнді
тазартады.

## 15.4. `Rc<T>` — есептелетін иеленушілер ақылды сілтемесі

`Box<T>` бір ие — мәннің бір ғана иесі. Кей жағдайда сізге **бірнеше
ие** қажет: бір мәнге бірнеше нысан сілтейтін болуы керек, мәннің
тазалануы — соңғысы кеткеннен кейін болсын.

Соған `Rc<T>` (Reference Counted) арналған. Ол ішкі есептеуішті
ұстайды; әр клон есептеуішті бір көтереді, әр `drop` бір азайтады;
есептеуіш нөлге жетсе — мән тазаланады.

> **Маңызды ескерту:** `Rc<T>` тек **бір ағында** қолдануға арналған.
> Ағындар арасында бөлесу үшін `Arc<T>` бар (16-тарауда қарастырамыз).

### `Rc<T>` мысалы

Cons-тізіммен қайталайық, бірақ екі тізім бір бөлікті ортақ ие болсын:

```rust
use std::rc::Rc;

enum List {
    Cons(i32, Rc<List>),
    Nil,
}

use crate::List::{Cons, Nil};

fn main() {
    let a = Rc::new(Cons(5, Rc::new(Cons(10, Rc::new(Nil)))));
    println!("a жасалғаннан кейін санақ = {}", Rc::strong_count(&a));

    let b = Cons(3, Rc::clone(&a));
    println!("b жасалғаннан кейін санақ = {}", Rc::strong_count(&a));

    {
        let c = Cons(4, Rc::clone(&a));
        println!("c жасалғаннан кейін санақ = {}", Rc::strong_count(&a));
    }

    println!("c аумақтан шыққанан кейін санақ = {}", Rc::strong_count(&a));
}
```

Шығатыны:

```text
a жасалғаннан кейін санақ = 1
b жасалғаннан кейін санақ = 2
c жасалғаннан кейін санақ = 3
c аумақтан шыққанан кейін санақ = 2
```

Әр `Rc::clone` есептеуішті бір көтерді, әрі `c` аумақтан шыққан кезде
оның `drop` есептеуішті бір азайтты. `a`-ның мәні әлі бар — `b` оны
ие.

`Rc::clone` — арзан: ол үймеде орналасқан мәнді **көшірмейді**, тек
есептеуішті бір көтереді. `String::clone`-мен шатастырмаңыз: ол
шынында жадты дублирлейді.

`Rc<T>` тек **тұрақты сілтеме** — оның мәнін өзгерту мүмкін емес. Бірнеше
ие бір мәнді өзгертсе — әркімнің көзқарасы мәнге басқа болу мүмкін.
Сондықтан `Rc<T>` көп оқитын, азғантай өзгертетін деректерге ыңғайлы.

## 15.5. `RefCell<T>` мен ішкі өзгермелілік үлгісі

Қарапайым `Rust`-та қарыз ережелері — компиляция кезінде. Бірақ кейбір
тестілеу мен жобалар үшін бұл шектен тыс қатаң. **Ішкі өзгермелілік
(interior mutability)** — бір тип сырттан тұрақты сияқты көрінсе де,
ішінен мәнін өзгерте алатын үлгі. Сонымен қатар қарыз ережелері
сақталатынын жұмыс уақытында тексереміз.

Соған `RefCell<T>` арналған.

### `RefCell<T>` пен жұмыс уақытындағы қарыз тексеру

`RefCell<T>` бекіткен мәнді сыртқы көрініс **тұрақты** болады. Бірақ
ішінде өзгертуге келеді:

- `borrow()` — `Ref<T>` (тұрақты сілтеме) қайтарады.
- `borrow_mut()` — `RefMut<T>` (өзгермелі сілтеме) қайтарады.

`RefCell<T>` есептеуішті ұстайды: қанша белсенді тұрақты, қанша
өзгермелі сілтеме бар. Қарыз ережелері бұзылса — `panic!`:

- Бір уақытта тек **бір** өзгермелі сілтеме болуы мүмкін.
- Немесе **көп** тұрақты сілтеме.

Айырмашылығы тек: тексеру **жұмыс кезінде** жүреді. Компилятор
келісімсіз болғанда — `RefCell` оны ұстайды.

### Өзгермелілік ішке шегеру қажет ме?

Әдетте керек емес. `Rust`-тың дизайны сізге өзгермелілікті анық
білдіруге шакырады. Бірақ кейде:

- Тестілеу барысында **mock-нысан** жасау керек. Mock — нақты тип
  сияқты көрінетін, бірақ сынақ ішінде есептегі сақтайтын дайындалмыш
  нысан. Tip-тің әдістері `&self` ала отырып, ол ішкі есептегін
  өзгертуі керек. Сондай — `RefCell` арқылы.
- Графты немесе циклдік құрылымды жасау керек. Стандартты ережелер
  тым қатаң болады.

Ережені қысқа есте сақтаңыз: алдымен компиляция-кездік
қарыз-тексеруді қолданыңыз. `RefCell`-ді тек шынында қажет болғанда.

### Mock-нысан мысалы

Аналогты қажетті жұмысын тестілеу үшін:

```rust
use std::cell::RefCell;

pub trait Messenger {
    fn send(&self, msg: &str);
}

struct MockMessenger {
    sent_messages: RefCell<Vec<String>>,
}

impl Messenger for MockMessenger {
    fn send(&self, message: &str) {
        // `&self` — тұрақты сілтеме, бірақ ішкі векторды өзгерте аламыз.
        self.sent_messages.borrow_mut().push(String::from(message));
    }
}
```

`Messenger` трейті `send(&self, ...)` бекітеді — өзгермейтін
интерфейс. Бірақ `MockMessenger` тестілеу барысында хабарламаларды
есептеуі керек. `RefCell` арқылы `send` ішінде ішкі векторды
өзгерте аламыз.

### `Rc<T>` пен `RefCell<T>` бірге

Кейде бізге **бірнеше иеленуші** бар, әрі **өзгертуге болатын** тип
қажет. Шешімі: `Rc<RefCell<T>>`.

```rust
use std::cell::RefCell;
use std::rc::Rc;

enum List {
    Cons(Rc<RefCell<i32>>, Rc<List>),
    Nil,
}

use crate::List::{Cons, Nil};

fn main() {
    let value = Rc::new(RefCell::new(5));

    let a = Rc::new(Cons(Rc::clone(&value), Rc::new(Nil)));

    let b = Cons(Rc::new(RefCell::new(3)), Rc::clone(&a));
    let c = Cons(Rc::new(RefCell::new(4)), Rc::clone(&a));

    *value.borrow_mut() += 10;

    println!("a кейін: {:?}", a);
    println!("b кейін: {:?}", b);
    println!("c кейін: {:?}", c);
}
```

`value` 5-ке тағайындалған; кейін біз оны 15-ке өзгертеміз. Ол `a`,
`b`, `c` — үш тізімнің ортақ бөлігі. Барлық үшеуі жаңартылған мәнді
көреді.

## 15.6. Сілтеме циклдары жадтың ағытпасына апаруы мүмкін

`Rust`-тың иелік жүйесі ерекше: ол сізге қауіпсіз код жазуды кепілдейді,
бірақ **ағытпаны** (memory leak) толық алып тастамайды. `Rc<T>` мен
`RefCell<T>` бірге **сілтеме циклы** жасауға мүмкіндік береді: екі мән
бір-біріне сілтейді, есептеуіштер ешқашан нөлге жетпейді,
жад босатылмайды.

Мысалы, тізімнің бір элементі екіншіге сілтейді, екіншісі — біріншіге.
`Rc::strong_count` екеуінде де 1-ден жоғары — ешбір аумақ шықпайды,
ешбір тазалау болмайды. Бұл — ағытпа. Бағдарлама орналасты бекіттеу,
бірақ жадын ұзақ ұстайды.

### `Weak<T>` арқылы циклдерді болдырмау

`Rc<T>` — **күшті** сілтеме. Оның есептеуіші иелік сипатын білдіреді:
есептеуіш нөлге жетпегенше — мән өшпейді.

`Weak<T>` — **әлсіз** сілтеме. Ол есептеуішке әсер етпейді (`weak_count`
бөлек есептеуіш). Иелік ережесі: мәнді тек күшті сілтемелер ұстайды.
Күшті есептеуіш нөлге жетсе — мән тазаланады, әлсіз сілтемелер бос
болады.

`Weak<T>`-тің мәніне қол жеткізу үшін `upgrade` әдісі: ол `Option<Rc<T>>`
қайтарады. Мән әлі тірі болса — `Some(Rc::clone)`. Тазаланған болса —
`None`.

### Атасы-баласы құрылымы

Мысал — атасы мен баласы: ата бар балаларға күшті сілтейді, бала атасына
әлсіз сілтейді. Цикл жоқ — ата кеткенде бала да жоғалады; бала кеткенде
ата сақталады (өйткені бала оны әлсіз ұстайды).

```rust
use std::cell::RefCell;
use std::rc::{Rc, Weak};

struct Node {
    value: i32,
    parent: RefCell<Weak<Node>>,
    children: RefCell<Vec<Rc<Node>>>,
}

fn main() {
    let leaf = Rc::new(Node {
        value: 3,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![]),
    });

    let branch = Rc::new(Node {
        value: 5,
        parent: RefCell::new(Weak::new()),
        children: RefCell::new(vec![Rc::clone(&leaf)]),
    });

    *leaf.parent.borrow_mut() = Rc::downgrade(&branch);

    // ...
}
```

`Rc::downgrade(&branch)` — `Weak<Node>` жасайды. `branch` — `leaf`-тің
ата-нысаны; `leaf.parent` — `branch`-қа әлсіз сілтеме. Цикл жоқ —
есептеуіштер дұрыс жұмыс істейді.

`leaf.parent.borrow().upgrade()` — `Option<Rc<Node>>`. Ата тірі болса —
сізге `Rc<Node>` беріледі.

## 15.7. Тарау қорытындысы

Ақылды сілтемелер `Rust`-тың сілтеме жүйесінің кеңейтілген бөлігі.
Әр жұмыс түрі үшін өзіндік құралы бар:

- **`Box<T>`** — үймеде сақтау, рекурсивті типтер, белгісіз-мөлшерлі
  типтер.
- **`Rc<T>`** — бірнеше ие, тек оқуға арналған, бір ағында.
- **`RefCell<T>`** — ішкі өзгермелілік, орындалу кезіндегі қарыз
  тексеру.
- **`Weak<T>`** — ие емес сілтеме, циклдерді болдырмау үшін.
- **`Deref`** трейті — `*` операторын өзіңіздің ақылды сілтемелеріңізге
  бекіту, деref-итеру.
- **`Drop`** трейті — мән аумақтан шыққанда тазалау логикасы.

Бұл құралдар көпшілігі бір ағындағы қолданысқа арналған. Көп ағынды
бағдарламалар — олардың `Arc<T>` пен `Mutex<T>` нұсқалары — келесі
тарауда қарастырылатын болады.
