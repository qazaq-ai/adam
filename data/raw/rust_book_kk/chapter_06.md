# 6-тарау. Енам мен үлгіге сай келтіру

Бұл тарауда **енам** (enumeration) — кейде «енам» немесе «алгебралық тип»
деп те аталатын — типтің мүмкіндіктерін қарастырамыз. Енам арқылы біз
тиісті тип мынадай немесе мынадай бір нұсқада бола алады деген ережені
бекітеміз. Алдымен `Rust`-та енамды қалай анықтауға және оны қалай
қолдануға болатынын көрсетеміз. Содан кейін `Option` деп аталатын аса
пайдалы енамды қарастырамыз — ол арқылы мән болуы немесе болмауы мүмкін
екендігін бірден типтегі сипаттаймыз. Тарау соңында үлгіге сай келтіру
(pattern matching) арқылы әр нұсқаны жеке-жеке өңдеуді үйренеміз — бұл
`Rust`-тағы `match` өрнегінің күші.

## 6.1. Енамды анықтау

Структ — бірнеше байланысты өрістерді бір орынға жинақтайды. Енам —
басқаша мәселені шешеді: типтің бір ғана мәні болады, бірақ ол сол
мүмкін нұсқалардың бірінде ғана. Мысалы, IP мекенжайы IPv4 немесе IPv6
форматында бола алады. Бір IP мекенжайы екеуі бірден бола алмайды —
тек біреуі. Енам осыны типтегі тура көрсетуге мүмкіндік береді:

```rust
enum IpAddrKind {
    V4,
    V6,
}
```

`IpAddrKind` — біз жариялаған тип. Оның екі **нұсқасы** бар: `V4` пен
`V6`. Енамның данасын осы нұсқалар арқылы жасаймыз:

```rust
let four = IpAddrKind::V4;
let six = IpAddrKind::V6;
```

Назар аударыңыз: `IpAddrKind::V4` пен `IpAddrKind::V6` бір типке —
`IpAddrKind`-қа жатады. Бұл — енамның бірінші артықшылығы: екі түрлі
нұсқа бір типтік шатырдың астында жиналған. Сондықтан сіз кез келген
`IpAddrKind` мәнін ескере алатын функция жаза аласыз:

```rust
fn route(ip_kind: IpAddrKind) {
    // ...
}

route(IpAddrKind::V4);
route(IpAddrKind::V6);
```

### Нұсқаға деректерді бекіту

Жоғарыдағы мысал нұсқаны ажыратады, бірақ мекенжайдың өзін сақтамайды.
Структпен қосайық:

```rust
enum IpAddrKind {
    V4,
    V6,
}

struct IpAddr {
    kind: IpAddrKind,
    address: String,
}

let home = IpAddr {
    kind: IpAddrKind::V4,
    address: String::from("127.0.0.1"),
};
```

Бұл жұмыс істейді, бірақ `Rust`-та неғұрлым ыңғайлы тәсіл бар — енамның
нұсқасына **тікелей деректерді бекіту**:

```rust
enum IpAddr {
    V4(String),
    V6(String),
}

let home = IpAddr::V4(String::from("127.0.0.1"));
let loopback = IpAddr::V6(String::from("::1"));
```

Енді әр нұсқа өзінің деректерін «ішіне» алады. Структ бөлек керек емес —
енам осы рөлді өзі атқарады.

Әр нұсқаның түрлі деректері болуы мүмкін:

```rust
enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}

let home = IpAddr::V4(127, 0, 0, 1);
let loopback = IpAddr::V6(String::from("::1"));
```

`V4` нұсқасы төрт байтты сақтайды, `V6` — жолды. Бір типте әртүрлі
ішкі деректер. Стандартты кітапханада бар `IpAddr` түрі дәл осындай
құрылған.

Енам нұсқалары — структтың кез келген түріне ұқсас бола алады:

```rust
enum Message {
    Quit,                       // деректі жоқ
    Move { x: i32, y: i32 },    // аталған өрістер (структ-ұқсас)
    Write(String),              // бір жол
    ChangeColor(i32, i32, i32), // үш сан (кортеж-ұқсас)
}
```

Бір типте: ешбір деректі жоқ нұсқа, аталған өрістері бар нұсқа, кортеж-
нұсқа. Бұл — енамның күші: біз жөн ережелерді типтегі тікелей бекітеміз.

### Енамға әдістер

Структ сияқты, енамда да `impl` блогында әдістерді жариялауға болады:

```rust
impl Message {
    fn call(&self) {
        // ...
    }
}

let m = Message::Write(String::from("сәлем"));
m.call();
```

Әдіс ішінен `self`-тің қандай нұсқа екенін `match` арқылы анықтайды.
`match` туралы алда айтамыз.

### `Option` енамы

`Option` — `Rust`-тың стандартты кітапханасында тұрған, бағдарламашының
күн сайын қолданатын ең маңызды енамдарының бірі. Ол: «мән болуы мүмкін
немесе мән болмауы мүмкін» деген тұжырымдаманы типтік жүйеге келтіреді.

Көп тілдерде «нөлдік мән» (`null`) ұғымы бар: айнымалы белгілі мәнге
сілтейді немесе ешнәрсеге сілтемейді. `Tony Hoare`, `null` ұғымын ойлап
тапқан адам, оны өзінің «миллиард долларлық қатесі» деп сипаттады:
бағдарламашылар тұрақты түрде «нөл бола алатынын» ұмытып, бағдарламалары
құлайды.

`Rust`-та `null` жоқ. Оның орнына `Option` енамы бар:

```rust
enum Option<T> {
    None,
    Some(T),
}
```

`<T>` — жалпылама тип параметрі (10-тарауда қарастырамыз). Қазірге
оны: «`T` — кез келген тип» деп ұғынайық. Сондықтан `Option<i32>` — бұл
бар не жоқ `i32`; `Option<String>` — бар не жоқ `String`.

`Option` стандартты кітапханада бекітілген, оны импорттаудың керегі жоқ.
Сонымен қатар оның `Some` мен `None` нұсқалары да тікелей қолжетімді —
`Option::Some` немесе тек `Some` деп жазуға болады:

```rust
let some_number = Some(5);
let some_char = Some('e');
let absent_number: Option<i32> = None;
```

`Some(5)` — компилятор `Option<i32>` екенін шеше алады. `None` жағдайында
тип белгісіз болғандықтан, оны ашық белгілеу керек: `Option<i32>`.

`Option<T>` пен `T` бөлек типтер. Сонымен бірден жұмыс істей алмайсыз:

```rust
let x: i8 = 5;
let y: Option<i8> = Some(5);
let sum = x + y; // ҚАТЕ
```

Қате: `i8` мен `Option<i8>` — әртүрлі типтер. Бір санды басқаға қосу
үшін біз `y`-тің ішінде шынымен сан барын тексеріп, оны шығарып алуымыз
керек. Бұл — `Rust`-тың `null`-ге тыйым салуы: компилятор сізге мәнді
қолданбас бұрын оның бар-жоғын тексеруді міндеттейді.

Сонымен — `Option<T>`-нің ішіндегі `T`-ге қалай қол жеткізуге болады?
Соған арналған негізгі құрал — `match` өрнегі.

## 6.2. `match` басқару құрылымы

`match` — `Rust`-тың ең күшті басқару құрылымдарының бірі. Ол мәнді
бірнеше **үлгіге** қарсы салыстырады және сай келген тармақтың кодын
орындайды. Үлгілер литералдардан, айнымалылардан, орынтолтырғыштардан,
тіпті құрама құрылымдардан тұра алады. `match` мүмкіндігі компилятордың
**әр нұсқаны қамтығаныңызды тексеруі**нен туындайды — бірде-бір нұсқа
ұмытылмайды.

Қарапайым мысал: тиынның мәнін анықтау.

```rust
enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

`match` арқылы біз `coin`-ды әр нұсқаға қарсы тексереміз. Стрелка `=>`
үлгіні нәтижеге байланыстырады. Әр тармақ `,` арқылы бөлінеді.

Тармақтардың денесі бір өрнек болуы мүмкін (мысал жоғарыда), немесе
бірнеше сөйлемнен тұратын блок:

```rust
fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => {
            println!("Сәлем, тиын!");
            1
        }
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}
```

### Мәнге байланысатын үлгілер

Енамның нұсқасы дерек ұстаса, `match` арқылы оны шығарып ала аламыз.
Мысалы, штатпен байланысты `Quarter` нұсқасын қарастырайық:

```rust
#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
    // ...
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter(state) => {
            println!("Штаттан тиын: {:?}!", state);
            25
        }
    }
}
```

`Coin::Quarter(state)` үлгісі — `Quarter` нұсқасына сәйкес келгенде,
ішіндегі мән `state` атты айнымалыға бекітіледі. Бұл — үлгіге сай
келтірудің күшті жағы: үлгі арқылы біз нұсқаны таниды әрі оның ішкі
мәнін шығарып аламыз.

### `Option<T>`-мен сай келтіру

`Option<T>`-ге практикалық мысал. Айталық, `Some(i32)` мәніне 1 қосып,
`None` болса оны өзгертпейтін функция жазайық:

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}

let five = Some(5);
let six = plus_one(five);   // Some(6)
let none = plus_one(None);  // None
```

`Some(5)`-ке `plus_one` шақырылғанда `Some(i)` үлгісі сай, `i` 5 болады,
сондықтан `Some(i + 1)` = `Some(6)` қайтарылады. `None` шақырылғанда
`None` үлгісіне сай — `None` қайтарылады.

### `match` барлық нұсқаны қамтуы керек

Маңызды кепілдік: `match` өрнегінде сіз барлық мүмкін нұсқаларды
қарастыруыңыз керек. Әйтпесе компилятор қателікке барады:

```rust
fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        Some(i) => Some(i + 1),
    }
}
```

Қате:

```text
error[E0004]: non-exhaustive patterns: `None` not covered
```

`Rust` сізге ұмытылған нұсқаны атап беріп тұр. Бұл — компилятордың сіздің
кодыңызды толық қамтығаныңызды тексеруі. Әрбір ықтимал жағдай ескерілген.

### Жалпы тармақ және `_`

Кейде сізге барлық қалған нұсқаларды бір жалпы тармақпен өңдеу керек.
Мысалы, ойын бағдарламасында тек 3 пен 7 саны арнайы әсер береді:

```rust
let dice_roll = 9;
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    other => move_player(other),
}

fn add_fancy_hat() {}
fn remove_fancy_hat() {}
fn move_player(num_spaces: u8) {}
```

`other` — айнымалы үлгісі. Ол бұрын жазылғандардан тыс кез келген мәнге
сай келеді және сол мәнді өзіне байланыстырады.

Егер сізге қалған мәннің өзі керек емес болса — оны еш жерге қолданбасаңыз
— `_` (астыңғы сызық) орынтолтырғышын қолданыңыз. `_` кез келген мәнге
сай келеді, бірақ оны ешбір айнымалыға байланыстырмайды:

```rust
let dice_roll = 9;
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => reroll(),
}

fn reroll() {}
```

Тіпті ештеңе жасамай, бір кортежді (`()`) қайтарып шықсаңыз болады:

```rust
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => (),
}
```

Бұл «басқа барлық жағдайда — ешнәрсе істеме» дегенді білдіреді.

## 6.3. `if let` арқылы қысқа басқару

Кейде `match` шектен тыс ұзын болады — сізге бір ғана нұсқаны қамту
керек. Сондай үшін `Rust`-та `if let` синтаксисі бар.

Мысалы, `Option<u8>`-та мән бар болса оны шығаратын код:

```rust
let config_max = Some(3u8);
match config_max {
    Some(max) => println!("Максимум: {}", max),
    _ => (),
}
```

Бұл коды `if let` арқылы қысқартайық:

```rust
let config_max = Some(3u8);
if let Some(max) = config_max {
    println!("Максимум: {}", max);
}
```

`if let` синтаксисі үлгіні (`Some(max)`) және өрнекті (`config_max`)
теңдікпен бөлеп жазады. Үлгі сай болса, ішіндегі айнымалы (`max`)
бекітіледі және блок орындалады. Сай болмаса — өткізіледі.

`if let` `match`-тен қысқа, бірақ бір кемшілік бар: `Rust` сізге
қалған нұсқаларды қамтуды міндеттемейді. Бұл — сіз қалаған тарифтерден
тыс жатыр; өзіңіз шеше беріңіз. Сонымен бірге `else` тармағын қосуға
болады:

```rust
let mut count = 0;
if let Coin::Quarter(state) = coin {
    println!("Штаттан тиын: {:?}!", state);
} else {
    count += 1;
}
```

Бұл `match coin { Coin::Quarter(state) => ..., _ => count += 1 }` дегенге
тең.

`if let` — `match`-тың қысқа баламасы. Тек бір нұсқаны өңдегіңіз келсе —
`if let` жинақы әрі оқуға жеңіл. Бірақ толық қамту керек жағдайда `match`
дұрыс таңдау. Кодыңызды оқыған адам үшін екеуі де айқын болсын.

## 6.4. Тарау қорытындысы

Енам — `Rust` типтік жүйесінің ең күшті құралдарының бірі. Ол арқылы
сіз: бір типтің бірнеше нұсқасы болатынын; әр нұсқаның өзіндік деректері
болуы мүмкін екенін; және компилятор сізге әр нұсқаны қамтуды
міндеттейтінін бекітесіз. `Option<T>` арқылы «нөлдік мән» проблемасы
типтік жүйеде шешілген: мәнді қолданбас бұрын `match` немесе `if let`
арқылы бар-жоғын тексеруге тура келеді.

`match` өрнегі — `Rust`-тағы тармақтаудың негізі. Үлгіге сай келтіру
арқылы біз: нұсқаны таниды; ішкі деректерді шығарады; компилятор
қамтудың толықтығын тексереді. `if let` — `match`-тың қысқа баламасы
бір нұсқаны ғана өңдегенде ыңғайлы.

Келесі тарауда үлкенірек жобалар жайында айтамыз: модулдер,
сандықтар, жұмыс кеңістігі. `Rust`-тың құрылыс жүйесі арқылы кодты
бөлуге, қайта пайдалануға және басқа адамдармен бірге жұмыс істеуге
болады.
