# 16-тарау. Қорқынышсыз қатарлас орындау

Қатарлас орындау (concurrency) — бағдарламаның бөлектерін бір уақытта
жүргізу мүмкіндігі. Әдетте қателер табуы өте қиын: екі ағын бір
жадқа бір сәтте жазғанда — нәтиже тосын болады. Жарыс шарттары
(data races), өзара тосқауылдар (deadlocks), әлсіз синхрондау —
осы санаттан қателер.

`Rust`-тың дизайн тұжырымдамасы ерекше: **қорқынышсыз қатарлас
орындау**. Тілдің типтік жүйесі мен қарыз тексергіш ағын-қауіпсіздіктің
негізгі санатын **компиляция кезінде** ұстайды. Сондықтан сіздің
бағдарламаңыз орындалу кезінде көп жағдайда жарыс шартына кездеспейді
— ол одан да құрастырылмайды.

Бұл тарауда:

1. Ағындарды құру және басқару.
2. Ағындар арасында **хабарлама арқылы алмасу** (`mpsc::channel`).
3. **Ортақ күйді** қорғау (`Mutex<T>` пен `Arc<T>`).
4. `Send` пен `Sync` трейттері — қатарлас орындаудың типтік негізі.

## 16.1. Бір уақытта код орындау үшін ағындарды қолдану

**Ағын** (thread) — операциялық жүйенің ұсынатын орындалудың параллель
жолы. `Rust`-та жаңа ағынды бастау үшін `std::thread::spawn` функциясын
қолданамыз:

```rust
use std::thread;
use std::time::Duration;

fn main() {
    thread::spawn(|| {
        for i in 1..10 {
            println!("Қосалқы ағыннан: {}", i);
            thread::sleep(Duration::from_millis(1));
        }
    });

    for i in 1..5 {
        println!("Негізгі ағыннан: {}", i);
        thread::sleep(Duration::from_millis(1));
    }
}
```

`thread::spawn` жабу қабылдайды; жабу — жаңа ағында орындалатын код.
Шығатын тізбек кездейсоқ түрде араласуы мүмкін:

```text
Негізгі ағыннан: 1
Қосалқы ағыннан: 1
Негізгі ағыннан: 2
Қосалқы ағыннан: 2
...
```

Бірақ бір нәрсе анық: `main` функциясы аяқталғанда — барлық ағындар
тоқтатылады. Қосалқы ағынның 9-ға дейінгі есептеулері жетпей қалуы
мүмкін.

### Ағынды күту

Ағынның аяқталуын күту үшін **қосылу-білдіргіш** (join handle)
қолданамыз. `spawn` `JoinHandle<T>` қайтарады; оның `join` әдісі —
ағынның аяқталуын күтеді:

```rust
let handle = thread::spawn(|| {
    for i in 1..10 {
        println!("Қосалқы: {}", i);
        thread::sleep(Duration::from_millis(1));
    }
});

for i in 1..5 {
    println!("Негізгі: {}", i);
    thread::sleep(Duration::from_millis(1));
}

handle.join().unwrap();
```

Енді `main` `handle.join()` шакыруда тоқтайды; қосалқы ағын аяқталғанда
жалғасады. Сондықтан барлық 9 рет шығады.

`join` `Result` қайтарады — ағын `panic!`-қа барса, `Err`. `unwrap` —
оны қарапайым өңдеу.

### `move` арқылы иелікті ағынға беру

Жабу ішінен қоршаған ортадан мән керек болса — мәселе туады:

```rust
let v = vec![1, 2, 3];

let handle = thread::spawn(|| {
    println!("Вектор: {:?}", v); // ҚАТЕ
});
```

Қате: `Rust` `v`-ні шакырушы аумақтан қарызға алу әрекетін көреді,
бірақ ағынның қаншалықты ұзақ жұмыс істейтінін білмейді. Шакырушы
аумақ бітсе де ағын әлі тірі болуы мүмкін — `v` жабайы сілтемеге
айналады.

Шешімі — `move` жабуы:

```rust
let v = vec![1, 2, 3];

let handle = thread::spawn(move || {
    println!("Вектор: {:?}", v);
});

handle.join().unwrap();
```

`move ||` — жабу `v`-нің **иелігін** алады. `v` сыртта енді жарамсыз;
ағын ішінде ол толық тірі. Иелік алмасу — қарапайым әрі қауіпсіз.

## 16.2. Хабарлама арқылы ағындар арасында дерек жіберу

Қатарлас орындау моделдерінің бірі — **хабарлама арқылы алмасу**: бір
ағын мәнді басқа ағынға «жібереді», иелікті де солай ауыстырады.
Мұндай моделді Go тілі айқын мақтайды («Don't communicate by sharing
memory; share memory by communicating»).

`Rust`-та хабарлама-каналдарды стандартты кітапхана `std::sync::mpsc`
ұсынады. **MPSC** — *multiple producer, single consumer* (көп
жіберуші, бір қабылдаушы).

### Қарапайым канал

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let val = String::from("сәлем");
        tx.send(val).unwrap();
    });

    let received = rx.recv().unwrap();
    println!("Алынды: {}", received);
}
```

`mpsc::channel()` — екі ұшты қайтарады: `tx` (transmitter, жіберуші)
және `rx` (receiver, қабылдаушы).

Қосалқы ағында: `tx.send(val)` — `val`-ды каналға жібереді. `send`
иелікті алады — содан соң `val`-ды басқа жерде қолдану мүмкін емес.
Бұл — `Rust`-тың кепілдігі: жіберілген мәнді жіберуші өзгерте алмайды,
дер бір ғана нысан оны иеленеді.

Негізгі ағында: `rx.recv()` — каналдан хабарлама күтеді. Ол блок
жасайды (ағын мәнді алғанша тоқтап тұрады). `Result` қайтарады:
`Ok(val)` мән алынды; `Err` — канал жабылған.

`recv` блокты, `try_recv` — блок жасамай тексереді; жоқ болса —
бірден `Err`.

### Бірнеше мән жіберу

```rust
let (tx, rx) = mpsc::channel();

thread::spawn(move || {
    let vals = vec![
        String::from("сәлем"),
        String::from("дүние"),
        String::from("қалай"),
        String::from("халдер"),
    ];
    for val in vals {
        tx.send(val).unwrap();
        thread::sleep(Duration::from_secs(1));
    }
});

for received in rx {
    println!("Алынды: {}", received);
}
```

`for received in rx` — `rx`-ты итератор ретінде қарастырамыз. Ол блок
жасайды; жаңа хабарлама келгенше күтеді; канал жабылғанда (`tx`
аумақтан шыққанда) цикл аяқталады.

### Бірнеше жіберуші

`mpsc` — *multiple producer*. Бір каналға бірнеше ағын жібере алады.
`tx`-ты `clone` арқылы көшіреміз:

```rust
let (tx, rx) = mpsc::channel();

let tx1 = tx.clone();
thread::spawn(move || {
    let vals = vec!["1-1", "1-2", "1-3"];
    for v in vals {
        tx1.send(String::from(v)).unwrap();
    }
});

thread::spawn(move || {
    let vals = vec!["2-1", "2-2", "2-3"];
    for v in vals {
        tx.send(String::from(v)).unwrap();
    }
});

for received in rx {
    println!("Алынды: {}", received);
}
```

Екі ағын бір каналға жібереді. Қабылдаушы — бір. Ретсіз араласқан
тізбек шығады. Ағындар арасында ешбір синхрондау — `mpsc`-тің өзі
бәрін жөндейді.

## 16.3. Ортақ күйді қорғау: `Mutex<T>` пен `Arc<T>`

Хабарлама-канал — бір модель. Басқа модель — **ортақ күй**: бірнеше
ағын бір мәнге қол жеткізеді, синхрондау арқылы.

`Rust`-тың стандартты кітапханасы ортақ күйді қорғау үшін `Mutex<T>`
ұсынады. **Mutex** — Mutual Exclusion (өзара эксклюзивтілік): бір
уақытта бір ғана ағынға мәнге қол жеткізуге рұқсат. Қалғандары —
кезек күтеді.

### Бір ағынды Mutex

```rust
use std::sync::Mutex;

fn main() {
    let m = Mutex::new(5);

    {
        let mut num = m.lock().unwrap();
        *num = 6;
    }

    println!("m = {:?}", m);
}
```

`Mutex::new(5)` — мәнді ұстайтын мутекс. `lock()` — мутекстіге
блокпен иелік сұрайды; `Result` қайтарады; әдеттегідей `unwrap`.
Алынған `MutexGuard` — `RefMut`-қа ұқсас, `Deref` арқылы ішкі мәнге
қол жеткіземіз. Аумақ біткенде — `Drop` арқылы автоматты түрде
босатылады.

Үлгі — қарызды ағындар арасында бөлуге арналған, бірақ алдымен бір
ағында жұмыс істегенін көреміз.

### Бірнеше ағында Mutex қолдану

`Mutex<T>` ағындар арасында бөлу үшін бірнеше ие болуы керек. `Rc<T>`
бар — бірақ ол **бір ағында ғана**. Көп ағын үшін — `Arc<T>` (Atomic
Reference Counted): аналог, бірақ есептеуіш атомдық, ағын-қауіпсіз.

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("Нәтиже: {}", *counter.lock().unwrap());
}
```

10 ағын — әрқайсысы есептеуішті бір көтереді. `Arc::clone(&counter)` —
есептеуішті көтереді (атомдық!). `move` арқылы клон ағынға өтеді.
Әр ағын: `lock` — мутекс ал; `*num += 1` — есептеуішті жоғары шығар;
аумақ бітсе — мутекс автоматты босатылады.

Барлық ағындар аяқталғанда — `counter` 10-ға тең. Ешбір жарыс шарты,
ешбір тосқауыл.

### `Mutex<T>` пен `RefCell<T>` ұқсастығы

`Mutex<T>` пен `RefCell<T>` арасында үлгі ұқсастығы бар:

- Екеуі де **ішкі өзгермелілік** ұсынады. Сыртта тұрақты, ішінде
  өзгертуге келеді.
- `RefCell<T>` тек бір ағында; `Mutex<T>` — ағындар арасында.
- `RefCell<T>` қарыз ережелерін **жұмыс кезінде** тексереді — бұзылса
  panic. `Mutex<T>` — басқа ағын `lock` ұстаса блок жасайды.

Қос-нышан:

| Бір ағын | Ағындар арасында |
|---|---|
| `Cell<T>` / `RefCell<T>` | `Mutex<T>` / `RwLock<T>` |
| `Rc<T>` | `Arc<T>` |

Бір ағында әдетте `RefCell` пен `Rc` жеткілікті; ағындар арасында —
`Mutex` пен `Arc`.

## 16.4. `Send` пен `Sync` — типтік қатарлас орындау интерфейсі

Барлық алдындағы кепілдіктер — компиляция кезіндегі. Олардың тірегі —
екі трейт:

- **`Send`** — типтің мәнін бір ағыннан екіншіге **иелікпен ауыстыруға**
  болады.
- **`Sync`** — типке бірнеше ағын **бір уақытта сілтеме арқылы**
  қол жеткізе алады.

Бұл екеуі — **маркер трейттер**. Олардың әдіс-денесі жоқ; тек тип
ағын-қауіпсіздікке сай ма, жоқ па деп білдіреді.

Көп тип `Send` мен `Sync`-ті бекітеді — компилятор оларды құрылымның
ішкі жайтына қарап анықтайды. Мысалы:

- `i32`, `String`, `Vec<i32>` — `Send` пен `Sync`.
- `Rc<T>` — `Send` емес, `Sync` емес. Себебі есептеуіші атомдық емес.
- `Arc<T>` — `Send` пен `Sync` (егер `T: Send + Sync` болса).
- `Mutex<T>` — `Send` пен `Sync` (`T: Send` шарты).
- `RefCell<T>` — `Send`, бірақ `Sync` емес. Себебі бірнеше ағын
  бір RefCell-ге сілтеме ұстаса — runtime borrow check қаупі бар.

`thread::spawn` функциясы өзінің жабуынан `Send` талап етеді:

```rust
fn spawn<F, T>(f: F) -> JoinHandle<T>
where
    F: FnOnce() -> T + Send + 'static,
    T: Send + 'static,
{
    // ...
}
```

`F: Send` — жабу ағынға ауысатынын; `'static` — ол өзі тарыққан
аумақтан кейін де тірі болатынын білдіреді. Сондықтан компилятор
тура осы шарттарды тексереді.

### Қолмен `Send`/`Sync` бекіту

`Send` мен `Sync` көп жағдайда автоматты түрде бекітіледі. Қолмен
бекіту — `unsafe`. Мысал, шикі көрсеткіш (`*const T` / `*mut T`) —
`Send` пен `Sync` емес. Бірақ сіздің кодыңыз белгілі бір инвариантты
кепілдеп, көрсеткішті ағындар арасында қауіпсіз қолдана алса —
`unsafe impl Send` жазып жасай аласыз. Бірақ онда қауіпсіздік сіздің
жауапкершілікке жатады.

Қарапайым ереже: бірінші — стандартты типтерді (`Mutex`, `Arc`)
қолданыңыз. `unsafe impl` — өте сирек.

## 16.5. Тарау қорытындысы

`Rust`-тың қатарлас орындау моделі екі тірекке негізделген:

1. **Компиляция-кездік типтік тексерулер** — `Send` пен `Sync`
   трейттері арқылы, иелік ережелерімен бірге.
2. **Кітапхана құралдары** — `thread::spawn`, `mpsc::channel`,
   `Mutex<T>`, `Arc<T>` — әрқайсысы өз сценарийі үшін.

Хабарлама-канал моделі — мәнді ағындар арасында **жіберу**. Ортақ
күй моделі — `Arc<Mutex<T>>` арқылы **бөлу**. Екі модель де `Rust`-та
тең қолданылатын; жобаға не жөндірек — соны таңдайсыз.

`Send` мен `Sync` — Rust қатарлас орындау моделінің концептуалды
негізі. Көбіне сізде олармен тікелей жұмыс істеудің керегі болмайды —
бірақ компилятор сізге ағын-қауіпсіздік туралы хабарлама шығарғанда —
осы ұғымдар арқылы оны түсінуге болады.

Келесі тарауда — `Rust`-тың объектілі-бағытталған бағдарламалау
тұрғысын қарастырамыз. Иерархияны жалпылама типтер мен трейттер арқылы
қалай жасауға болатынын; динамикалық сипаттаулы трейт-нысандарды
қалай қолдануды; кейбір объектілі-бағытталған үлгілерді `Rust`-та
қалай өрнектеуге болатынын көреміз.
