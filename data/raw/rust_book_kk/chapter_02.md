# 2-тарау. Санды табу ойыны

Осы тарауда нақты бағдарлама — санды табу ойыны — арқылы `Rust`-тың
бірнеше негізгі ұғымымен танысамыз. Ойынның жұмысы қарапайым: бағдарлама
кездейсоқ түрде 1-ден 100-ге дейінгі сан таңдайды, ойыншыдан санды табуды
сұрайды, әр болжамға «тым кіші», «тым үлкен» немесе «дұрыс» деп жауап
береді. Ойынша санды дұрыс тапқанша жалғасады.

Жол барысында сіз мынаны үйренесіз: айнымалыларды жариялау, пайдаланушы
енгізген мәтінді оқу, `Cargo` арқылы сыртқы сандықты қосу, `Result`
енамымен қатені өңдеу, `match` өрнегі, цикл құру және цикл ішінде шартқа
қарап шығу. Бұл тарау бүкіл кітаптың қалған бөлігінде кездесетін негізгі
ұғымдарды бір ұйымдасқан мысалда көрсетеді.

## 2.1. Жаңа жоба құру

`projects` қалтасына өтіп, `Cargo` арқылы жаңа жоба жасаңыз:

```sh
$ cargo new guessing_game
$ cd guessing_game
```

`cargo new` командасы сізге даяр құрылым ұсынады: `Cargo.toml` жобаны
сипаттайды, `src/main.rs` файлы алғашқы кодты сақтайды. Алдыңғы тарауда
қарағанымыздай, Cargo сізге дайын `Hello, world!` бағдарламасын ұсынып
қояды:

```rust
fn main() {
    println!("Hello, world!");
}
```

Жобаны бірден іске қосып, дайын құрылыс жұмыс істеп тұрғанын тексерейік:

```sh
$ cargo run
   Compiling guessing_game v0.1.0 (file:///projects/guessing_game)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 1.50 secs
     Running `target/debug/guessing_game`
Hello, world!
```

Жақсы. Енді нақты ойынды жазуға кірісейік.

## 2.2. Болжамды өңдеу

Бағдарламаның бірінші бөлігі — пайдаланушыдан санды сұрау, оны енгізуді
күту, енгізілген мәтінді оқу және оны экранда қайта көрсету. Файл
`src/main.rs` ішіндегі кодты келесімен ауыстырыңыз:

```rust
use std::io;

fn main() {
    println!("Санды табу ойыны!");
    println!("Болжамыңызды енгізіңіз.");

    let mut guess = String::new();

    io::stdin()
        .read_line(&mut guess)
        .expect("Жолды оқу мүмкін болмады");

    println!("Сіздің болжамыңыз: {guess}");
}
```

Жолма-жол қарастырайық. Бірінші жолда `use std::io;` сөйлемі тұр. Бұл
сөйлем `Rust`-тың стандартты кітапханасының `io` модулін аумаққа
енгізеді. Стандартты кітапхана көптеген пайдалы заттарды ұсынады, бірақ
олардың бәрі автоматты түрде қолжетімді емес: `io` сияқты бөлектерді
қолданғанда оларды нақты `use` арқылы шақыру керек.

`main` функциясының ішіндегі бірінші екі сөйлем — `println!` арқылы
ойынның тақырыбын мен нұсқауын экранға шығару.

Содан кейін:

```rust
    let mut guess = String::new();
```

Бұл сөйлем `guess` атты жаңа айнымалы жариялайды. `Rust`-та айнымалылар
әдетте тұрақты болады — олардың мәнін қайта тағайындау мүмкін емес. Бірақ
бізге пайдаланушының жауабын осы айнымалыға жазу керек, сондықтан `mut`
кілтсөзін қосамыз. `mut` өзгермелі дегенді білдіреді, яғни мәнін кейін
ауыстыруға болады.

Теңдік белгісінің оң жағында `String::new()` шақыруы тұр. Бұл стандартты
кітапхана ұсынатын `String` типінің жаңа, бос данасын қайтаратын
байланысты функция. `String` — өсетін, өзгертілетін, UTF-8 кодталған жол
түрі.

Әрі қарай:

```rust
    io::stdin()
        .read_line(&mut guess)
```

`io::stdin()` функциясы стандартты енгізуді көрсететін `Stdin` данасын
қайтарады. `.read_line(&mut guess)` әдісі стандартты енгізуден бір жолды
оқиды және оны `guess` айнымалысына жазады. Назар аударыңыз: біз `guess`
айнымалысына өзгермелі сілтеме береміз — `&mut guess`. `&` белгісі —
сілтеме, ал `mut` — өзгермелі. Сілтеме иелікті ауыстырмастан мәнге қол
жеткізуге мүмкіндік береді; өзгермелі сілтеме мәнді өзгертуге рұқсат
етеді.

`read_line` тек жолды жинақтап қана қоймайды, ол сонымен бірге `Result`
типін қайтарады. `Result` — стандартты кітапхана ұсынатын енам; нұсқалары
екеу — `Ok` (сәтті) және `Err` (қате). Көптеген функциялар сондай түрде
жазылған: операцияның сәтті немесе сәтсіз болуы мүмкін, нәтиже соған сай
қапталады.

Біз `Result`-ты `.expect("Жолды оқу мүмкін болмады")` арқылы өңдейміз.
Егер нәтиже `Err` болса, `expect` бағдарламаны тоқтатып, берілген
хабарламаны экранда көрсетеді. Егер нәтиже `Ok` болса, ол ішіндегі мәнді
қайтарады. Бұл — оңай, бірақ дөрекі тәсіл. 9-тарауда қатені нақтырақ
өңдеудің тәсілдерін көрсетеміз.

Соңғы сөйлем:

```rust
    println!("Сіздің болжамыңыз: {guess}");
```

`{guess}` — `println!` макросының орынтолтырғышы. Айнымалының мәнін
қапсырылған жолдың тиісті жеріне қояды. Бір сөйлемде бірнеше орынтолтырғыш
болуы мүмкін; олар тізбе бойынша аргументтерге сай келеді.

Бағдарламаны іске қосып көрейік:

```sh
$ cargo run
   Compiling guessing_game v0.1.0 (file:///projects/guessing_game)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 1.97 secs
     Running `target/debug/guessing_game`
Санды табу ойыны!
Болжамыңызды енгізіңіз.
42
Сіздің болжамыңыз: 42
```

Бірінші бөлік дайын — пайдаланушы енгізген сан экранға шықты.

## 2.3. Құпия санды генерациялау

Енді ойыншы табуы керек құпия санды жасайық. Әр ойын кезінде сан өзгеретін
болғаны жақсы — әйтпесе ойын қызықсыз. Кездейсоқ сан жасау үшін `rand`
сандығын қолданамыз. Бұл `Rust` стандартты кітапханасында жоқ, сондықтан
оны сыртқы тәуелділік ретінде қосу керек.

### Сыртқы сандықты қосу

`Cargo.toml` файлын ашып, `[dependencies]` бөліміне `rand` сандығын қосыңыз:

```toml
[dependencies]
rand = "0.8.5"
```

`Cargo.toml` ішіндегі әр сөйлем нақты бір бөлімге жатады. `[dependencies]`
бөлімі — жобаның тәуелділіктерін тізбелейтін орын. Мұнда `rand = "0.8.5"`
сөйлемі: «`rand` сандығының 0.8.5 нұсқасына сай немесе үйлесімді нұсқаны
қолдан» дегенді білдіреді. Бұл — семантикалық нұсқалаудың (semver)
шартты түрі: `^0.8.5` дегенге пара-пар, яғни 0.8.5-тен жоғары, бірақ
0.9.0-ден төмен кез келген нұсқа жарамды.

Енді `cargo build` командасын іске қосыңыз. Cargo сандықтың
тізілімі — crates.io — арқылы `rand` пен оған тәуелді басқа сандықтарды
жүктеп алады және бәрін бірге құрастырады:

```sh
$ cargo build
    Updating crates.io index
  Downloaded rand v0.8.5
  Downloaded ppv-lite86 v0.2.10
  Downloaded getrandom v0.2.10
  ...
   Compiling rand_core v0.6.4
   Compiling rand_chacha v0.3.1
   Compiling rand v0.8.5
   Compiling guessing_game v0.1.0 (file:///projects/guessing_game)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 2.53s
```

Алғашқы құрастырудан кейін `Cargo.lock` файлы жасалады. Ол сіздің жобаңыз
қандай нақты нұсқалармен құрастырылғанын тіркейді — басқа машинада
немесе келесі құрастыруда дәл сондай нұсқалар қолданылсын дегенді
кепілдейді.

### Кездейсоқ санды жасау

Енді кодқа кездейсоқ сан генерациясын қосайық. `src/main.rs` файлының
басына жаңа `use` сөйлемін, ал кодтың ішіне сан жасайтын бөлік қосыңыз:

```rust
use std::io;
use rand::Rng;

fn main() {
    println!("Санды табу ойыны!");

    let secret_number = rand::thread_rng().gen_range(1..=100);

    println!("Құпия сан: {secret_number}");

    println!("Болжамыңызды енгізіңіз.");

    let mut guess = String::new();

    io::stdin()
        .read_line(&mut guess)
        .expect("Жолды оқу мүмкін болмады");

    println!("Сіздің болжамыңыз: {guess}");
}
```

`use rand::Rng;` сөйлемі `Rng` трейтін аумаққа енгізеді. Бұл трейт
кездейсоқ сан генераторларынан күтілетін әдістерді сипаттайды. 10-тарауда
трейттер туралы толығырақ айтамыз; қазір бізге `Rng` арқылы `gen_range`
әдісіне қол жеткізу қажет екенін білу жеткілікті.

`rand::thread_rng()` функциясы біз қолданатын кездейсоқ сан генераторын
қайтарады — ол операциялық жүйеден алынады және ағынға тиесілі.
`.gen_range(1..=100)` шақыруы 1-ден 100-ге дейінгі (екі шеті де қосылған)
кездейсоқ бүтін санды қайтарады. `1..=100` — бұл диапазон өрнегі.

Сонымен, `let secret_number = rand::thread_rng().gen_range(1..=100);`
сөйлемі құпия санды құрып, оны `secret_number` атты тұрақты айнымалыға
жазады. Бағдарламаны бірнеше рет іске қосыңыз — әр рет әр түрлі сан
шығатын болады.

> **Ескерту.** Әзірге `Құпия сан: {secret_number}` жолын қалдырыңыз — ол
> жөндеу үшін ыңғайлы. Кейін, ойынды толық жасап болғанда, бұл жолды
> өшіріп тастаймыз.

## 2.4. Болжамды құпия санмен салыстыру

Енді ойыншының болжамын құпия санмен салыстырып, нәтижесін айту керек.
Бұл үшін `match` өрнегі мен `std::cmp::Ordering` енамын қолданамыз. Файлдың
басына тағы бір `use` сөйлемін қосыңыз және салыстыратын `match` блогын
жазыңыз:

```rust
use std::cmp::Ordering;
use std::io;
use rand::Rng;

fn main() {
    // ... алдыңғы код өзгеріссіз ...

    println!("Сіздің болжамыңыз: {guess}");

    match guess.cmp(&secret_number) {
        Ordering::Less => println!("Тым кіші!"),
        Ordering::Greater => println!("Тым үлкен!"),
        Ordering::Equal => println!("Дәл таптыңыз!"),
    }
}
```

`Ordering` — стандартты кітапхана ұсынатын енам, оның үш нұсқасы бар:
`Less` (кіші), `Greater` (үлкен), `Equal` (тең). `cmp` әдісі екі мәнді
салыстырып, осы нұсқалардың бірін қайтарады.

`match` өрнегі салыстыру нәтижесіне қарай тармақ таңдайды. Әр тармақ
өрнектің ұқсастық үлгісімен (`Ordering::Less`, `Ordering::Greater`,
`Ordering::Equal`) және оған сай орындалатын кодпен жазылады. `match`
өрнектерін `Rust` бағдарламашысы күн сайын қолданады; 6-тарауда
оларға арналған толық бөлім бар.

### Тип сәйкестігі мәселесі

Бірақ бұл код әлі құрастырылмайды. `cargo build` командасын іске қосыңыз
және мынандай қатені көресіз:

```text
error[E0308]: mismatched types
   --> src/main.rs:22:21
    |
22  |     match guess.cmp(&secret_number) {
    |                 --- ^^^^^^^^^^^^^^ expected `&String`, found `&{integer}`
    |                 |
    |                 arguments to this method are incorrect
```

Қате — типтердің сәйкес келмеуі. `guess` — `String`, ал `secret_number` —
бүтін сан. `Rust` екеуін салыстыруға келіспейді, өйткені олар әртүрлі
типтер.

Бұл мәселені шешу үшін `guess` ішіндегі мәтінді бүтін санға айналдыру
керек. Кодты келесімен жалғастырыңыз:

```rust
    let guess: u32 = guess.trim().parse().expect("Сан енгізіңіз!");
```

Маңызды нәрсе: біз `guess` атты жаңа айнымалы жариялаймыз. Бұл бұрынғы
`guess`-ті *көлеңкелейді*, яғни жаңа атпен жаңа мәнді бекітеміз. `Rust`-та
көлеңкелеу — әдеттегі тәсіл; ол бір айнымалының түрін өзгертіп, бірақ оған
жаңа атау таппай-ақ жалғастырғанда қолайлы. Мысалы, бұл жерде ескі `guess`
— `String`, жаңа `guess` — `u32`.

`guess.trim()` — жолдың алды-артындағы бос орындар мен жол аяқталу
символын алып тастайды (`read_line` соңында `\n` қалдырады). `parse()`
әдісі жолды бір санға айналдырады. `parse` қандай түрге айналдыратынын
білу үшін біз нақты тип белгіледік: `let guess: u32`. Бұл — `Rust`-қа
«жолды 32 биттік таңбасыз бүтін санға айналдыр» деген сигнал.

`parse` қатеге ұшырауы мүмкін: егер енгізілген мәтін санға келмесе
(мысалы, «hello»). Сондықтан `parse` `Result` қайтарады, ал біз оны
`.expect("Сан енгізіңіз!")` арқылы өңдейміз.

`u32` түрін `secret_number`-мен салыстыру үшін `Rust` екі санның түрін
бірге шешеді: `secret_number` бұрын белгілі бір бүтін сан түрі болмаған
(тек `i32` әдепкісі), бірақ `guess` — `u32`. Сондықтан `secret_number`
де `u32` болады. Бұл түр-қорытынды (type inference) деп аталады.

Енді `cargo run` командасын тағы іске қосыңыз — салыстыру жұмыс істейді:

```sh
$ cargo run
   Compiling guessing_game v0.1.0 (file:///projects/guessing_game)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.43s
     Running `target/debug/guessing_game`
Санды табу ойыны!
Құпия сан: 7
Болжамыңызды енгізіңіз.
50
Сіздің болжамыңыз: 50
Тым үлкен!
```

## 2.5. Цикл арқылы бірнеше болжамға рұқсат беру

Қазір ойын тек бір болжамды қабылдайды да тоқтайды. Жөндейік: бағдарлама
ойыншыдан бірнеше рет санды сұрап, әр рет жауап беріп отырсын. Ол үшін
`loop` кілтсөзін қолданамыз. Кодтың `let secret_number` сөйлемінен
кейінгі бөлігін `loop { ... }` блогына салыңыз:

```rust
    println!("Құпия сан: {secret_number}");

    loop {
        println!("Болжамыңызды енгізіңіз.");

        let mut guess = String::new();

        io::stdin()
            .read_line(&mut guess)
            .expect("Жолды оқу мүмкін болмады");

        let guess: u32 = guess.trim().parse().expect("Сан енгізіңіз!");

        println!("Сіздің болжамыңыз: {guess}");

        match guess.cmp(&secret_number) {
            Ordering::Less => println!("Тым кіші!"),
            Ordering::Greater => println!("Тым үлкен!"),
            Ordering::Equal => println!("Дәл таптыңыз!"),
        }
    }
```

`loop` шарт жоқ цикл — бағдарлама циклден тек `break` арқылы шыға алады.
Қазір `break` жоқ, сондықтан ойын мәңгілікке жалғасады. Терминалда
бағдарламаны `Ctrl-C` арқылы тоқтатыңыз.

### Дұрыс болжам кезінде шығу

Ойынды дұрыс жобалау үшін: ойыншы дұрыс жауап тапса, бағдарлама циклден
шықсын. `Ordering::Equal` тармағында `break` сөзін қосыңыз:

```rust
        match guess.cmp(&secret_number) {
            Ordering::Less => println!("Тым кіші!"),
            Ordering::Greater => println!("Тым үлкен!"),
            Ordering::Equal => {
                println!("Дәл таптыңыз!");
                break;
            }
        }
```

Назар аударыңыз: `Ordering::Equal` тармағында енді бір емес, екі сөйлем
бар. Сондықтан оның денесін фигуралық жақшалармен ораймыз. `break`
циклден ағымдағы итерацияны үзіп шығады.

## 2.6. Қате енгізілген санды қалай өңдеу

Қазіргі бағдарлама `parse` ішіндегі `expect` арқасында қателі енгізу
кезінде apatка ұшырайды:

```sh
$ cargo run
Болжамыңызды енгізіңіз.
hello
thread 'main' panicked at 'Сан енгізіңіз!: ...', src/main.rs:19:36
```

Бұл — пайдаланушы үшін жұмсақ тәжірибе емес. Қатеге сүрінбей, бағдарлама
тек қана сұрауды қайталайтын болсын. Парсингтің `Result` түрін `match`
арқылы тікелей өңдеуге болады:

```rust
        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,
            Err(_) => continue,
        };
```

`parse` `Result` қайтарады. Біз оны `match` арқылы тексереміз: егер `Ok`
болса (сан сәтті алынса), `num` мәнін шығарамыз; егер `Err` болса, цикл
басына қайтып, келесі итерацияны бастаймыз. `_` — бұл «қандай нақты
қатеге қарамаймыз» деген ұқсастық үлгісі. `continue` цикл денесінің
қалған бөлігін өткізіп жіберіп, келесі итерацияға өтеді.

Енді бағдарлама толық болды:

```rust
use std::cmp::Ordering;
use std::io;
use rand::Rng;

fn main() {
    println!("Санды табу ойыны!");

    let secret_number = rand::thread_rng().gen_range(1..=100);

    loop {
        println!("Болжамыңызды енгізіңіз.");

        let mut guess = String::new();

        io::stdin()
            .read_line(&mut guess)
            .expect("Жолды оқу мүмкін болмады");

        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,
            Err(_) => continue,
        };

        println!("Сіздің болжамыңыз: {guess}");

        match guess.cmp(&secret_number) {
            Ordering::Less => println!("Тым кіші!"),
            Ordering::Greater => println!("Тым үлкен!"),
            Ordering::Equal => {
                println!("Дәл таптыңыз!");
                break;
            }
        }
    }
}
```

> **Ескерту.** Жөндеу мақсатында тұрған `println!("Құпия сан: ...")`
> жолын кодтан өшіріңіз — ойыншы құпия санды көрмеуі керек.

`cargo run` арқылы ойынды бірнеше рет ойнап көріңіз. Бағдарлама енді:

- құпия санды кездейсоқ түрде таңдайды;
- пайдаланушы дұрыс санды табуды күтіп, болжамдарды қабылдайды;
- әр болжамға «тым кіші», «тым үлкен» деп жауап береді;
- қате енгізілген мәтінді тыныш қабылдап, қайтадан сұрайды;
- дұрыс жауап табылғанда «Дәл таптыңыз!» деп шығады.

## 2.7. Тарау қорытындысы

Бұл тарауда сіз `Rust`-тың бірнеше негізгі ұғымымен жұмыс істеуді
үйрендіңіз: `let` арқылы айнымалыларды жариялау, `mut` арқылы оларды
өзгермелі ету, `match` өрнегімен тармақтау, `loop` арқылы цикл жасау,
`break` арқылы циклден шығу, `Result` енамымен қатені өңдеу, `Cargo.toml`
арқылы сыртқы сандықты қосу. Сонымен қатар сіз `String`, `u32` сияқты
типтермен танысып, көлеңкелеу мен типке сай талдау сияқты тәсілдерді
пайдаландыңыз.

Келесі тарауда осы ұғымдардың әрқайсысын тереңірек қарастырамыз: `Rust`-тың
негізгі типтері, айнымалылар мен тұрақтылар, функциялар мен басқару
құрылымдары жайлы егжей-тегжей айтамыз.
