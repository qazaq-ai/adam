# 8-тарау. Жалпы ұжымдар

`Rust`-тың стандартты кітапханасы бірнеше пайдалы **ұжымдық** деректер
құрылымын ұсынады. Ұжым — бір немесе бірнеше мәнді сақтайтын құрылым.
Жиым (`array`) мен кортежден айырмашылығы — ұжымдар үймеде сақталады,
сондықтан олардың саны компиляция кезінде белгілі болмауы мүмкін.
Бағдарлама орындалу кезінде ұжым өсіп немесе кішірейе алады.

Бұл тарауда үш жалпы қолданылатын ұжыммен танысамыз:

- **Vector (`Vec<T>`)** — өсетін, сан бойынша индекстелетін бір типтегі
  мәндердің тізбегі.
- **String (`String`)** — UTF-8 кодталған өсетін мәтін.
- **Hash Map (`HashMap<K, V>`)** — кілт пен мәнді жұптап сақтайтын,
  кілт бойынша жылдам іздейтін құрылым.

Үшеуінің ортақ қасиеті — ұзындығы орындалу кезінде өзгереді әрі деректері
үймеде сақталады. Әр ұжымның мақсаты, өнімділік сипаттамасы, ыңғайлы
қолданысы әр түрлі. Стандартты кітапханада басқа да ұжымдар бар (`VecDeque`,
`BTreeMap`, `HashSet` және басқалары) — олар арнайы жағдайларға арналған.

## 8.1. Vector — мәндер тізбегін сақтау

`Vec<T>` (ауызекі тілде «вектор») — өсетін жиым іспетті, бір типтегі
мәндерді сан бойынша индекстеп сақтайды. Барлық элементтер бір типке
тиесілі болуы керек.

### Жаңа вектор жасау

Бос вектор жасау үшін `Vec::new` шақырамыз. Тип белгісі бергеніміз —
вектордың қандай мәндерді ұстайтынын `Rust`-қа айту:

```rust
let v: Vec<i32> = Vec::new();
```

Көп жағдайда вектор алғашқы мәндермен бірден жасалады, сондықтан түрді
қолмен белгілеудің керегі болмайды. Соған арналған `vec!` макросы:

```rust
let v = vec![1, 2, 3];
```

`vec![1, 2, 3]` — `Vec<i32>` түріндегі вектор, үш элементі бар. `Rust`
типті өзі шығарып алады.

### Векторды жаңарту

Векторға элемент қосу үшін `push` әдісін қолданамыз. Бұл вектордың
өзгеретінін білдіреді — оны `mut` етіп жариялау керек:

```rust
let mut v = Vec::new();
v.push(5);
v.push(6);
v.push(7);
v.push(8);
```

`push` алғашқы шақыруда `i32` берілгендіктен, `Rust` `v`-ні `Vec<i32>`
ретінде шешеді. Бұдан былай оған тек `i32` мәндерін қосуға болады.

### Вектор элементтеріне қол жеткізу

Вектор элементін екі тәсілмен оқуға болады: индекс арқылы (`&v[i]`) және
`get` әдісі арқылы (`v.get(i)`).

```rust
let v = vec![1, 2, 3, 4, 5];

let third: &i32 = &v[2];
println!("Үшінші элемент: {}", third);

let third: Option<&i32> = v.get(2);
match third {
    Some(third) => println!("Үшінші элемент: {}", third),
    None => println!("Үшінші элементі жоқ."),
}
```

Айырмашылық — қате болған жағдайда не болады:

- `&v[i]` — индекс шектен шықса panic-қа барады. Бағдарлама тоқтайды.
- `v.get(i)` — `Option<&T>` қайтарады. Шектен тыс болса `None`. Бұл —
  күтілетін, өңдеуге келетін қате.

Қайсысын таңдау керек? Көбіне индекс шегінде болатынына сенімді болсаңыз —
`&v[i]` оқуға жеңіл. Сенімді болмасаңыз — `get` арқылы Option-ды нақты
өңдеген жөн.

### Иелік пен сілтемелерге қатысты ережелер

Векторды қарызға алу мен өзгерту әдеттегі сілтеме ережелеріне бағынады.
Мысалы, бар тұрақты сілтеме мен өзгермелі сілтеме бір уақытта болмауы
керек:

```rust
let mut v = vec![1, 2, 3, 4, 5];

let first = &v[0];

v.push(6); // ҚАТЕ: first әлі белсенді тұрса

println!("Бірінші: {}", first);
```

Неге қате? Себебі `push` векторды үлкейтуі мүмкін — соған арналған жадты
жаңа жерге көшіруі мүмкін. Сонда `first`-тің көрсеткіші ескі (босатылған)
жерге сілтеп қалады. `Rust` бұған тыйым салады — компилятор `push`
үшін өзгермелі сілтеме қажет екенін, ал бұрыннан `first` тұрақты сілтеме
ұстайтынын біледі.

### Вектор бойынша жүру

Вектордың әр элементін аралау үшін `for` цикл қолданамыз:

```rust
let v = vec![100, 32, 57];
for i in &v {
    println!("{}", i);
}
```

Әр элементті өзгерту керек болса — өзгермелі сілтемемен:

```rust
let mut v = vec![100, 32, 57];
for i in &mut v {
    *i += 50;
}
```

`*i` — **деreferense** (сілтеме қарама-қарсы). `i` — `&mut i32`
түріндегі сілтеме; `*i` — оның ішіндегі `i32`-ге қол жеткізеді.
Сілтемемен жұмыс істегенде осы дереференс операторы кездесетін болады.

### Әр түрлі типтерді бір векторда сақтау: енам арқылы

Векторда барлық элементтер бір типте болуы керек. Бұл шектеу ыңғайсыз
болуы мүмкін. Шешімі — енамды қолдану. Енам әртүрлі нұсқаларды бір
типке біріктіреді:

```rust
enum SpreadsheetCell {
    Int(i32),
    Float(f64),
    Text(String),
}

let row = vec![
    SpreadsheetCell::Int(3),
    SpreadsheetCell::Text(String::from("көк")),
    SpreadsheetCell::Float(10.12),
];
```

Енді вектор үш әр түрлі мәнді сақтайды, бірақ типтік жүйе тұрғысынан
олардың бәрі — `SpreadsheetCell`. `match` арқылы қай нұсқа екенін шешуге
болады.

### Вектор аяқталғанда

Вектор өз аумағынан шыққанда — `drop` шақырылады, оның барлық элементтері
де өшіріледі. `Rust` бұл туралы алаңдамауыңызды кепіл етеді: бір вектор
жоғалғанда, оның элементтері де жойылады, ешбір ағытпа болмайды.

## 8.2. UTF-8 кодталған мәтінді String арқылы сақтау

`String` — `Rust`-та өсетін, өзгертілетін, иелігі бар UTF-8 кодталған
мәтінді сақтайтын тип. Жол литералы (`&str`) — компиляция кезінде белгілі
тұрақты мәтін; `String` — орындалу кезінде құрылатын динамикалық мәтін.

`Rust`-тың жолдары — басқа тілдермен салыстырғанда күрделі. Себебі
`Rust` UTF-8 кодтауын бекітіп, барлық жолдарды қазақ, эмодзи, әр түрлі
тілдердің таңбаларымен жұмыс істеуге дайындайды. Бұл — қуат, бірақ
кейбір қарапайым операциялар (мысалы, индекстеу) күтпеген жерде күрделі
болады.

### Жаңа жол жасау

Бос жолды `String::new` арқылы жасаймыз:

```rust
let mut s = String::new();
```

Жол литералынан жасау үшін `to_string` немесе `String::from`:

```rust
let data = "бастапқы";
let s = data.to_string();
let s = "бастапқы".to_string();
let s = String::from("бастапқы");
```

Үшеуі де бір нәтиже береді. Стиль — қалаған жолыңыз.

### Жолды жаңарту

Жолды өсіру жалпыдан арнайыға қарай: бір таңба, жол үстіне, екі жолды
қосу.

`push_str` — жолға басқа жолды (`&str`) жалғайды:

```rust
let mut s = String::from("сәлем");
s.push_str(", дүние!");
println!("{}", s); // сәлем, дүние!
```

`push_str` өзіне берілген аргументтің иелігін алмайды — `&str` сілтеме
ғана. Сондықтан төмендегі код жұмыс істейді:

```rust
let mut s1 = String::from("сәлем, ");
let s2 = String::from("дүние!");
s1.push_str(&s2);
println!("s2 — {}", s2); // s2 әлі бар
```

`push` — бір таңба қосады:

```rust
let mut s = String::from("Рас");
s.push('т');
println!("{}", s); // "Раст"
```

`+` операторы — екі жолды біріктіреді, бірақ нюанс бар: иелікті
ауыстырады:

```rust
let s1 = String::from("сәлем, ");
let s2 = String::from("дүние!");
let s3 = s1 + &s2; // s1 енді жарамсыз
```

Себебі `+` операторы шынында `add` әдісін шақырады, оның қолтаңбасы
шамамен мынадай: `fn add(self, other: &str) -> String`. `self` иелікті
алады; нәтиже жаңа `String`. Сондықтан `s1` иелігін жоғалтады.

Үш жолды біріктіру керек болса — `+` операторы шатастырады. Сондықтан
`format!` макросын қолданайық:

```rust
let s1 = String::from("шие");
let s2 = String::from("қызыл");
let s3 = String::from("гүл");

let s = format!("{}-{}-{}", s1, s2, s3);
println!("{}", s); // шие-қызыл-гүл
// s1, s2, s3 — әлі жарамды
```

`format!` арзан, оқуға жеңіл, иелікті алмайды. Бірнеше жолды біріктіру
үшін — соны қолданыңыз.

### Жолды индекстеу

Көп тілдерде жолдағы белгілі бір таңбаны `s[i]` арқылы алуға болады.
`Rust`-та бұл мүмкін емес:

```rust
let s = String::from("сәлем");
let h = s[0]; // ҚАТЕ: String — индекстелмейді
```

Неге? Себебі UTF-8 кодтауында бір таңба бір байт алмауы мүмкін. Мысалы:

```rust
let hello = "Здравствуйте";
```

— бұл орыс жолының ұзындығы 24 байт (12 таңба × 2 байт), бірақ көзбен
қарасаңыз 12 таңба. Қазақ кириллицасы да солай — әр таңба екі байт алады.
Эмодзи — төрт байт.

Сондықтан `&s[0]` нені қайтару керек? Бірінші **байтты** ма? Бірінші
**таңбаны** ма? Бірінші **графема кластерін** ма (көзбен бір таңба
ретінде көрінетін бөлік)? Әртүрлі сұраққа әртүрлі жауап. `Rust` бұл
шатастыратын мәселеден аулақ — индекстеуге тыйым салынған.

Орнына — әр түрлі сұраққа арналған әдістер бар.

### Жолдан тілім алу

Аралық тілім (`&s[a..b]`) — байт аралығымен:

```rust
let hello = "Здравствуйте";
let s = &hello[0..4]; // "Зд" — алғашқы 4 байт
```

Бұл әлі де байт-аралықпен жұмыс істейді. Егер сіз байт шегінде емес
жерде қиып алсаңыз — panic. Мысалы `&hello[0..1]` — panic, өйткені бір
байт «З» таңбасының жартысы ғана.

### Жолды аралау

Әр **таңбаны** (Unicode скаляр-мән) аралау үшін `chars`:

```rust
for c in "сәлем".chars() {
    println!("{}", c);
}
// с
// ә
// л
// е
// м
```

Әр **байтты** аралау үшін `bytes`:

```rust
for b in "сәлем".bytes() {
    println!("{}", b);
}
```

Графема кластерлерін (көзге бір таңба ретінде көрінетін бөліктерді)
аралау стандартты кітапханада жоқ — оған сыртқы сандық қолдану керек.
Бұл саналы шешім: графема кластерін есептеу күрделі.

## 8.3. Кілт-мән жұптарын Hash Map арқылы сақтау

`HashMap<K, V>` — кілт пен мәнді жұптап сақтайтын ұжым. Кілт арқылы
мәнді жылдам табуға болады. Кейбір тілдерде «dictionary», «associative
array», «hash» немесе «map» деп аталатын құрылым.

### Жаңа hash map жасау

Hash map стандартты кітапханада тұр, бірақ оны қолдану үшін `use`
сөйлемі керек:

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();
scores.insert(String::from("Көк"), 10);
scores.insert(String::from("Сары"), 50);
```

`HashMap::new` — бос hash map жасайды. Кілт мен мәннің типтері алғашқы
`insert` шақыруларынан шығарылады. Барлық кілттер бір типте, барлық
мәндер бір типте болуы керек.

### Мәндерге қол жеткізу

Кілтпен мәнді табуға `get` әдісін қолданамыз:

```rust
use std::collections::HashMap;

let mut scores = HashMap::new();
scores.insert(String::from("Көк"), 10);
scores.insert(String::from("Сары"), 50);

let team_name = String::from("Көк");
let score = scores.get(&team_name).copied().unwrap_or(0);
```

`get` `Option<&V>` қайтарады — кілт жоқ болса `None`. `.copied()` ішкі
мәнді көшіріп, `Option<V>`-ге айналдырады. `.unwrap_or(0)` `Some`-ты
ашады, `None` болса 0 қайтарады.

Hash map бойынша жүру — `for` циклмен. Бірақ ретте: hash map ретсіз —
итерацияның реті анықталмаған:

```rust
for (key, value) in &scores {
    println!("{}: {}", key, value);
}
```

### Hash map пен иелік

`Copy` бекітілмеген типтерді (`String` сияқты) hash map-қа кіргізсеңіз —
иелік hash map-қа ауысады:

```rust
use std::collections::HashMap;

let field_name = String::from("Жасы");
let field_value = String::from("30");

let mut map = HashMap::new();
map.insert(field_name, field_value);
// field_name мен field_value енді жарамсыз
```

`i32` сияқты `Copy` типтер — hash map-қа көшіріледі, бастапқысы әлі
жарамды.

### Hash map жаңарту

Hash map-та бір кілттің тек бір мәні болады. Жаңарту тәсілдері:

**1. Мәнді басып ауыстыру.** `insert` тек қосып қана қоймайды — кілт
бұрыннан болса, оның мәнін жаңалап жазады:

```rust
scores.insert(String::from("Көк"), 10);
scores.insert(String::from("Көк"), 25); // 10 → 25
```

**2. Тек жоқ болғанда қосу.** `entry` API-ды қолданамыз:

```rust
scores.entry(String::from("Сары")).or_insert(50);
scores.entry(String::from("Көк")).or_insert(50); // Көк бұрыннан бар → еш нәрсе болмайды
```

`or_insert` кілт бар болса — оның бар мәніне сілтеме қайтарады; жоқ
болса — берген мәнді кіргізіп, оның сілтемесін қайтарады.

**3. Бар мәнге сүйеніп жаңарту.** Сөздерді санайтын мысал:

```rust
use std::collections::HashMap;

let text = "Сәлем дүние Сәлем әлем";
let mut map = HashMap::new();
for word in text.split_whitespace() {
    let count = map.entry(word).or_insert(0);
    *count += 1;
}
```

`or_insert` — `&mut V` сілтеме қайтарады. Біз оны `*count += 1` арқылы
дереференстеп өзгертеміз. Циклдан кейін `map`-та әр сөз қайталанған
санын ұстайды.

### Hash функциялары

`HashMap` әдепкі бойынша **SipHash** атты криптографиялық қауіпсіз hash
функциясын қолданады. Бұл — DoS шабуылдарына қарсы қорғаныс. Мысалы,
сіз веб-серверде көп пайдаланушыдан кілт қабылдасаңыз — олар арнайы
кілттерді жасап, hash функциясын тиімсіз жұмыс істетуге тырысуы мүмкін.
SipHash осыған қарсы тұрады — ал тұрады биік ескертумен: ол ең жылдам
hash функциясы емес.

Жылдамдық сізге өте маңызды болса — `BuildHasher` трейтін бекітетін
басқа hash алгоритмін қолдануға болады. `crates.io`-да бірнеше дайын
нұсқа бар (`fxhash`, `ahash` және басқалары).

## 8.4. Тарау қорытындысы

Үш жалпы ұжым: `Vec<T>` — индекстелетін бір типтегі тізбек;
`String` — UTF-8 мәтін; `HashMap<K, V>` — кілт-мән жұптары. Әр ұжымның
өзіндік пайдасы, шектеулері мен идиомалары бар.

`Vec` индекстеу мен push/pop үшін жылдам, бір типті ұжымдар үшін әдепкі
таңдау. `String` — `Rust`-тың UTF-8-ге адалдығының салдары; индекстеу
жоқ, ал `chars` пен `bytes` арқылы аралау бар. `HashMap` — кілт бойынша
іздеу қажет болғанда; иелік ережелеріне бағынады.

Стандартты кітапхана басқа ұжымдарды да ұсынады: `VecDeque` — екі жағынан
қосуға келетін кезек, `BTreeMap` — реттелген кілт-мән, `HashSet` —
қайталанбайтын элементтер жинағы. Олар арнайы жағдайларда жеңіл әрі
тиімді шешімдер береді.

Келесі тарауда `Rust`-тың қателерді өңдеу жүйесін қарастырамыз — `panic!`
пен `Result<T, E>` түрінің практикалық қолданыстарын, `?` операторын,
қашан панелге барып, қашан қатені шақырушыға қайтарып жіберу керектігін.
