# 18-тарау. Үлгілер мен сай келтіру

Үлгі (pattern) — `Rust`-тың арнайы синтаксисі, мәннің құрылымына сай
келтіру үшін. Үлгі арқылы біз: мәннің ішкі бөліктерін ажырата аламыз;
оларды атаулы айнымалыларға бөлектей аламыз; қандайда бір нұсқаны
сүзе аламыз. `match` пен `if let`-ті біз бұрын көрдік — олар үлгі
қолданатын жерлер.

Бұл тарауда:

1. Үлгілер қандай орындарда қолданылады.
2. **Жоққа шығаруға келетін** vs **жоққа шығаруға келмейтін**
   үлгілер.
3. Үлгілер синтаксисінің толық тізімі.

## 18.1. Үлгілер қандай орындарда қолданылады

### `match` тармақтары

Ең әйгілі қолданыс — `match` өрнегі:

```rust
match value {
    pattern1 => expression1,
    pattern2 => expression2,
    _ => default_expression,
}
```

Әр тармақ бір үлгіні және оған сай орындалатын кодты ұстайды. `match`
толық қамтылуы керек — барлық мүмкін мәндерді қарастыру.

### `if let` өрнектері

`if let` — бір үлгіге сай болғанда орындалатын код:

```rust
if let Some(color) = favorite_color {
    println!("Сізге {} түсі ұнайды!", color);
} else if is_tuesday {
    println!("Бүгін сейсенбі — жасыл түс жақсы!");
} else if let Ok(age) = age {
    if age > 30 {
        println!("Күлгін түсі қызық!");
    } else {
        println!("Қызыл түс ұнайды!");
    }
} else {
    println!("Көк түсі көрімді.");
}
```

`if let` жоққа шығаруға келетін үлгілерді қабылдайды (көп жағдайда
`Option` немесе `Result`). Сай болмаса — `else` тармағы орындалады.

### `while let` циклы

`while let` — үлгі сай болғанша қайталанатын цикл:

```rust
let mut stack = Vec::new();
stack.push(1);
stack.push(2);
stack.push(3);

while let Some(top) = stack.pop() {
    println!("{}", top);
}
```

`stack.pop()` `Option<T>` қайтарады. `Some(top)` сай болса — цикл
жалғасады; `None` болса — тоқтайды. Шығарады: `3`, `2`, `1`.

### `for` цикл

`for` циклында да үлгі қолданылады:

```rust
let v = vec!['a', 'b', 'c'];

for (index, value) in v.iter().enumerate() {
    println!("{} — {} индексте", value, index);
}
```

`(index, value)` — кортежді бөлшектеу үлгісі. `enumerate` `(usize, T)`
жұптарын береді; `for` оларды бөлшектейді.

### `let` сөйлемдері

`let` те үлгі қабылдайды:

```rust
let (x, y, z) = (1, 2, 3);
```

`(x, y, z)` — үлгі. Кортежді бөлшектеп, үш атаулы айнымалыға бөлеміз.

`let x = 5;` — қарапайым жағдай, бірақ ол да үлгі: `x` — атау-үлгісі;
ол кез келген мәнге сай болады.

### Функция параметрлері

Функция параметрлері — солай үлгі. Қарапайым `fn foo(x: i32)`-те `x`
атау-үлгісі. Бірақ кортежді тікелей бөлшектеп қабылдауға болады:

```rust
fn print_coordinates(&(x, y): &(i32, i32)) {
    println!("Координаттар: ({}, {})", x, y);
}

fn main() {
    let point = (3, 5);
    print_coordinates(&point);
}
```

`&(x, y)` — параметрді бөлшектеу. Функцияға кортеж сілтемесі келеді,
ол ішінде екі бөлек айнымалыға таратылады.

## 18.2. Қабылдамаушылық: үлгінің сай болмауы мүмкіндігі

Үлгінің екі үлкен түрі бар:

- **Жоққа шығаруға келмейтін** (irrefutable) үлгі — кез келген мәнге
  сай. Сай болмауы мүмкін емес.
- **Жоққа шығаруға келетін** (refutable) үлгі — кейбір мәндерге сай
  болмауы мүмкін.

Мысалы:

- `let x = 5;` — `x` атау-үлгісі, **жоққа шығаруға келмейтін**.
  Кез келген мәнге сай.
- `if let Some(x) = a_value` — `Some(x)` **жоққа шығаруға келетін**.
  `a_value` `None` болса — сай болмайды.

`let`, `for`, функция параметрлері — **жоққа шығаруға келмейтін**
үлгілерді ғана қабылдайды (өйткені сай болмаса не істеу керек —
ережесі жоқ).

`if let`, `while let`, `match` — қабылдамаушы үлгілерді қабылдайды
(олардың сай болмауын өңдейтін бөлек тармақтары бар).

Сондықтан мынадай код қате:

```rust
let Some(x) = some_option_value;
```

Қате: `Some(x)` — қабылдамаушы; `let` оны қолдамайды. Шешім —
`if let` немесе `let else`:

```rust
let Some(x) = some_option_value else {
    return;
};
```

`let else` (Rust 1.65+) — `Some` сай болса жалғасады; `None` болса
`else` блогы орындалады, ол мүлдем `return` немесе `panic` сияқты
шығарушы өрнекпен аяқталу керек.

## 18.3. Үлгілер синтаксисі

Үлгілер синтаксисі мол. Бар нұсқаларды қарастырайық.

### Литералдарға сай келтіру

```rust
let x = 1;

match x {
    1 => println!("бір"),
    2 => println!("екі"),
    3 => println!("үш"),
    _ => println!("басқа"),
}
```

Литерал — `1`, `'a'`, `"сәлем"` — нақты мәнге сай.

### Атаулы айнымалылар

Атаулы үлгі — кез келген мәнге сай, мәнді сол атпен ұстайды:

```rust
let x = Some(5);

match x {
    Some(50) => println!("Елу"),
    Some(y) => println!("y = {}", y),
    _ => println!("басқа"),
}
```

`Some(y)` — `Some`-ге сай. Ішіндегі мәнді `y`-ке байлайды. Бірақ
ескертіңіз: `match` өзінің жаңа аумағын ашады, сондықтан ішкі `y`
сыртқы кез келген `y`-ден бөлек.

### Көп үлгі (`|`)

Бірнеше үлгіні бір тармаққа біріктіру:

```rust
let x = 1;

match x {
    1 | 2 => println!("бір немесе екі"),
    3 => println!("үш"),
    _ => println!("басқа"),
}
```

`1 | 2` — `1` де, `2` де сай.

### Диапазонға сай (`..=`)

Сан немесе символ диапазондары:

```rust
let x = 5;

match x {
    1..=5 => println!("1 мен 5 арасында"),
    _ => println!("басқа"),
}

let c = 'c';

match c {
    'a'..='j' => println!("ерте әріп"),
    'k'..='z' => println!("кеш әріп"),
    _ => println!("басқа символ"),
}
```

`1..=5` — `1` бен `5` арасы (екі шеті де қосылған). Тек сан немесе
`char` түрлерімен қолданылады.

### Бөлшектеу

Бөлшектеу — мәннің ішкі бөліктерін атаулы айнымалыларға тарату.

#### Структты бөлшектеу

```rust
struct Point { x: i32, y: i32 }

fn main() {
    let p = Point { x: 0, y: 7 };

    let Point { x: a, y: b } = p;
    assert_eq!(0, a);
    assert_eq!(7, b);
}
```

Стандартты қысқа жазу — өрістің атын сақтау:

```rust
let Point { x, y } = p; // a, b емес — x, y
```

`match`-те жекелеген жайттарды бөлектеу:

```rust
match p {
    Point { x, y: 0 } => println!("x осінде, x = {}", x),
    Point { x: 0, y } => println!("y осінде, y = {}", y),
    Point { x, y } => println!("({}, {})", x, y),
}
```

#### Енамды бөлшектеу

```rust
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(i32, i32, i32),
}

let msg = Message::ChangeColor(0, 160, 255);

match msg {
    Message::Quit => println!("Quit"),
    Message::Move { x, y } => println!("Move to ({}, {})", x, y),
    Message::Write(text) => println!("Write: {}", text),
    Message::ChangeColor(r, g, b) => println!("Color({}, {}, {})", r, g, b),
}
```

Енамның әр нұсқасы өзіне сай бөлшектелу синтаксисімен жазылады.

#### Тоғыспалы құрылымдарды бөлшектеу

```rust
enum Color {
    Rgb(i32, i32, i32),
    Hsv(i32, i32, i32),
}

enum Message {
    ChangeColor(Color),
    // ...
}

let msg = Message::ChangeColor(Color::Hsv(0, 160, 255));

match msg {
    Message::ChangeColor(Color::Rgb(r, g, b)) => {
        println!("RGB: ({}, {}, {})", r, g, b);
    }
    Message::ChangeColor(Color::Hsv(h, s, v)) => {
        println!("HSV: ({}, {}, {})", h, s, v);
    }
    _ => (),
}
```

Тоғыспалы — енам ішінде енам, кортеж ішінде кортеж және басқалары.
Үлгі сондай тереңдікте жазылады.

#### Структ пен кортежді бірге

```rust
let ((feet, inches), Point { x, y }) = ((3, 10), Point { x: 3, y: -10 });
```

Бір сөйлемде кортеж + структ + ішкі кортеж бөлшектелді.

### Үлгіде мәнді елемеу

Кейде үлгінің бір бөлігі сізге қызық емес.

#### `_` (астыңғы сызық)

```rust
fn foo(_: i32, y: i32) {
    println!("y-ті ғана қолданамыз: {}", y);
}
```

`_` — кез келген мәнге сай, бірақ оны айнымалыға байламайды.

`match` ішінде:

```rust
let numbers = (2, 4, 8, 16, 32);
match numbers {
    (first, _, third, _, fifth) => {
        println!("Бір, үш, бес: {}, {}, {}", first, third, fifth);
    }
}
```

#### `_x` (атаулы, бірақ елемеу)

`_`-пен басталатын атау — компилятор «қолданбадың» деген ескертуді
шығармайды:

```rust
let _x = 5;
let y = 10; // y-ті қолданбасаңыз — ескерту
```

Бірақ `_x` — атау-үлгісі әрі де иелікті алады. `_` — иелікті алмайды.
Айырмашылық:

```rust
let s = Some(String::from("hello"));

if let Some(_s) = s {
    println!("Some бар");
}
// s енді жарамсыз — _s ішкі мәнді алды

let s = Some(String::from("hello"));

if let Some(_) = s {
    println!("Some бар");
}
// s әлі жарамды — _ иелікті алмады
```

#### `..` — бөліктің қалғанын елемеу

```rust
struct Point { x: i32, y: i32, z: i32 }

let origin = Point { x: 0, y: 0, z: 0 };

match origin {
    Point { x, .. } => println!("x = {}", x),
}
```

`..` — структтың қалған өрістерін елемейді. Кортеждерде де:

```rust
let numbers = (2, 4, 8, 16, 32);

match numbers {
    (first, .., last) => {
        println!("Алғашқы: {}, соңғы: {}", first, last);
    }
}
```

`..` — кортеждің ортасында, тек бір рет қана. Анық емес болса
компилятор жол бермейді.

### `match` шарттары (match guards)

Кейде үлгідегі шартты тереңірек тексеру керек:

```rust
let num = Some(4);

match num {
    Some(x) if x % 2 == 0 => println!("Жұп сан: {}", x),
    Some(x) => println!("Тақ сан: {}", x),
    None => println!("None"),
}
```

`if x % 2 == 0` — **match шарты**. Үлгі сай болғаннан кейін
қосымша шартты тексереді. Шарт орындалмаса — тармақ өткізіледі.

Match шарттары — үлгілерден тыс шарттарды қосу үшін; үлгілердің
өздерінде өрнектер жоқ.

Бірнеше үлгі мен match шартын біріктіруге болады:

```rust
let x = 4;
let y = false;

match x {
    4 | 5 | 6 if y => println!("y шын"),
    _ => println!("басқа"),
}
```

Бұл жерде `4 | 5 | 6` және `if y` — екеуі бірге. `if y` барлық үш
санға қатысты.

### `@` байланыстыру

`@` — мәнді ұстауымен қатар оны үлгіге де тексергім келеді:

```rust
enum Message {
    Hello { id: i32 },
}

let msg = Message::Hello { id: 5 };

match msg {
    Message::Hello { id: id_variable @ 3..=7 } => {
        println!("3-7 диапазонында: {}", id_variable);
    }
    Message::Hello { id: 10..=12 } => {
        println!("10-12 диапазонында");
    }
    Message::Hello { id } => {
        println!("Басқа id: {}", id);
    }
}
```

`id_variable @ 3..=7` — `id` 3-7 арасында ма дегенді тексереді **және**
оны `id_variable`-ге байлайды. Сонда бір үлгіде екеуі — диапазон
тексеру + айнымалыға байлау.

`@`-сіз `Message::Hello { id: 10..=12 }` жазсаңыз — `id`-ні
қолдану мүмкіндігіңіз жоқ. `@`-пен бар.

## 18.4. Тарау қорытындысы

Үлгілер `Rust`-тың айқын-күшті құралы. Олар арқылы:

- Мәннің құрылымына сай келтіріп ажыратамыз.
- Ішкі бөліктерді атаулы айнымалыларға бөлеміз.
- Сүзу логикасын типтік деңгейде жасаймыз.

Үлгі қолданылатын орындар: `match`, `if let`, `while let`, `for`,
`let`, функция параметрлері. Әрқайсысы өзіндік қажеттілік
бойынша қабылдамаушы (refutable) немесе жоққа шығаруға келмейтін
(irrefutable) үлгілерді талап етеді.

Бар синтаксис: литералдар, атаулар, `|` көп үлгі, `..=` диапазон,
бөлшектеу (структ, енам, кортеж, тоғыспалы), елемеу (`_`, `..`, `_x`),
match шарттары (`if ...`), `@` байланыстыру.

Үйренуі кейде күрделі болуы мүмкін, бірақ тәжірибеде үлгілер
кодты қысқа әрі мағыналы етеді — ондаған жолдағы шарттарды бір
ыңғайлы тармаққа жинайды.

Келесі тарауда — `Rust`-тың жетіленген тақырыптары туралы. Ол
`unsafe` блок, жетіленген трейттер, жетіленген типтер, жетіленген
функциялар, макростар сияқты — ілгері Rust бағдарламашысы үшін
маңызды құралдар.
