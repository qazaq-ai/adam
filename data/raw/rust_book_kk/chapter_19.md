# 19-тарау. Жетілдірілген мүмкіндіктер

Бұл тарауда `Rust`-тың әлдеқайда сирек кездесетін, бірақ кейбір
жобаларда маңызды мүмкіндіктерін қарастырамыз: `unsafe` блоктар,
жетілдірілген трейттер мен типтер, функция көрсеткіштері, кейін
макростар. Бұлар — күн сайынғы код үшін емес, белгілі жағдайларда
қажет болатын құралдар. Олардың әрқайсысы туралы кітап деңгейіндегі
кітап жазуға болады; біз — қысқаша шолу.

## 19.1. Қауіпсіз емес `Rust`

`Rust` сізге қауіпсіздіктің бірқатар кепілдіктерін береді — иелік,
қарыз ережелері, тип қауіпсіздігі, шексіздіктен қорғау. Бірақ бар
бағдарламалау мәселелерін шешуге сол кепілдіктер жеткіліксіз. Кейде
сізге `Rust`-тың тыйымдарынан тыс шығу керек:

- Аппараттық қамтамасыз етумен төменгі деңгейде сөйлесу.
- C тілінде жазылған кітапханамен мендесу.
- Стандартты кітапхананың ішкі (төменгі деңгейлі) бөліктерін жасау.

Соған **`unsafe`** блок арналған. `unsafe` ішінде сіз `Rust`-тың
кепілдіктеріне сенуге емес, өзіңіздің инвариантыңызды дәлелдейтін
кодты жаза аласыз.

`unsafe` бес «суперкүш» береді. Әдеттегі `Rust`-та мынау мүмкін емес:

1. **Шикі көрсеткішке дереф жасау** (`*const T`, `*mut T`).
2. **Қауіпсіз емес функция немесе әдіс шакыру** (`unsafe fn`).
3. **Өзгермелі статикалық айнымалыға қол жеткізу немесе оны
   өзгерту** (`static mut`).
4. **Қауіпсіз емес трейтті бекіту** (`unsafe impl`).
5. **Одақтың (`union`) өрістеріне қол жеткізу**.

`unsafe` сізге **қарыз тексергішті өшірмейді**. Сіз әлі де иелік
ережелеріне сай жазасыз. Тек жоғарыдағы бес мүмкіндік қосылады.

### Шикі көрсеткіш

Шикі көрсеткіштер — C-нің көрсеткіштеріне ұқсайды:

- Кез келген сан мәніне сілтей алады (тіпті дұрыс емес адреске).
- Бір уақытта көп шикі көрсеткіш бір жадқа сілтей алады, оның
  кейбірі өзгермелі болғанмен.
- Иесіз — тазалаудың автоматты жолы жоқ.
- `null` болуы мүмкін.

Шикі көрсеткішті жасау `unsafe`-те **қажет емес**. Бірақ оны
**дереф жасау** — `unsafe`-те ғана:

```rust
let mut num = 5;

let r1 = &num as *const i32;
let r2 = &mut num as *mut i32;

unsafe {
    println!("r1 = {}", *r1);
    println!("r2 = {}", *r2);
}
```

`unsafe` арқылы сіз компиляторға айтасыз: «Мен бұл шикі көрсеткіштің
жарамдылығына кепіл боламын». Қателесіп қалсаңыз — бағдарламаңыз бұзылады.

### Қауіпсіз емес функция

`unsafe fn` — функция өзінің барлық шакырушысына қандай да бір
шартты қою керек, әйтпесе оны шакыру қауіпсіз емес. Шакырғанда
`unsafe` блогы ішінде болуы керек:

```rust
unsafe fn dangerous() {
    // ...
}

unsafe {
    dangerous();
}
```

Стандартты кітапханада `unsafe fn` мысалдары бар: `String::from_utf8_unchecked`
(байттардың дұрыс UTF-8 екенін сіз кепілдеп жатсыз),
`Vec::set_len` (көрсеткен ұзындықтың дұрыстығын сіз кепілдеп жатсыз)
және басқалары.

### Қауіпсіз абстракцияны қауіпсіз емес кодтың үстінен жасау

`unsafe` блок ішінде функция бола береді — бірақ функцияның **сыртқы
интерфейсі** қауіпсіз болуы мүмкін. Стандартты кітапхана `Vec`,
`HashMap`, тіпті `String` — бәрі ішінде `unsafe` қолданады. Бірақ
сіз олармен жұмыс істегенде ешбір `unsafe` керек емес.

Мысал — `split_at_mut`. Стандартты функция: вектордың ортасынан
екі тілімге бөліп қайтарады. Әр тілім — өзгермелі. Қарыз ережелері
бойынша бір вектордан екі өзгермелі сілтеме жасау тыйым. Бірақ
**әр тілім бөлек байттарды қамтыса** — қауіпсіз. Қарыз тексергіш
бұны түсінбейді; ал бағдарламашы — біледі.

```rust
use std::slice;

fn split_at_mut(values: &mut [i32], mid: usize) -> (&mut [i32], &mut [i32]) {
    let len = values.len();
    let ptr = values.as_mut_ptr();

    assert!(mid <= len);

    unsafe {
        (
            slice::from_raw_parts_mut(ptr, mid),
            slice::from_raw_parts_mut(ptr.add(mid), len - mid),
        )
    }
}
```

`unsafe` блогы ішінде шикі көрсеткіштермен жұмыс. Бірақ функцияның
сыртқы интерфейсі (`fn split_at_mut`) — қауіпсіз. Шакырғыш ешбір
`unsafe`-ке кірмейді.

`assert!(mid <= len)` — функцияның инвариантын тексеру. Бұл функция
сертификациясы.

### `extern` функциялар: FFI

**Foreign Function Interface (FFI)** — басқа тілде жазылған кодпен
сөйлесу. C кітапханалары — мысал. `Rust`-та `extern` блок арқылы:

```rust
extern "C" {
    fn abs(input: i32) -> i32;
}

fn main() {
    unsafe {
        println!("|−3| = {}", abs(-3));
    }
}
```

`"C"` — Application Binary Interface (ABI). C-нің стандартты ABI-ы.
`extern` блок ішінде сіз қандай функциялар сыртта бар екенін мәлімдейсіз.
Шакыру — `unsafe`, өйткені `Rust` C функциясының дұрыс жазылғанын
кепілдей алмайды.

Қарама-қарсы бағыт — басқа тілдер `Rust` функцияларын шакыру керек
болса:

```rust
#[no_mangle]
pub extern "C" fn call_from_c() {
    println!("C-ден шакырылды");
}
```

`#[no_mangle]` атрибуты — компиляторға «бұл функцияның атауын
өзгертпе» деп айтады. Сонда C коды оны атаумен таба алады.

### Өзгермелі статикалық айнымалылар

Әдеттегі `static` — глобальды тұрақты:

```rust
static HELLO: &str = "Сәлем, дүние!";

fn main() {
    println!("{}", HELLO);
}
```

`static` `const`-тан өзгеше: оның жадтағы орналасуы тұрақты, ал `const`
қолданғанша ішке тиеленеді.

`static mut` — ағындар арасында ортақ. Сондықтан оған қол жеткізу
**қауіпсіз емес**:

```rust
static mut COUNTER: u32 = 0;

fn add_to_count(inc: u32) {
    unsafe {
        COUNTER += inc;
    }
}

fn main() {
    add_to_count(3);
    unsafe {
        println!("COUNTER: {}", COUNTER);
    }
}
```

Бірнеше ағыннан `add_to_count` шакырылса — жарыс шарты болуы мүмкін.
Сондықтан `static mut` өте сирек қолданылады. Көп жағдайда оған
балама — `Mutex<T>` немесе атомдық типтер.

### Қауіпсіз емес трейттер

Кейде трейтті бекіткеннің инварианттарын `Rust` өзі тексере алмайды.
Сондықтан тип сол трейтке сай ма деп бағдарламашы кепіл бола алады:

```rust
unsafe trait Foo {
    // әдістер
}

unsafe impl Foo for i32 {
    // ...
}
```

`Send` және `Sync` — стандартты `unsafe` трейттердің мысалы. Қарапайым
типтерде олар автоматты түрде бекітіледі; сирек жағдайда сіз `unsafe
impl Send for ...` жазып, инвариантты қолмен дәлелдейсіз.

### Одақ

`union` — C-нің одағына ұқсас тип. Барлық өрістер бір жадты
бөлесіп ұстайды, бірақ бір сәтте бір ғана өрісті оқысаңыз дұрыс.
Дұрыс емес өрісті оқу — қауіпсіз емес.

`union` — С-кодпен өзара әрекеттесуде сирек қолданылады; әдетте
`enum` (тегtagged union) дұрыс таңдау.

## 19.2. Жетілдірілген трейттер

### Байланысты типтер

**Байланысты тип** — трейттің жариялауында қолданылатын типтік
орынтолтырғыш. Жалпылама тип параметріне ұқсас, бірақ айырмашылық бар.

`Iterator` трейтін еске сақтаймыз:

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
}
```

`type Item` — байланысты тип. Әр бекіту үшін **бір ғана** тип
болады:

```rust
impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<u32> {
        // ...
    }
}
```

Жалпылама түрмен айырмашылық: `<T>` түрімен бір типке бір нұсқадан
артық бекіту мүмкін болар еді (`impl Iterator<i32> for Counter`,
`impl Iterator<f64> for Counter` — екеуі де). Байланысты типтермен —
тек **бір** бекіту, сондықтан `next()` қандай типке қайтаратынын
шакырушы кодта анық.

### Әдепкі жалпылама тип параметрлері

`<T = SomeType>` — жалпылама параметрге әдепкі тип беруге болады:

```rust
trait Add<Rhs = Self> {
    type Output;
    fn add(self, rhs: Rhs) -> Self::Output;
}
```

`Rhs = Self` — әдепкі бойынша оң жақ операнд `Self` түрінде. Сондықтан
`a + b`-та екеуі бір типте болады. Бірақ басқа типпен қосу керек
болса:

```rust
struct Millimeters(f64);
struct Meters(f64);

impl Add<Meters> for Millimeters {
    type Output = Millimeters;

    fn add(self, other: Meters) -> Millimeters {
        Millimeters(self.0 + (other.0 * 1000.0))
    }
}
```

Әдепкі типті ауыстырып, `Millimeters + Meters` амалын анықтадық.

### Толық қолтаңбамен айқындау

Бір типке бірнеше трейт ұқсас атпен әдіс берсе — қайсы шакырылады?
**Толық қолтаңбамен айқындау** көмекке келеді:

```rust
trait Pilot {
    fn fly(&self);
}

trait Wizard {
    fn fly(&self);
}

struct Human;

impl Pilot for Human {
    fn fly(&self) {
        println!("Бұл сіздің капитан-ұшқыш");
    }
}

impl Wizard for Human {
    fn fly(&self) {
        println!("Тағы көтеріл!");
    }
}

impl Human {
    fn fly(&self) {
        println!("*Қолымды үстіне көтердім*");
    }
}

fn main() {
    let person = Human;
    person.fly();              // Human::fly — әдеттегідей
    Pilot::fly(&person);       // Pilot::fly
    Wizard::fly(&person);      // Wizard::fly
}
```

Әдістер `&self`-пен — байланысты функциялар (`Self` параметрсіз)
үшін толық қолтаңбамен жазу:

```rust
<Human as Pilot>::fly(&person);
```

`<Type as Trait>::method` — толық, әрқашан жұмыс істейтін форма.

### Ата-трейттер

Кейде сіздің трейтіңізге басқа трейттің бекітуі қажет:

```rust
use std::fmt;

trait OutlinePrint: fmt::Display {
    fn outline_print(&self) {
        let output = self.to_string();
        let len = output.len();
        println!("{}", "*".repeat(len + 4));
        println!("*{}*", " ".repeat(len + 2));
        println!("* {} *", output);
        println!("*{}*", " ".repeat(len + 2));
        println!("{}", "*".repeat(len + 4));
    }
}
```

`OutlinePrint: fmt::Display` — `OutlinePrint` тек `Display`
бекіткен типтерге бекітіледі. **Ата-трейт** — `Display`. Сондықтан
`outline_print` ішінде `self.to_string()` жұмыс істейді.

### Орфан-ережені айналдыру: жаңатип үлгісі

7-тарауда орфан-ережесі туралы айттық: трейтті типке тек өзіңіздің
сандығыңыздан болса ғана бекіте аласыз. Бірақ сізге `Display`
трейтін `Vec<T>` түріне бекіткіңіз келсе ше? Екеуі де стандартты
кітапханада. Тыйым.

Шешім — **жаңатип** (newtype) үлгісі: `Vec<T>`-ті өзіңіздің тегіңізбен
орап аласыз:

```rust
use std::fmt;

struct Wrapper(Vec<String>);

impl fmt::Display for Wrapper {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "[{}]", self.0.join(", "))
    }
}

fn main() {
    let w = Wrapper(vec![String::from("сәлем"), String::from("дүние")]);
    println!("w = {}", w);
}
```

`Wrapper` — біздің сандықта; `Display` — стандартты кітапханада;
бекіту — заңды.

Кемшілігі — `Wrapper`-да `Vec`-тің әдістері тікелей қол жетімді
емес. Қажетін `impl` арқылы өткізе аласыз, бірақ ыңғайсыз. Жалпы —
жаңатип шектеулі мақсат үшін.

## 19.3. Жетілдірілген типтер

### Жаңатип үлгісі — типтік қауіпсіздік үшін

Жаңатип үлгісінің тағы пайда: бір негізгі типтен бірнеше әртүрлі
семантикалық типті жасау. Мысалы:

```rust
struct Years(i64);
struct Days(i64);

impl Years {
    pub fn to_days(&self) -> Days {
        Days(self.0 * 365)
    }
}
```

Енді `Years` пен `Days` екеуі — әртүрлі типтер. Тіпті ішінде екеуі
де `i64` болса да, біреуінің орнына екіншісін беру мүмкін емес
— компилятор қателікке барады. Бұл — типтік жүйе арқылы мағыналы
қателерді тыюшы.

### Тип бүркеншегі

`type` кілтсөзі — бар типке басқа атау беруге:

```rust
type Kilometers = i32;

let x: i32 = 5;
let y: Kilometers = 5;

println!("x + y = {}", x + y);
```

`Kilometers` — `i32`-нің бүркеншегі. Жаңатип емес — біреуін екіншісімен
араластыруға болады. Тек код оқуға айқындау үшін.

Көп жағдайда қолданылатын ұзын типтерді қысқартуға арналған:

```rust
type Thunk = Box<dyn Fn() + Send + 'static>;

fn takes_long_type(f: Thunk) { /* ... */ }
```

`Thunk` — оқуға бір қарап түсінікті; ал `Box<dyn Fn() + Send + 'static>`
әр функция сигнатурасында ұзын болады.

### Ешқашан-тип `!`

Кейбір өрнектер ешқашан мән қайтармайды — олар `panic!`-қа барады,
қайтарусыз орындалады, не циклге қайтарылмай қалады. Олардың типі —
**ешқашан-тип `!`** (the never type, «empty type»).

Мысалы:

```rust
fn bar() -> ! {
    panic!("Ешқашан қайтармаймын");
}
```

`panic!` `!` қайтарады. `!` кез келген типке аударылады, сондықтан
оны кез келген контексте қолдануға болады.

`continue` — `!` қайтарады:

```rust
let guess: u32 = match guess.trim().parse() {
    Ok(num) => num,
    Err(_) => continue,
};
```

`Ok(num) => num` — `u32` қайтарады. `Err(_) => continue` — `!`. Екі
тармақтың типі бір болуы керек, бірақ `!` кез келгенге сай — сондықтан
жалпы тип `u32` болады.

### Динамикалық өлшемді типтер мен `Sized`

`Rust`-та көп тип компиляция кезінде белгілі мөлшерде болады. Бірақ
кейбір типтер — өлшемі тек орындалу кезінде белгілі болады.

`str` — мысал. Бір жолдың мөлшері әр жолға бөлек. Сондықтан тікелей
`str` түрінде айнымалы жасау мүмкін емес:

```rust
let s: str = "сәлем"; // ҚАТЕ: size of str cannot be known
```

Бірақ `&str` — жұмыс істейді, өйткені `&str` — екі мән (көрсеткіш
+ ұзындық), мөлшері белгілі.

Жалпылама функцияда жалпылама тип параметрі әдетте `Sized` (мөлшері
белгілі) болуы шартталған. Жасырын сипат:

```rust
fn generic<T>(t: T) { /* ... */ }
```

— шынында:

```rust
fn generic<T: Sized>(t: T) { /* ... */ }
```

`T: ?Sized` — динамикалық мөлшерлі типтерді де қабылдайтын ерекше
синтаксис:

```rust
fn generic<T: ?Sized>(t: &T) { /* ... */ }
```

`?Sized` тек жалпылама параметрлерге қолданылады. `t` — `&T` ретінде
келуі керек, өйткені `T` мөлшері белгісіз болуы мүмкін.

## 19.4. Жетілдірілген функциялар мен жабулар

### Функция көрсеткіштері

Әдеттегі жабудан басқа — **функция көрсеткіштері**. Тек `Fn` емес,
**`fn`** (шағын `f`):

```rust
fn add_one(x: i32) -> i32 { x + 1 }

fn do_twice(f: fn(i32) -> i32, arg: i32) -> i32 {
    f(arg) + f(arg)
}

fn main() {
    let answer = do_twice(add_one, 5);
    println!("{}", answer); // 12
}
```

`fn(i32) -> i32` — `Fn`/`FnMut`/`FnOnce`-ке ұқсас, бірақ нақты
функция көрсеткіші. Барлық функция көрсеткіштері `Fn` де. Сондықтан
жалпылама `<F: Fn(...)>` орнына нақты `fn(...)` қолдану — кейбір
жағдайларда дұрыс (мысалы, FFI-да C функция көрсеткіштерін қабылдау).

### Жабуды қайтару

Функцияға жалпылама параметр беру оңай. Бірақ функциядан жабу
қайтару — қиынырақ. Себебі әр жабудың типі — анонимді, сізде
нақты атау жоқ.

Шешім — `Box<dyn Fn(...) -> ...>`:

```rust
fn returns_closure() -> Box<dyn Fn(i32) -> i32> {
    Box::new(|x| x + 1)
}
```

Box арқылы жабудың дәл түрін жасырамыз; шакырушы `Fn(i32) -> i32`
интерфейсіне сүйенеді.

Кейбір жағдайда `impl Fn(...) -> ...` де жұмыс істейді (Rust 1.26+):

```rust
fn returns_closure() -> impl Fn(i32) -> i32 {
    |x| x + 1
}
```

— қарапайым; бір нақты жабуға арналған. Бірақ `match` ішінде әр түрлі
жабу қайтаратын болсаңыз — `Box<dyn Fn>` керек, өйткені `impl Trait`
тек бір нақты типке арналған.

## 19.5. Макростар

Макрос — кодты *кодтан* шығаратын код. Функциядан өзгеше: функция
мәндерді алады, нәтиже қайтарады. Макрос — синтаксистік ағашты алады,
жаңа синтаксистік ағаш қайтарады.

`Rust`-та екі үлкен макрос түрі бар:

1. **Декларативті макростар** (`macro_rules!`).
2. **Процедуралық макростар** — үш кіші түрі: derive, attribute,
   function-like.

### Декларативті макростар

`vec!` мысалы:

```rust
let v: Vec<u32> = vec![1, 2, 3];
```

`vec!` — макрос. Оның жариялауы шамамен былай:

```rust
#[macro_export]
macro_rules! vec {
    ( $( $x:expr ),* ) => {
        {
            let mut temp_vec = Vec::new();
            $(
                temp_vec.push($x);
            )*
            temp_vec
        }
    };
}
```

`$( $x:expr ),*` — синтаксис: бөлектелген өрнектердің жиыны. `$x`
— айнымалы; `expr` — оның түрі; `,` — бөлектегіш; `*` — қайталану.

Шакырылғанда — `vec![1, 2, 3]` деп — макрос ішке `1, 2, 3`-ті алады
және әр өрнек үшін `temp_vec.push($x)` қайталауын жасайды. Нәтиже —
жаңа кіші вектор.

`#[macro_export]` — макросты сандықтан сыртқа экспорттайды.

Декларативті макростар — қарапайым шаблон-сай келтіру. Көп жағдайда
жеткілікті, бірақ шектеулері бар.

### Процедуралық макростар

Процедуралық макрос — синтаксистік ағашты қабылдайды, өзгертеді,
қайтарады. Нақты функция, бірақ компиляция кезінде орындалады.

#### derive макросы

`#[derive(Debug)]` — derive макросының мысалы. Біз де өзіміздің
жасай аламыз. Олар бір сандықта — `proc-macro` сандығында —
жүзеге асырылады, басқа сандықтан қолданылады.

Қарапайым `HelloMacro` мысалы:

```rust
// hello_macro/src/lib.rs (трейтті жариялайтын сандық)
pub trait HelloMacro {
    fn hello_macro();
}
```

```rust
// hello_macro_derive/src/lib.rs (proc-macro сандығы)
extern crate proc_macro;

use proc_macro::TokenStream;
use quote::quote;
use syn;

#[proc_macro_derive(HelloMacro)]
pub fn hello_macro_derive(input: TokenStream) -> TokenStream {
    let ast = syn::parse(input).unwrap();
    impl_hello_macro(&ast)
}

fn impl_hello_macro(ast: &syn::DeriveInput) -> TokenStream {
    let name = &ast.ident;
    let gen = quote! {
        impl HelloMacro for #name {
            fn hello_macro() {
                println!("Hello, Macro! Менің атым {}!", stringify!(#name));
            }
        }
    };
    gen.into()
}
```

Шакырылғанда:

```rust
use hello_macro::HelloMacro;
use hello_macro_derive::HelloMacro;

#[derive(HelloMacro)]
struct Pancakes;

fn main() {
    Pancakes::hello_macro();
}
```

Шығатыны: `Hello, Macro! Менің атым Pancakes!`

`syn` — синтаксис талдау; `quote` — синтаксис генерация. Олар —
екі стандартты сандық.

#### Атрибут макростары

`#[route(GET, "/")]` сияқты атрибуттарды жасауға арналған. Web-кең
жобаларда (Actix, Rocket) маршрут жариялау сондай.

#### Функция-сияқты макростар

Декларативті `macro_rules!`-ке ұқсайды, бірақ кеңірек. Айырмашылық —
шакыру синтаксисі: `my_macro!(...)`. Мысалы, SQL-сұраныстарды
компиляция кезінде тексеретін `sqlx::query!` — функция-сияқты
процедуралық макрос.

### Функция vs макрос

Қашан макрос, қашан функция:

- **Айнымалы санды аргумент.** `println!`, `vec!` — қандай саны
  болса да аргумент қабылдайды. Функция — фиксацияланған сан.
- **Кодтан кодты шығару.** `derive` арқылы трейттің бекітуін
  автоматты. Функция мұндайды жасай алмайды.
- **Әр трейт бекітуінің атауын қолдан жазудың керегі болмауы.**
  Бір рет макрос жазсаңыз — барлық типке бекітіледі.

Бірақ макростар:

- Күрделі, түсіну мен debug қиынырақ.
- Қателер хабарламасы жетімсіз болуы мүмкін.

Бірінші — функция; қажет болса — макрос.

## 19.6. Тарау қорытындысы

Бұл — `Rust`-тың жетілдірілген тұсы:

- **`unsafe`** — қарапайым ережелерден тыс шығу. Шикі көрсеткіш,
  қауіпсіз емес функция, FFI, `static mut`, `unsafe trait`,
  `union`. Сирек қолданылады, көбіне стандартты кітапхана ішінде.
- **Жетілдірілген трейттер** — байланысты типтер, әдепкі жалпылама
  параметрлер, толық қолтаңба, ата-трейттер, жаңатип үлгісі.
- **Жетілдірілген типтер** — жаңатип, тип бүркеншегі, ешқашан-тип
  `!`, динамикалық өлшемді типтер мен `?Sized`.
- **Функция көрсеткіштері мен жабу қайтару** — `fn` түрі,
  `Box<dyn Fn>`, `impl Fn`.
- **Макростар** — `macro_rules!` (декларативті), процедуралық
  (derive, attribute, function-like).

Бұл құралдар — әр жобада қажет емес. Бірақ бар Rust бағдарламашысы
өзінің бір күні `unsafe`-ке кірмесін, тип бүркеншегін қолданбасын,
макрос жазып шықпасын деп біле алмайды.

Соңғы — 20-тарау — бар бұрын үйренгенімізді бір жобаға жинау:
көп ағынды веб-сервер.
