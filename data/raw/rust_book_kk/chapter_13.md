# 13-тарау. Функционал тілдік мүмкіндіктер: итераторлар мен жабулар

`Rust`-тың дизайны функционал бағдарламалаудан көп нәрсе алды. Lisp,
Haskell, ML сияқты тілдерден шыққан **жабу** (closure) мен **итератор**
ұғымдары `Rust`-тың ажырамас бөлігі болды. Олар арқылы кодты әлдеқайда
жинақы әрі мағыналы жазуға болады — арнайы өнімділік шығынынсыз.

Бұл тарауда:

1. Жабу не екенін, оның қалай қоршаған ортадан мәндерді ұстайтынын,
   `Fn` / `FnMut` / `FnOnce` трейттерінің айырмашылығын.
2. Итератор не екенін, лазай есептеу стилін, итератор-бейімдеуіштер
   тізбегін.
3. Алдыңғы тарауда жасаған `minigrep` бағдарламасын осы құралдармен
   қалай тазалауға болатынын.
4. Цикл vs итератор: өнімділік қалай.

## 13.1. Жабулар: қоршаған ортасын ұстайтын атаусыз функциялар

**Жабу** — атаусыз функция; ол функцияға жалақыша, бірақ айырмашылығы
сонда — оны бір айнымалыға бекітіп, кейін шакыруға, басқа функцияға
аргумент ретінде беруге болады. Қосылған пайда: жабу қоршаған ортадан —
ол жасалған кодтың аумағынан — мәндерді ұстай алады.

### Жабу мысалы

Қарапайым жабу:

```rust
let plus_one = |x| x + 1;
println!("{}", plus_one(5)); // 6
```

`|x| x + 1` — жабудың денесі. Тік жақшалардың ішінде параметрлер;
артынан — өрнек. Кешенді жабудың денесі фигуралы жақшалардың ішіне
қойылады:

```rust
let plus_one = |x: i32| -> i32 {
    println!("санға бір қостым");
    x + 1
};
```

`Rust` жабудың параметр түрі мен қайтару түрін көп жағдайда өзі шеше
алады. Әдеттегідей біз тек қажет кезде нақты белгілейміз.

### Қоршаған ортасын ұстау

Жабу функциядан айырмасы — ол өзі жасалған аумақтан айнымалыларды
ұстай алады:

```rust
fn main() {
    let list = vec![1, 2, 3];
    println!("Бар тізім: {:?}", list);

    let only_borrows = || println!("Жабу ішінен: {:?}", list);

    only_borrows();
    only_borrows();
}
```

`only_borrows` — `list`-ке тұрақты сілтеме арқылы сілтейтін жабу.
`list` әлі бар тұра береді — жабу оны еленуге ғана пайдаланды.

Өзгермелі сілтеме арқылы ұстау:

```rust
fn main() {
    let mut list = vec![1, 2, 3];
    println!("Бар тізім: {:?}", list);

    let mut borrows_mutably = || list.push(7);

    borrows_mutably();
    println!("Шакырылғаннан соң: {:?}", list);
}
```

`borrows_mutably` `list`-ке өзгермелі сілтеме ұстайды (өйткені `push`
оны өзгертеді). Жабу шакырылғанға дейін `list`-ке басқа жерден қол
жеткізе алмаймыз.

### `move` арқылы иелікті ұстау

Кейбір жағдайларда сізге **иелікті** жабудың ішіне ауыстыру керек.
Әсіресе жабуды басқа ағынға беру керек болса. Соған `move` кілтсөзі:

```rust
use std::thread;

fn main() {
    let list = vec![1, 2, 3];
    println!("Бар тізім: {:?}", list);

    thread::spawn(move || println!("Ағыннан: {:?}", list))
        .join()
        .unwrap();
}
```

`move ||` — жабу `list`-ті өз иелігіне алады. `thread::spawn` жаңа
ағынды бастайды; егер `list` сілтеме арқылы алынса — ағындар арасында
тіршілік мерзімі шатастырады. `move` арқылы иелік жабуға өтеді, ағында
қауіпсіз қалады.

### `Fn`, `FnMut`, `FnOnce` трейттері

Жабудың қоршаған орта мәндерімен жұмыс жасау тәсілі — оның қандай
трейтті бекітуіне әсер етеді. Стандартты кітапханада үш трейт бар:

- **`FnOnce`** — жабу бір рет шакырылса жетеді. Иелікті алып,
  «жұтып» жіберетін жабулар осы трейтті ғана бекітеді.
- **`FnMut`** — жабу бірнеше рет шакырылса болады, бірақ ол ұсталған
  мәндерді өзгерте алады.
- **`Fn`** — жабу бірнеше рет шакырылса болады, әрі ол ешбір мәнді
  өзгертпейді (тек оқу үшін).

Барлық жабу `FnOnce` бекітеді. Олардың кейбірі `FnMut`-ты да; кейбірі
үшеуін де. Қарапайым ереже: жабу қаншалықты «тыныш» — соншалықты көп
трейт бекітіледі.

Мысал — стандартты кітапхананың `Option::unwrap_or_else`:

```rust
impl<T> Option<T> {
    pub fn unwrap_or_else<F>(self, f: F) -> T
    where
        F: FnOnce() -> T
    {
        match self {
            Some(x) => x,
            None => f(),
        }
    }
}
```

`F: FnOnce() -> T` — `f` бір рет ғана шакырылуы мүмкін. Бұл — қатаң
шектеу: `unwrap_or_else` бір ғана рет жабуды шакырады, сондықтан
жабу кеңірек трейтті бекіту керек емес.

Әдетте параметрді `Fn`-ден бастап шектеудің керегі жоқ — `FnOnce`
немесе `FnMut` керектігін анықтап, сол ғана жазсаңыз — кодты қажетсіз
тарылтпайсыз.

### Жабудың қызметтік мысалы

`Vec`-ке арналған `sort_by_key` әдісі — жабу қабылдайтын мысал:

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let mut list = [
        Rectangle { width: 10, height: 1 },
        Rectangle { width: 3, height: 5 },
        Rectangle { width: 7, height: 12 },
    ];

    list.sort_by_key(|r| r.width);
    println!("{:#?}", list);
}
```

`|r| r.width` — әр элемент үшін кілт қайтаратын жабу. `sort_by_key`
жабуды әр элементте шакырып, кілт бойынша сұрыптайды.

## 13.2. Итераторлар арқылы элементтер тізбегін өңдеу

**Итератор** — элементтердің тізбегінде бір-бірлеп жүруге арналған
типтік интерфейс. `Rust`-та итератор — `Iterator` трейтін бекітетін
кез келген тип. `Iterator` трейтінің негізі — `next` әдісі:

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;

    // ... көптеген әдепкі әдістер
}
```

`type Item` — итератор қайтаратын элементтердің типі. `next` — келесі
элементті қайтарады; элементтер бітсе — `None`.

### Итератор жасау

Кез келген ұжымнан итератор жасауға болады:

- `iter()` — тұрақты сілтемелер беретін итератор.
- `iter_mut()` — өзгермелі сілтемелер беретін итератор.
- `into_iter()` — иелікті алатын итератор.

```rust
let v1 = vec![1, 2, 3];
let v1_iter = v1.iter();

for val in v1_iter {
    println!("{val}");
}
```

`for` циклы итераторды аралайды. Әр итерация — `next` шакыруы. Цикл
аяқталу — `next` `None` қайтарғанда.

### Итераторлар лазай

`Rust`-тың итераторлары — **лазай**. Сіз итераторды жасаған кезде ешбір
есептеу болмайды. Тек `next` шакырылғанда нақты жұмыс жасалады:

```rust
let v1 = vec![1, 2, 3];
let v1_iter = v1.iter();
// әзір ешбір элемент оқылмады

for val in v1_iter {
    println!("{val}");
    // енді әр итерацияда next шакырылады
}
```

Бұл — лазай есептеудің күші: сіз бірнеше итератор-операцияны тізбекке
қосып шакыра аласыз; бірақ нақты жұмыс ең соңында ғана басталады.

### Тұтынатын бейімдеулер

Кейбір әдістер итераторды **тұтынады** — оны соңына дейін аралап шығып,
бір нәтиже қайтарады. Мысалы, `sum`:

```rust
let v1 = vec![1, 2, 3];
let v1_iter = v1.iter();
let total: i32 = v1_iter.sum();
println!("{total}"); // 6
```

`sum` — итератордың барлық элементін қосып, нәтижені қайтарады.
Шакырылғаннан соң итератор жоғалады — оны қайта пайдалану мүмкін емес.
Сондықтан тұтынатын деп аталады.

Басқа тұтынатын бейімдеулер: `count`, `last`, `min`, `max`, `collect`,
`for_each`.

### Жаңа итератор шығаратын бейімдеулер

Кейбір әдістер итерторды бір итератордан екіншіге **айналдырады**.
Олар лазай — нәтиже әлі есептелмейді, бірақ кейін шакырылады.

`map` — әр элементті жабудан өткізеді:

```rust
let v1: Vec<i32> = vec![1, 2, 3];
let v2: Vec<_> = v1.iter().map(|x| x + 1).collect();
println!("{:?}", v2); // [2, 3, 4]
```

`map(|x| x + 1)` — жаңа итератор қайтарады. Бірақ есептеу әлі жасалмаған.
`.collect()` — соңында итераторды тұтынып, нәтижені векторға жинайды.

`filter` — шартқа сай элементтерді ғана өткізеді:

```rust
let v: Vec<i32> = (1..=10).filter(|x| x % 2 == 0).collect();
println!("{:?}", v); // [2, 4, 6, 8, 10]
```

Бұл екі бейімдеу — `map` пен `filter` — `Rust`-та (және басқа функционал
тілдерде) ең жиі қолданылады.

Тізбек құруға болады:

```rust
let result: i32 = (1..=10)
    .filter(|x| x % 2 == 0)
    .map(|x| x * x)
    .sum();

println!("{}", result); // 4 + 16 + 36 + 64 + 100 = 220
```

Бір тізбекте: 1-ден 10-ға дейінгі сандар → жұптарын сүзу → әрбірін
квадраттау → қосу. Бірнеше итератор-бейімдеу бір өрнекте; нәтиже —
бір сан. Цикл арқылы жазсаңыз 5-6 жол кетер еді.

### Жабу арқылы қоршаған ортаны ұстау

Итератор-бейімдеулерге берілген жабулар сол жабудың туылған аумағынан
мәндерді ұстай алады. Мысалы:

```rust
#[derive(Debug, PartialEq)]
struct Shoe {
    size: u32,
    style: String,
}

fn shoes_in_size(shoes: Vec<Shoe>, shoe_size: u32) -> Vec<Shoe> {
    shoes.into_iter().filter(|s| s.size == shoe_size).collect()
}
```

`|s| s.size == shoe_size` жабуы `shoe_size` параметрін ұстайды. Әр
аяқкиім үшін оның өлшемі сұралған өлшемге тең бе тексеріледі. Сондай
жеңіл, оқуға анық код.

## 13.3. Кіріс-шығыс жобамызды жетілдіру

Алдыңғы тарауда `minigrep` бағдарламасын жасадық. Енді оны итераторлар
мен жабулар арқылы тазалап-жинақтайық.

### `Config::build`-тен `clone` алып тастау

Бұрынғы код:

```rust
impl Config {
    pub fn build(args: &[String]) -> Result<Config, &'static str> {
        if args.len() < 3 {
            return Err("аргументтер жетіспейді");
        }
        let query = args[1].clone();
        let file_path = args[2].clone();
        Ok(Config { query, file_path, ignore_case: env::var("IGNORE_CASE").is_ok() })
    }
}
```

`clone` арқылы біз `String`-терді көшіріп жатырмыз. `args` — `&[String]`;
оған иелік жоқ. Бірақ `Config` иелікке мұқтаж — өйткені оны `main`-нен
сыртқа тарату керек. Сондықтан `clone`.

Енді `args` орнына итераторды берсек ше? Иеліктегі мәндерді шығарып
ала аламыз:

```rust
impl Config {
    pub fn build(
        mut args: impl Iterator<Item = String>,
    ) -> Result<Config, &'static str> {
        args.next(); // бірінші — бағдарлама атауы

        let query = match args.next() {
            Some(arg) => arg,
            None => return Err("ізделетін сөз жоқ"),
        };

        let file_path = match args.next() {
            Some(arg) => arg,
            None => return Err("файл жолы жоқ"),
        };

        let ignore_case = env::var("IGNORE_CASE").is_ok();

        Ok(Config { query, file_path, ignore_case })
    }
}
```

`impl Iterator<Item = String>` — кез келген `String` беретін итератор.
`next` шакыруы итерациядан `String` — иелігімен — алады. Сондықтан
`clone` керек емес.

`main` ішінде:

```rust
fn main() {
    let config = Config::build(env::args()).unwrap_or_else(|err| {
        eprintln!("Аргумент қатесі: {err}");
        process::exit(1);
    });
    // ...
}
```

Біз `env::args()`-ті тура `Config::build`-ке береміз. `env::args` —
итератор қайтарады; ол `String`-ті — иелігімен — береді.

### `search` функциясын итератор-бейімдеулермен қысқарту

Бұрынғы:

```rust
pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str> {
    let mut results = Vec::new();
    for line in contents.lines() {
        if line.contains(query) {
            results.push(line);
        }
    }
    results
}
```

Итератор стилімен:

```rust
pub fn search<'a>(query: &str, contents: &'a str) -> Vec<&'a str> {
    contents
        .lines()
        .filter(|line| line.contains(query))
        .collect()
}
```

Бір өрнекте: жолдарды аралап → `query` бар жолдарды сүзіп → жинау. 6
жолдан 1 жолға қысқарды. Әрі — мағыналырақ.

`search_case_insensitive` де ұқсас:

```rust
pub fn search_case_insensitive<'a>(
    query: &str,
    contents: &'a str,
) -> Vec<&'a str> {
    let query = query.to_lowercase();
    contents
        .lines()
        .filter(|line| line.to_lowercase().contains(&query))
        .collect()
}
```

### Цикл vs итератор: қайсы дұрыс?

Көп `Rust` бағдарламашылары — егер сізге қандай да бір ыңғайлы стиль
болса — оны қолдана беруді ұсынады. Бірақ практика бойынша:

- **Итератор** — деректерді өңдеу кездесінде ыңғайлы. Сүзу, түрлендіру,
  жинақтау — итератор стилі айқынырақ.
- **Цикл** — қарапайым жағдайда (бір айнымалыны өсіру, есеп) ыңғайлы.

Уақыт өте келе бір көзқарас — функционал стиль *неғұрлым* идиомаға
сай болады. `Rust` қауымдастығы оны жиі таңдайды.

Және ең маңыздысы — өнімділік сондай.

## 13.4. Өнімділік: цикл vs итератор

Біріншіден, теориялық уағдалас: «жалпы абстракция баяу болады» дегенмен
келісеміз. Бірақ `Rust` бұған тыйым салады. `Rust`-та абстракциялар —
**нөлдік шығынды** (zero-cost) деп аталады. Бұл: «егер сіз кодты
қолмен оптимизацияланған цикл түрінде жазсаңыз, ол итератор-стильдегі
кодпен бір жылдамдықта жұмыс істейді» дегенді білдіреді.

Бұның дәлелі — `rustc` компиляторында. Итератор-тізбегі мономорфтаудан
кейін шындыққа айналады: `map` мен `filter` — арнайы функциялар емес;
олар компиляцияда жоғарыдағы циклмен бір қалыпта орналастырылады.
Кейінгі оптимизация (мысалы, цикл-ішінде функция шакыруларын ішке
сіңіру) — олар жасалғандай, тура цикл түрінде орындалады.

Іс жүзінде, кейбір итератор-кодтар қолмен жазылған циклден де **тиімдірек**
жұмыс істейді — себебі компиляторға қатаң спецификация беріледі: «осы
қадамдар тізбегі — әр элементте лек-лек жүреді». Ондай тура спецификация
жоғары деңгейде сақталғанда — компилятор оны жоғарырақ оптимизациялауы
жеңілірек.

Сондықтан, дұрыс шешім: **жабулар мен итераторларды еркін қолданыңыз.**
Олар сізге кодты қысқа жазуға, идеяны айқын жеткізуге, бағдарламаны
тазалауға мүмкіндік береді — әрі өнімділік үшін «төлеудің» керегі жоқ.

## 13.5. Тарау қорытындысы

Жабу — қоршаған ортаны ұстайтын атаусыз функция. Жабудың үш мінез-түрі —
`FnOnce`, `FnMut`, `Fn` — олардың нені қалай ұстайтынына сай. `move`
арқылы жабуға иелікті ауыстыруға болады.

Итератор — элементтер тізбегімен жұмыс істеудің типтік интерфейсі.
`Iterator` трейті, `next` әдісі, лазай есептеу — олардың негізі.
`map`, `filter`, `collect`, `sum` — стандарт құралдар.

`Rust`-та функционал стиль — өнімділіктен айырылмайтын стиль. Сондықтан
кодты тазарту үшін, мағынасын анық беру үшін, циклдерді қысқарту
үшін — итераторлар мен жабуларды еркін қолданыңыз.

Келесі тарауда — `Cargo` мен `crates.io`-ның жетіленген мүмкіндіктерін
қарастырамыз: бейіндер (профилдер), құжаттама жариялау, жұмыс
кеңістіктері, бинарларды орнату, өзіндік командалар.
