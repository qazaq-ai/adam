# 4-тарау. Иелікті түсіну

Иелік (ownership) — `Rust`-тың ең ерекше және ең күшті ұғымы. Ол `Rust`
бағдарламаларына жадыны қоқыс жинаушысыз да, әрбір `malloc`/`free`
жұбын қолмен басқарусыз да басқаруға мүмкіндік береді. Бұл иеліктің
қалай жұмыс істейтіні маңызды — өйткені сіздің кодыңыз `Rust`-та қалай
жазылатыны иелікке тікелей байланысты.

Иелік жүйесі үш ұғымнан тұрады: **иелік**, **қарызға алу** (борровинг)
және **тілімдер** (slices). Осы тарауда үшеуін бірге қарастырамыз.

## 4.1. Иелік дегеніміз не?

Иелік — `Rust` бағдарламасы жадыны қалай басқарады деген сұраққа жауап.
Кейбір тілдер қоқыс жинаушыны (garbage collector) қолданады — ол
бағдарлама жүріп жатқанда қажетсіз жадты іздеп тазалайды. Басқа тілдерде
программист жадты қолмен бөліп, қолмен босатады. `Rust` үшінші жолды
таңдады: жад **иелік ережелерімен** басқарылады, оларды компилятор
тексереді. Егер ережелер бұзылса, бағдарлама құрастырылмайды. Орындалу
кезінде иелік жүйесі ешбір жылдамдықты «жемейді» — бар тексерулер
компиляцияда жасалады.

### Стек пен үйме

Иелікті түсіну үшін стек пен үйме арасындағы айырмашылықты есте сақтау
маңызды.

**Стек** — соңғы кірген бірінші шығатын (LIFO) тәртіппен жұмыс істейтін
жад аймағы. Стекке нәрсе салу мен алу жылдам, өйткені орналасу қашан да
үстіңгі жағында болады. Бірақ стекке тек өлшемі белгілі мәндерді сақтауға
болады.

**Үйме** — өлшемі алдын ала белгісіз, не өлшемі орындалу кезінде өзгеретін
мәндер сақталатын жад аймағы. Үймеге бір нәрсе сақтау үшін операциялық
жүйе жеткілікті орынды табу керек, оны белгілеу керек және оның
**сілтемесін** қайтару керек. Бұл стекпен салыстырғанда баяуырақ. Үймедегі
жадқа қол жеткізу үшін бір қосымша «секіру» (сілтемесімен жүру) керек.

Функция шақырылған кезде оның параметрлері мен жергілікті айнымалылары
(олардың кейбірі үймедегі мәндерге сілтемелер болуы мүмкін) стекке
тіркеледі. Функция аяқталғанда сол мәндердің бәрі стектен босатылады.

Иелік жүйесі осы жадты қалай ұтымды басқару дегенге жауап береді.
Дұбликаттарды табу, қажетсіз жадты тазалау, артық көшіру жасамау —
бұларды компилятор иелік ережелері арқылы автоматты түрде шешеді.

### Иелік ережелері

Иеліктің үш негізгі ережесі бар; оларды есте ұстаңыз — қалған бәрі осы
үшеуінің салдары:

1. `Rust`-тағы әрбір мәннің **иесі** деп аталатын бір айнымалысы бар.
2. Бір уақытта мәннің тек бір ғана иесі болуы мүмкін.
3. Иесі аумақтан шыққанда мән өшіріледі.

Алғашқы екі ереже статикалық: компилятор тексереді. Үшінші ереже —
динамикалық: бағдарлама орындалу кезінде орындалады.

### Айнымалы аумағы

Айнымалының аумағы — бағдарламаның ол жарамды болатын бөлігі. Айнымалы
жарияланғаннан бастап оны қамтитын блок аяқталғанша жарамды:

```rust
fn main() {
    {                      // s әлі белгісіз
        let s = "сәлем";   // s осы жерден бастап жарамды
        // s-пен жұмыс істеуге болады
    }                      // блок бітті, s енді жарамсыз
}
```

Бұл `s` айнымалысы ішкі блок ішінде ғана белсенді. Блок бітісімен `s`-тің
аумағы аяқталады.

### `String` түрі

Бұрын біз `&str` (жол литералы) типімен таныстық. Жол литералы кодқа
қатты жазылатын тұрақты мән — оның мазмұны мен ұзындығы компиляция кезінде
белгілі. Бірақ кейбір жағдайларда бізге **өсетін, өзгертілетін** жол
керек. Соған арналған тип — `String`.

`String` үймеде сақталады, ұзындығы орындалу кезінде өзгеруі мүмкін.
Жол литералынан `String` жасау үшін `from` функциясын қолданамыз:

```rust
let s = String::from("сәлем");
```

`::` операторы мұнда `from` функциясы `String` түріне тиесілі екенін
білдіреді. `String` түрі өзгертуге келеді — егер `mut` қойсаңыз, оған
жаңа таңбаларды қосуға болады:

```rust
let mut s = String::from("сәлем");
s.push_str(", дүние!");
println!("{s}"); // "сәлем, дүние!"
```

Енді негізгі сұрақ: **неге** `String` өзгертуге келеді, ал жол литералы
келмейді? Жауабы — олардың жадпен жұмыс істеу тәсілінде.

### Жад және бөлу

Жол литералы жылдам, өйткені оның мазмұны компилятор тарапынан тікелей
орындалатын файлға жазылады. Жол литералын өзгерту мүмкін емес — өйткені
оның көлемі компиляция кезінде белгілі.

`String` үшін орындалу кезінде көлемі өзгеруі мүмкін мәтінді сақтай білу
керек. Бұған жадты үймеден алу қажет:

- Жадты бөлу үшін біз операциялық жүйеден бір рет сұраймыз — `String::from`
  ішінде.
- Жадты пайдалану біткеннен кейін біз операциялық жүйеге қайтаратын тәсіл
  керек.

Бірінші бөлік сізге таныс — бірақ екінші бөлік әртүрлі тілдерде әртүрлі
жасалады. Қоқыс жинаушысы бар тілдерде ол автоматты түрде орындалады
(GC қолданылмайтынды табады және босатады). Қоқыс жинаушысы жоқ тілдерде
программист жадты қолмен босатады — бұған бір ұмытсаңыз, жад жоғалады
(ағытпа); екі рет босатсаңыз — қате; кеш босатсаңыз — қате.

`Rust`-тың тәсілі екеуінен де өзгеше: айнымалы өзінің аумағынан шыққан
сәтте жад **автоматты түрде** қайтарылады. Бұл механизмді `drop` функциясы
жасайды; ол ішкі әрі үнсіз шақырылады, бірақ оны мысалдың соңында ашық
түрде көрсетсек:

```rust
{
    let s = String::from("сәлем"); // s жарамды
    // ... s-пен жұмыс ...
} // блок аяқталды → drop(s) → жад босатылды
```

Бұл — иеліктің ең қарапайым көрінісі.

### Иеліктің көшуі

Бірнеше айнымалы бір мәнмен әртүрлі түрде «байланысты» болуы мүмкін.
`Rust`-та бұл түрлер тиянақты түрде ажыратылған.

Бүтін санның жағдайын қарастырайық:

```rust
let x = 5;
let y = x;
```

`x` пен `y` екеуі де 5 мәнімен байланысты. Стектегі бүтін санның көлемі
тұрақты және белгілі — `5`-тің көшірмесі стектегі `y`-ге сақталады. Әр
айнымалы өз дербес мәніне ие.

Енді `String` үшін:

```rust
let s1 = String::from("сәлем");
let s2 = s1;
```

Бір қарағанда бұл бүтін санның мысалына ұқсайды. Бірақ `String` басқаша
жасалған. `String` ішкі түрде **үш бөліктен** тұрады: үймедегі мәтінге
сілтейтін көрсеткіш, ұзындық және сыйымдылық. Осы үшеуі стекте сақталады.

`let s2 = s1;` сөйлемі стектегі үш мәнді (көрсеткіш + ұзындық + сыйымдылық)
көшіреді — бірақ үймедегі мәтін көшірілмейді. Олай болса, `s1` мен `s2`
екеуі де бір үймедегі мәтінге сілтейді. Бұл — мәселе! Аумақ біткенде екеуі
де `drop`-қа барады, екеуі де бір жадты босатуға тырысады. **Қос босату
қатесі** — қауіпсіздік мәселесі.

`Rust`-тың шешімі: `let s2 = s1;` сөйлемінен кейін **`s1` ары қарай
жарамсыз болады**. `Rust` көрсеткішті `s1`-ден `s2`-ге **«ауыстырды»
(moved)**, ал `s1`-ді істен шығарды:

```rust
let s1 = String::from("сәлем");
let s2 = s1;

println!("{s1}"); // ҚАТЕ: s1 енді жарамсыз
```

Компилятор бұған қателік бере алады:

```text
error[E0382]: borrow of moved value: `s1`
```

Бұл — иелікті ауыстыру (move). `String`-ке арналған «терең көшіру» жоқ —
тек иелік ауысады.

### Терең көшіру: `clone`

Егер сізге `String` мәнін шынымен көшіріп алу қажет болса, `clone`
әдісін қолданыңыз. Ол үймедегі мазмұнды да көшіреді:

```rust
let s1 = String::from("сәлем");
let s2 = s1.clone();

println!("s1 = {s1}, s2 = {s2}"); // дұрыс жұмыс істейді
```

`clone` шақырылғанда бағдарлама шынымен жадтан жаңа орын сұрайды және
мазмұнды копиялайды. Бұл арзан емес — `clone` бар жерде сіз жадты
дублирлеп жатқаныңызды біліп тұрсыз.

### `Copy` трейті

Бүтін сандар, `bool`, `char`, `f32`/`f64`, ұқсас стек қана түрлер —
жоғарыдағы «move» проблемасы оларда жоқ. `let y = x;` соларда — әркез
жай көшірме. `Rust` мұны **`Copy` трейті** арқылы көрсетеді: `Copy`
бекітілген типті айнымалыны басқаға тағайындағанда — иелік ауыспайды,
тек жаңа көшірме жасалады.

`Copy` бекітілген түрлер:

- Барлық бүтін сан түрлері (`i32`, `u64`, `usize` т.б.).
- `bool`, `char`.
- `f32`, `f64`.
- Тек `Copy`-ден тұратын кортеж: `(i32, i32)` — `Copy`; `(i32, String)` —
  жоқ.

`Copy` бар жерде move жоқ. Бұл — `Rust` дизайнының тағы бір сипаты:
қарапайым стек-түрлерді еркін қолдана беруге болады.

### Иелік пен функциялар

Айнымалыны функцияға беру де `let`-ке тағайындау сияқты — иелікті
ауыстыру немесе көшіру жасайды. Қай тип екеніне байланысты:

```rust
fn main() {
    let s = String::from("сәлем");
    takes_ownership(s); // s-тің иелігі функцияға өтті
    // s енді жарамсыз

    let x = 5;
    makes_copy(x); // i32 — Copy, көшірме жасалды
    println!("{x}"); // x әлі де жарамды
}

fn takes_ownership(some_string: String) {
    println!("{some_string}");
} // some_string аумақтан шықты, drop шақырылды

fn makes_copy(some_integer: i32) {
    println!("{some_integer}");
} // some_integer тек стекте тұрды, ештеңе жоқ
```

Функцияға `String` берсеңіз — иелік сіздің иеліктен функцияның иелігіне
ауысады. Функция аяқталғанда жад босатылады. Сіздің бастапқы айнымалыңыз
ары қарай жарамсыз.

Функцияға `i32` берсеңіз — `Copy` бекітілгендіктен, көшірме жасалады және
бастапқы айнымалы әлі де жарамды.

### Қайтару мәні мен иелік

Қайтару мәндері де иелікті ауыстыра алады:

```rust
fn main() {
    let s1 = gives_ownership();         // gives_ownership-тың қайтаратын
                                        // мәні s1-ге ауысады
    let s2 = String::from("сәлем");
    let s3 = takes_and_gives_back(s2);  // s2 функцияға өтеді,
                                        // функцияның қайтару мәні s3-ке өтеді
}

fn gives_ownership() -> String {
    let some_string = String::from("қайтару");
    some_string                         // қайтарылады, иелік ауысады
}

fn takes_and_gives_back(a_string: String) -> String {
    a_string                            // қабылдағанымызды қайта қайтарамыз
}
```

Бірақ әр функция шақырылған сайын иелікті алып, кейін қайтарып отыру
қолайсыз. Сондықтан `Rust`-та **қарызға алу** ұғымы бар.

## 4.2. Сілтемелер мен қарызға алу

Сілтеме (reference) — мәнге **иеленбей** қол жеткізуге мүмкіндік
беретін құрылым. Сілтеме арқылы біз функцияға мәнді көрсетеміз, бірақ
оның иелігін бермейміз. Сілтемені жасау — `&` операторы:

```rust
fn main() {
    let s1 = String::from("сәлем");
    let len = calculate_length(&s1);
    println!("'{s1}' жолының ұзындығы — {len}");
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```

`&s1` — `s1`-ге сілтеме. Сілтеме мәнді көрсетеді, бірақ оған иелік
бермейді. Сондықтан `calculate_length` функциясы аяқталғанда `s` аумақтан
шықса да, ол сілтеген мән (яғни `s1`-дегі мәтін) `drop`-қа бармайды.
`s1` `main`-да әлі де жарамды, оны әрі қарай қолдана беруге болады.

Біз сілтемені жасап, оны функцияға беру тәсілін **қарызға алу**
(borrowing) деп атаймыз. Сілтеме арқылы біз мәнді уақытша «қарызға»
аламыз. Қарыз біткенде иелік иесінде қалады.

### Өзгермелі сілтемелер

Әдепкі бойынша сілтеме арқылы біз мәнді **өзгерте алмаймыз**:

```rust
fn main() {
    let s = String::from("сәлем");
    change(&s); // ҚАТЕ: тұрақты сілтеме арқылы өзгерте алмайды
}

fn change(some_string: &String) {
    some_string.push_str(", дүние");
}
```

Өзгертуге рұқсат алу үшін **өзгермелі сілтеме** жасау керек:

```rust
fn main() {
    let mut s = String::from("сәлем");
    change(&mut s);
}

fn change(some_string: &mut String) {
    some_string.push_str(", дүние");
}
```

Үш өзгеріс: `s` `mut` болуы керек; `&mut s` арқылы өзгермелі сілтеме
жасаймыз; функцияның параметрі `&mut String` түрінде болады.

### Сілтемелердің екі ережесі

`Rust`-тың сілтеме жүйесінің негізі — екі қатаң ереже:

1. Бір уақытта мәнге **бір ғана** өзгермелі сілтеме болуы мүмкін; немесе
   **бірнеше** тұрақты сілтеме болуы мүмкін; бірақ екеуі бірдей бола
   алмайды.
2. Сілтемелер әрқашан жарамды болуы керек (жабайы сілтеме жоқ).

Бірінші ереже — **жарыс шарты** (data race) проблемасына қарсы
кепілдік. Жарыс шарты — екі ағын бір жадқа бір уақытта жазып
жатқанда болатын мәселе. Сондай қателер табуы өте қиын. `Rust` бұған
компиляцияда тыйым салады:

```rust
let mut s = String::from("сәлем");

let r1 = &mut s;
let r2 = &mut s; // ҚАТЕ: бір уақытта екі өзгермелі сілтеме болмайды

println!("{}, {}", r1, r2);
```

Бұл бір аумаққа екі өзгермелі сілтеме болуға тырысқан үшін `Rust`
рұқсат бермейді.

Бірнеше тұрақты сілтеме — мәнді тек оқу үшін — рұқсат:

```rust
let s = String::from("сәлем");
let r1 = &s;
let r2 = &s;
let r3 = &s;
println!("{}, {}, {}", r1, r2, r3); // дұрыс — бірнеше тұрақты сілтеме
```

Тұрақты және өзгермелі сілтемені бір аумақта араластыруға тыйым:

```rust
let mut s = String::from("сәлем");
let r1 = &s;
let r2 = &s;
let r3 = &mut s; // ҚАТЕ: r1/r2 әлі белсенді тұрса
println!("{}, {}, {}", r1, r2, r3);
```

Сілтеменің аумағы бұл жерде — соңғы пайдаланудан кейін жабылады. Сондықтан
келесі код заңды:

```rust
let mut s = String::from("сәлем");

let r1 = &s;
let r2 = &s;
println!("{} мен {}", r1, r2); // r1, r2 осы жерден кейін қолданылмайды

let r3 = &mut s;
println!("{}", r3); // енді өзгермелі сілтеме жасауға болады
```

### Жабайы сілтемелер

Жабайы сілтеме (dangling reference) — мән жоғалғаннан кейін соған сілтеп
тұратын сілтеме. Көп тілдерде бұл қателіктің бір көзі: бағдарлама
жадтан тазартылған орынды оқуға тырысады. `Rust` мұны компиляцияда
тыйым салады:

```rust
fn dangle() -> &String { // ҚАТЕ: жабайы сілтеме қайтарады
    let s = String::from("сәлем");
    &s
} // s аумақтан шықты, drop шақырылды;
  // s-ке сілтеме әлі қолданылса — жабайы
```

Компилятор:

```text
error[E0106]: missing lifetime specifier
this function's return type contains a borrowed value, but there is
no value for it to be borrowed from
```

Бұл қатеге шешім — функция мәнді қайтарып, оның иелігін шақырушыға
ауыстыру керек:

```rust
fn no_dangle() -> String {
    let s = String::from("сәлем");
    s
}
```

## 4.3. Тілім түрі

Тілім (slice) — иелігі жоқ, ұжымның бір бөлігіне сілтейтін сілтеме. Жиі
жолдың немесе жиымның белгілі бір бөлігін сипаттау үшін қолданылады.

### Жол тілімі

`String` ішінен бір сөзді алу мысалы:

```rust
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }

    &s[..]
}
```

`&s[0..i]` — `String` ішіндегі **жол тілімі**. Ол `&str` түрінде болады
және үймедегі `String`-тің бір бөлігіне сілтейді. Жол тілімі — иелігі
жоқ көрініс, өзіндік жадты алмайды.

`..` — диапазон синтаксисі: `[a..b]` `a`-дан бастап `b`-ге дейінгі (b
кірмейді) бөлікке сілтейді. `[..n]` нөлден бастап `n`-ге дейінгі;
`[m..]` `m`-нен жолдың соңына дейін; `[..]` бүкіл жолға.

### Жол тілімдері параметр ретінде

Функция параметр түрін `&String`-тен `&str`-ге өзгертсек, ол `String`
және жол литералы екеуімен де жұмыс істей алады:

```rust
fn first_word(s: &str) -> &str {
    // ... бұрынғы код өзгеріссіз ...
}

fn main() {
    let my_string = String::from("сәлем дүние");
    let word = first_word(&my_string);   // &String → &str автоматты түрде
    let word = first_word(&my_string[..]); // нақты тілім
    let word = first_word("сәлем дүние"); // жол литералы → &str
}
```

`&str` — `&String`-тен жалпырақ тип. Сол үшін `Rust`-та функция
параметрін `&str` түрінде жариялау жиі ұсынылады.

### Басқа тілімдер

Тілім тек жолға арналмаған. Жиым тілімі де бар:

```rust
let a = [1, 2, 3, 4, 5];
let slice = &a[1..3]; // [2, 3] — жиымның тілімі, &[i32] түрінде
```

Бұл `&[i32]` түрінде болады — i32 элементтерінен тұратын тілім. Тілім
жадтан жаңа орын алмайды; ол жай ғана бар жиымның бір бөлігіне сілтейді.

## 4.4. Тарау қорытындысы

Бұл тарауда сіз `Rust`-тың ең ерекше ұғымы — иелік жүйесімен таныстыңыз.
Иелік ережелері (бір ие, иелік ауысады немесе көшіріледі, аумақ біткенде
жад босатылады), сілтемелер мен қарызға алу (тұрақты және өзгермелі
сілтеме, екі ереже), жабайы сілтемелердің тыйым салынуы, тілімдер
(жолдың не жиымның бір бөлігіне иелігі жоқ көрініс) — `Rust`-тың
қауіпсіздік кепілдіктерінің барлығы осы ұғымдарға арналған.

Иелік жүйесі күрделі болып көрінуі мүмкін, бірақ оның бір артықшылығы
бар: компилятор сізге тиісті жерде ескертеді, бағдарлама орындалмастан
бұрын қателіктер табылады. Уақыт өте келе бұл ережелерге үйреніп, оларды
ой автоматы ретінде қолданасыз.

Келесі тарауда `Rust`-тың тағы бір негізгі құрылымына — структтарға —
көшеміз. Олар арқылы өзіңіздің тип жасап, оған әдістер бекіте аласыз.
