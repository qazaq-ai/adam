# 14-тарау. Cargo пен Crates.io туралы тереңірек

Cargo алдыңғы тарауларда жобаңызды құрастыру, орындау және сынау үшін
жеткілікті құрал болатынын көрсетті. Бірақ оның мүмкіндіктері әлдеқайда
кең. Бұл тарауда:

- Шығарылым **бейіндерін** (release profiles) баптау;
- Кодыңызды `crates.io`-ға **жариялау**;
- **Жұмыс кеңістігі** (workspaces) — бірнеше байланысты сандықтың
  жобасын ұйымдастыру;
- `crates.io`-дан бинарларды **орнату**;
- Cargo-ны өзіндік командалармен **кеңейту**.

Осы құралдар арқылы сіз кодыңызды қауымдастықпен бөлісуге, үлкен
жобаларды басқаруға, жұмысыңызды басқалардың пайдасына шығаруға
қабілетті боласыз.

## 14.1. Шығарылым бейіндерімен құрастыруды баптау

Cargo сізге **бейіндер** (profiles) ұсынады — әрқайсысы өз баптамасы
бар құрастыру конфигурациялары. Стандартты екеуі:

- **`dev`** бейіні — `cargo build` және `cargo run` шакыратынында
  қолданылады. Жылдам құрастыруға арналған, оптимизация төмен.
- **`release`** бейіні — `cargo build --release` шакыратынында
  қолданылады. Шығарылым (production) кодын оптимизациялауға
  арналған.

Әдепкі баптамалар жеткілікті көп жағдайда. Бірақ Cargo сізге `Cargo.toml`
ішінде оларды қайта анықтауға мүмкіндік береді:

```toml
[profile.dev]
opt-level = 0

[profile.release]
opt-level = 3
```

`opt-level` — оптимизация деңгейі. 0-ден 3-ке дейінгі бүтін сан. Жоғары
деңгей — кодыңыз жылдамырақ орындалады, бірақ құрастыру ұзағырақ. Әдепкі
бойынша:

- `dev` бейінінде `opt-level = 0` — мүлдем оптимизация жоқ. Сондықтан
  құрастыру жылдам.
- `release` бейінінде `opt-level = 3` — толық оптимизация.

Кейде `dev` бейінінде сізге қандай да бір оптимизация керек болады
(мысалы, тестілерді жылдамдатуға). Сонда `Cargo.toml`-да жоғарғы
мысалдағыдай `opt-level`-ді өзгертіп жасасаңыз болады. Cargo сіздің
баптамаңызды әдепкінің үстіне жазады.

## 14.2. Сандықты Crates.io-ға жариялау

[crates.io](https://crates.io) — Rust сандықтарының ресми орталық
қоймасы. Сіз де өзіңіздің сандығыңызды осында жариялап, басқа
программистерге пайдаланды шығара аласыз.

### Пайдалы құжаттама түсініктемелерін жасау

`Rust`-та арнайы **құжаттама түсініктемелері** бар. Олар `///` (үш
қиғаш) немесе `//!` арқылы басталады және Markdown синтаксисін
қолдайды. `cargo doc` командасы сол түсініктемелерден HTML құжаттама
жасайды.

Әдеттегі функцияға арналған құжаттама:

```rust
/// Берілген санға бір қосады.
///
/// # Мысалдар
///
/// ```
/// let arg = 5;
/// let answer = my_crate::add_one(arg);
///
/// assert_eq!(6, answer);
/// ```
pub fn add_one(x: i32) -> i32 {
    x + 1
}
```

`cargo doc --open` командасы құжаттаманы құрастырып, оны браузерде
ашады. Сіздің `add_one` функциясы әдемі түрде көрсетіледі — қысқа
сипаттамасымен, мысалымен, типтерімен.

### Кездейсоқ ескертулерге арналған бөлімдер

Құжаттама түсініктемелеріне арнайы бөлімдер қосу қабылданған:

- **`# Examples`** — мысалдар.
- **`# Panics`** — қандай жағдайда `panic!`-қа барады.
- **`# Errors`** — қандай қателер болуы мүмкін (функция `Result`
  қайтаратын болса).
- **`# Safety`** — функция `unsafe` болса, оны қалай қауіпсіз қолдану
  керек.

Бұл бөлімдер кездессе — `cargo doc` оларды арнайы пішімдейді.
Қауымдастық осыларды жаппай қолданады, сондықтан сіздің сандығыңыз
осы рәсімге сай болса — оны пайдаланушы тез түсінеді.

### Тестілеу мысалдар арқылы

Құжаттамадағы `### Мысалдар` бөлімінде жазылған код **орындалатын
тест** болады. `cargo test` оларды табады, құрастырады және орындайды.
Сондықтан сіз құжаттамадағы мысалдардың әрқашан жұмыс істейтінін
жастатуыңыз керек — оларды компилятор әр шакыруда тексереді.

Бұл — өте күшті: құжаттамадағы код *ескірмейді*. Сіздің API өзгерсе —
мысалдар да тура өзгеруі керек, әйтпесе тестілер кетеді.

### Контейнер ішіне арналған түсініктеме

`//!` түсініктемелері — модульдің **ішінен** жазылады және сол модульді
немесе сандықты сипаттайды:

```rust
//! # My Crate
//!
//! `my_crate` — өрнек жинағы; ұсақ-түйекті есептеулерді
//! қарапайым жасайды.

/// Берілген санға бір қосады.
pub fn add_one(x: i32) -> i32 {
    x + 1
}
```

`//!` `src/lib.rs` файлының басына қойылса — бүкіл сандыққа арналған
түсініктеме болады.

### `pub use` арқылы ыңғайлы ашық API экспорттау

Сандықтың ішкі ұйымдастыру жиі сыртқы пайдаланушыға қажетсіз. Олардың
қажеті — қарапайым ашық API, бір қалтада жинақталған. Соған `pub use`
қызмет етеді:

```rust
//! # Art
//!
//! Бұл сандық — өнердегі ұғымдарды үлгілейді.

pub use self::kinds::PrimaryColor;
pub use self::kinds::SecondaryColor;
pub use self::utils::mix;

pub mod kinds {
    pub enum PrimaryColor { Red, Yellow, Blue }
    pub enum SecondaryColor { Orange, Green, Purple }
}

pub mod utils {
    use crate::kinds::*;
    pub fn mix(c1: PrimaryColor, c2: PrimaryColor) -> SecondaryColor {
        // ...
        unimplemented!()
    }
}
```

Кітапхананы пайдаланушы:

```rust
use art::PrimaryColor;
use art::mix;
```

жаза алады — `art::kinds::PrimaryColor` жолы керек емес. Сонымен қатар
`art::kinds::PrimaryColor` де жұмыс істейді — сіз ішкі ұйымдастыруды
сақтайсыз. `pub use` тек қосымша «жанама» экспорт жасайды.

### `crates.io`-да есептік жазба құру

Жариялау үшін:

1. https://crates.io сайтына кіріңіз. GitHub арқылы кіруге болады.
2. https://crates.io/me бетінен API токен алыңыз.
3. Терминалда `cargo login <api-token>` орындаңыз.

Cargo токенді `~/.cargo/credentials.toml` файлында сақтайды. Ол —
жасырын. Оны бөтенге бермеңіз.

### Сандыққа метамәліметтерді қосу

`Cargo.toml`-да міндетті өрістер:

```toml
[package]
name = "guessing_game"
version = "0.1.0"
edition = "2021"
description = "Сандар табу ойыны — Rust Book оқуын демонстрациялайды."
license = "MIT OR Apache-2.0"
```

- **`name`** — бүкіл `crates.io`-да бір ғана. Біреу алып қойса —
  басқаны таңдау керек.
- **`version`** — әрбір жариялау сайын ауыстыру керек. **Семантикалық
  нұсқалау** ережелерін ұстаңыз: тек кейде өзгертіңіз
  (`MAJOR.MINOR.PATCH`).
- **`description`** — қысқаша сипаттама, бір сөйлем.
- **`license`** — әдеттегі ашық кодты лицензия идентификаторы. `MIT`,
  `Apache-2.0`, `MIT OR Apache-2.0` — Rust қауымдастығының жиі
  таңдауы.

Қосымша өрістер: `repository` (қойма URL-ы), `homepage`, `documentation`,
`keywords`, `categories`, `readme`. Көп болғаны — пайдаланушыға
жылдамырақ табуға, түсінуге көмек.

### Жариялау

```sh
$ cargo publish
```

`cargo publish` сандықты `crates.io`-ға жібереді. Жариялау — **қайтымсыз**:
бір рет жіберілген код әрдайым сол нұсқада қол жетімді болады. Себебі
басқа адамдар сіздің сандығыңызға тәуелді жобаларын құрастыруы мүмкін.

Сондықтан жариялауға дейін:

- `cargo publish --dry-run` — нені жіберетіндігіңізді көруге.
- Кодты — әсіресе ашық API-ды — жақсы тестілеп шығу керек.
- Семантикалық нұсқалауды нақты ұстау керек.

### Жаңа нұсқаны жариялау

Кодыңызды өзгертіп, жаңа нұсқаны жариялау керек болса — `Cargo.toml`-да
`version`-ды көтеріп, `cargo publish` шакырыңыз. Семантикалық
нұсқалауға сай:

- **PATCH** (`0.1.0 → 0.1.1`) — қосалқы ішкі түзетулер, бар API өзгерген
  жоқ.
- **MINOR** (`0.1.0 → 0.2.0`) — жаңа қосылған мүмкіндіктер; бар API
  бұзылған жоқ.
- **MAJOR** (`0.1.0 → 1.0.0` немесе `1.0.0 → 2.0.0`) — бар API өзгерді
  немесе бұзылды.

### Нұсқаларды `cargo yank` арқылы өшіру

Бір нұсқаны жариялап, кейін онда қате бар екенін көрсеңіз — оны жоюға
болмайды. Бірақ оны **«жойылған» деп белгілеу** мүмкін. Сонда жаңа
жобалар оны қолданбайды, ал бұрын солған жобалар әлі де жұмыс істей
береді.

```sh
$ cargo yank --vers 1.0.1
```

Кері қайтару:

```sh
$ cargo yank --vers 1.0.1 --undo
```

`yank` — қатесі бар нұсқалар үшін, мысалы қауіпсіздік мәселесі бар
нұсқа. Жоюға келеді, бірақ ескі қолданушылар сақталады.

## 14.3. Cargo жұмыс кеңістігі

Жоба үлкейіп, кітапхана сандығы бірнеше тәуелсіз бөліктен тұратын
болса — оларды бір бумада ұстау ыңғайсыз. **Жұмыс кеңістігі** дегеніміз
— бір тамырлы Cargo бумалар-жинағы, бір `target/` қалтасында
бірлескен.

### Жұмыс кеңістігін құру

`add` атты қалта жасайық, ішінде `Cargo.toml`:

```toml
[workspace]
members = [
    "adder",
    "add_one",
]
resolver = "2"
```

Содан кейін `cargo new adder` — `add` қалтасының ішінде. Және
`cargo new add_one --lib` тағы солай. Қалта құрылымы:

```text
add/
├── Cargo.toml
├── adder/
│   ├── Cargo.toml
│   └── src/
│       └── main.rs
└── add_one/
    ├── Cargo.toml
    └── src/
        └── lib.rs
```

`adder` — екілік сандық; `add_one` — кітапхана. Әрқайсының өз
`Cargo.toml`-ы бар. Жұмыс кеңістігінің ұялы файлы (root) ортақ
`target/` қалтасын анықтайды.

### Бумалардың арасындағы тәуелділік

`adder/Cargo.toml`-да:

```toml
[dependencies]
add_one = { path = "../add_one" }
```

Енді `adder/src/main.rs` ішінде:

```rust
use add_one;

fn main() {
    let num = 10;
    println!("Hello, world! {} плюс бір — {}", num, add_one::add_one(num));
}
```

Жұмыс кеңістігінде `cargo build` барлық сандықты бір өткелде
құрастырады. `cargo run -p adder` — нақты `adder` бумасын іске қосады.

### Жұмыс кеңістігіндегі сыртқы тәуелділіктер

Әр бума өзінің `Cargo.toml`-да тәуелділіктерді жеке тізбелейді.
Стандартты тұрақты — бір тәуелділікті бір ғана нұсқада қолдану.
Сондықтан жұмыс кеңістігінде ортақ `Cargo.lock` файлы қолданылады —
барлық бумалар бір нұсқалармен құрастырылады.

### Сынақтарды жұмыс кеңістігінде

`cargo test` бүкіл жұмыс кеңістігіндегі сынақтарды жүгіртеді.
`cargo test -p add_one` — нақты бумадағы сынақтарды.

## 14.4. `cargo install` арқылы бинарларды Crates.io-дан орнату

`cargo install` командасы `crates.io`-дан **бинарлы сандықтарды**
орнатуға мүмкіндік береді. Бинарлы сандық — `src/main.rs` (немесе
`src/bin/`) бар, орындалатын бағдарлама шығаратын сандық.

Бинарлы сандықты орнату:

```sh
$ cargo install ripgrep
```

`ripgrep` — `grep`-тің `Rust`-та жазылған, әлдеқайда жылдам нұсқасы.
`cargo install` оны `~/.cargo/bin/` қалтасына орнатады. Ол сіздің
`PATH`-те болуы керек — `rustup` орнатқан кезде ол автоматты түрде
қосылады.

Орнатылғаннан кейін `rg` командасы қол жетімді:

```sh
$ rg "search query" .
```

`cargo install` көп жағдайда сізге Rust қауымдастығының жасаған
қызыңды қызметтерді бір команданмен орнатуға мүмкіндік береді.
Әсіресе пайдалы: `ripgrep`, `bat`, `fd-find`, `tokei`, `cargo-watch`,
`cargo-edit`.

## 14.5. Cargo-ны өзіндік командалармен кеңейту

Cargo бір қызықты ерекшелікке ие: егер сізде `~/.cargo/bin/` ішінде
`cargo-something` атты бинар болса — оны `cargo something` арқылы
шакыруға болады. Cargo сол бинарды табады және оны ішкі команда
ретінде орындайды.

Мысалы, `cargo install cargo-watch` орнатқаннан кейін:

```sh
$ cargo watch -x test
```

— `cargo watch` командасы бар; іс жүзінде ол `cargo-watch` бинары.

Бұл — Cargo-ны кеңейтудің оңай жолы. Қауымдастық сансыз пайдалы
кеңейтулерді жасап шығарды:

- **`cargo-edit`** — `Cargo.toml`-ды редакторсыз өзгерту командалары
  (`cargo add`, `cargo rm`).
- **`cargo-watch`** — өзгерістерді бақылап, командаларды автоматты
  түрде қайта шакыру.
- **`cargo-audit`** — тәуелділіктерде белгілі қауіпсіздік
  ақаулықтарын тексеру.
- **`cargo-tree`** — тәуелділік ағашын көрсету.
- **`cargo-bloat`** — бинарды кім үлкейтіп жатқанын анықтау.

Сіз де өзіңіздің `cargo-foo` бинарын жазып, оны кейіннен `cargo foo`
арқылы пайдалана аласыз.

## 14.6. Тарау қорытындысы

Cargo — `Rust` экожүйесінің тірегі. Шығарылым бейіндері кодтың қалай
құрастырылуын баптауға, `crates.io` коды қауымдастықпен бөлісуге,
жұмыс кеңістіктері үлкен жобаларды ұйымдастыруға, `cargo install`
дайын құралдарды орнатуға, өзіндік командалар Cargo-ны кеңейтуге
мүмкіндік береді.

Бұл — сіздің күн сайынғы жұмысыңызды жеңілдететін практикалық
ұғымдар. `crates.io`-ға бір рет кодыңызды жариялап шыққаннан кейін —
осы экожүйенің бір бөлігі болатыныңыз.

Келесі тарауда — `Box`, `Rc`, `RefCell`, `Deref` мен `Drop` сияқты
