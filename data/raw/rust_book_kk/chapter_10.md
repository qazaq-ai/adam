# 10-тарау. Жалпылама типтер, трейттер мен тіршілік мерзімі

Әр бағдарламалау тілінде — қайталануды азайтуға арналған құралдар бар.
`Rust`-тың сондай құралдарының бірі — **жалпылама типтер**: типтің орнына
тип параметрін қойып, бір функцияны немесе бір типті бірнеше нақты тип
үшін қайтадан жазбай қолдану.

Жалпылама типтермен бірге **трейттер** келеді — олар жалпылама типтердің
қандай тәртіп көрсетуге қабілетті екенін бекітеді. Ал тіршілік мерзімі —
жалпылама түрдің ерекше түрі — компиляторға сілтемелердің жарамды болып
тұратын кезеңін айтады.

Бұл тарау — `Rust`-тағы абстракцияның үш қабатын біріктіреді: типтер,
тәртіп, тіршілік. Үшеуін бір-бірімен араластырып қолданып үйрену —
үлкен `Rust` жобасын жазудың кілті.

## 10.1. Жалпылама мәлімет түрлері

Жалпылама типтер арқылы біз функцияны, структты, енамды немесе әдісті
бірнеше нақты тип үшін бір рет жазамыз.

### Функцияда жалпылама

Жиымның ең үлкен элементін табатын функцияны қарастырайық. Алдымен
`i32` үшін:

```rust
fn largest_i32(list: &[i32]) -> &i32 {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}
```

Содан кейін `char` үшін:

```rust
fn largest_char(list: &[char]) -> &char {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}
```

Кодтың денесі бірдей — тек тип ауысады. Жалпылама типтер арқылы екеуін
біріктірейік:

```rust
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}
```

`<T: PartialOrd>` — бұл тип параметрі. `T` — кез келген тип бола алады,
бірақ `PartialOrd` трейтін бекітетін болуы керек (екі мәнді салыстыру
қабілеті). `largest::<i32>([1, 2, 3])` және `largest::<char>(['a',
'b'])` екеуі де жұмыс істейді.

### Структта жалпылама

Структтың өрістерін жалпылама типке байлайық. Қарапайым нүкте
координаттары:

```rust
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer = Point { x: 5, y: 10 };
    let float = Point { x: 1.0, y: 4.0 };
}
```

`Point<i32>` мен `Point<f64>` — әртүрлі типтер. Әр тип үшін `Rust` бөлек
машиналық код шығарады.

`x` мен `y` бөлек типтерде болуы мүмкін болса — екі параметр енгіземіз:

```rust
struct Point<T, U> {
    x: T,
    y: U,
}

fn main() {
    let p = Point { x: 5, y: 4.0 };  // T = i32, U = f64
}
```

### Енамда жалпылама

Сізге таныс `Option<T>` пен `Result<T, E>` — жалпылама типтер:

```rust
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

`Option<T>` бір параметрмен — мүмкін мәннің типі. `Result<T, E>` екеуімен
— сәтті нәтиже мен қате типі. Сондықтан стандартты кітапханада сансыз
жағдайда `Option<u32>`, `Option<String>`, `Result<File, io::Error>`
сияқты комбинациялар табылады.

### Әдісте жалпылама

Жалпылама структқа әдіс жариялау үшін `impl<T>` қолданамыз:

```rust
struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}
```

`impl<T>` — бұл тип параметрі `Point<T>` структының өзімен сай. Кез
келген `Point` (мысалы, `Point<i32>`, `Point<f64>`) `x()` әдісіне ие.

Кейде нақты бір типке арналған әдістерді жариялау керек:

```rust
impl Point<f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}
```

Бұл әдіс тек `Point<f64>`-та бар — `Point<i32>`-та жоқ. Себебі `powi`
мен `sqrt` тек қалқымалы үтірлі типтерде болады.

### Жалпылама кодтың өнімділігі

Жалпылама типтерді қолдану ешбір қосымша баяу жасамайды — себебі `Rust`
**мономорфтау** жасайды. Компилятор кодты әр нақты тип үшін бөлек
жасайды. Жоғарыдағы `largest<T>` функциясы `i32`-пен бір рет, `char`-мен
екінші рет шақырылса — компилятор екі бөлек функцияны жасайды:
`largest_i32` мен `largest_char` сияқты. Орындалу кезінде ешбір
динамикалық таңдау, ешбір қосымша көрсеткіш шақыруы жоқ.

Бұл — `Rust`-тың дизайн принципі: **абстракция — жоғары; өнімділік —
төмен қабылданбайды**. Сіз жалпылама код жазасыз, компилятор оны нақты
кодқа айналдырады.

## 10.2. Трейттер: ортақ тәртіпті анықтау

Трейт — әр түрлі типтерге ортақ болған тәртіпті бекітетін абстракция.
`Rust`-та трейт басқа тілдерде «интерфейс» немесе «протокол» деп аталатын
ұғымға ұқсайды.

### Трейт жариялау

Әр түрлі мәтін типтерінен қысқа қорытынды (`summarize`) жасайтын
трейт жариялайық:

```rust
pub trait Summary {
    fn summarize(&self) -> String;
}
```

`Summary` — трейттің атауы. Ішінде бір ғана әдіс қолтаңбасы бар —
`summarize`, ол `String` қайтарады. Әдістің денесі жоқ — ол әр типте
жеке іске асырылатын болады.

### Трейтті типке бекіту

Енді трейттің бекітуін жазайық. Бір мысал — мақала, екіншісі — твит:

```rust
pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

pub struct Tweet {
    pub username: String,
    pub content: String,
    pub reply: bool,
    pub retweet: bool,
}

impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("{}: {}", self.username, self.content)
    }
}
```

`impl <Trait> for <Type>` синтаксисі — бұл `<Type>`-ке `<Trait>`-тың
бекітуін бекітеді. Енді `NewsArticle` мен `Tweet` екеуі де
`summarize()` әдісіне ие.

### Орфан-ереже

Бір маңызды ереже — **орфан-ереже**: сіз трейтті типке тек өзіңіздің
сандығыңыздағы трейт болса немесе типтің өзі сіздің сандығыңыздан болса
ғана бекіте аласыз. Мысалы, сіз `Display`-ді `Vec<T>`-ке бекіте
алмайсыз — өйткені екеуі де стандартты кітапханадан. Бірақ
өзіңіздің `MyType`-ке `Display` бекіту — мүмкін; өзіңіздің `MyTrait`-ті
`Vec<T>`-ке бекіту — де мүмкін. Бұл ереже сандықтар арасындағы
дау-дамайды болдырмайды.

### Әдепкі іске асыру

Трейт ішінде әдістің **әдепкі денесін** жазуға болады. Тип бекітуінде ол
қайта анықталмаса — әдепкі қолданылады:

```rust
pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Толық емес...)")
    }
}

impl Summary for NewsArticle {} // әдепкіні алады

let article = NewsArticle { /* ... */ };
println!("{}", article.summarize()); // "(Толық емес...)"
```

Әдепкі іске асыру ішінде трейттің басқа әдістерін шақыруға болады —
тіпті ол әдістер әдепкі денесі жоқ болса да:

```rust
pub trait Summary {
    fn summarize_author(&self) -> String;

    fn summarize(&self) -> String {
        format!("({} жазған толық емес мәтін)", self.summarize_author())
    }
}
```

Енді `Summary`-ді бекітетін типтер тек `summarize_author`-ды жазса
жеткілікті — `summarize` әдепкі бойынша келеді.

### Трейт параметр ретінде

Функция кез келген `Summary`-ді бекітетін типті қабылдау керек болса:

```rust
pub fn notify(item: &impl Summary) {
    println!("Жаңалық! {}", item.summarize());
}
```

`&impl Summary` — «`Summary` трейтін бекітетін кез келген типке
сілтеме». `notify(&article)` мен `notify(&tweet)` — екеуі де жұмыс
істейді.

### Трейт шектеуі

`impl Trait` синтаксисі қысқа, бірақ кейде нақты тип параметрі ыңғайлы:

```rust
pub fn notify<T: Summary>(item: &T) {
    println!("Жаңалық! {}", item.summarize());
}
```

`<T: Summary>` — «`T` `Summary` трейтін бекітеді» дегенді білдіреді. Бұл
— **трейт шектеуі**. Көп параметрлер бір типте болуы керек жағдайда
шектеу синтаксисі қажет:

```rust
pub fn notify<T: Summary>(item1: &T, item2: &T) { /* ... */ }
```

`impl Summary` синтаксисімен `item1` мен `item2` бір типте болатынын
бекіту мүмкін емес.

### Бірнеше трейт шектеуі

Бір тип бірнеше трейтті бекітуі керек болса — `+` операторы:

```rust
pub fn notify<T: Summary + Display>(item: &T) { /* ... */ }
// немесе
pub fn notify(item: &(impl Summary + Display)) { /* ... */ }
```

Үлкен қолтаңбалар оқуға қиын болады. Сондықтан **`where` клаузасы**:

```rust
fn some_function<T, U>(t: &T, u: &U) -> i32
where
    T: Display + Clone,
    U: Clone + Debug,
{
    // ...
}
```

`where` арқылы шектеулер функция қолтаңбасынан кейінгі бөлек блокта
жазылады. Бұл — оқуға жеңіл.

### Трейтті қайтару

Функция трейтті бекітетін типті **қайтара** алады:

```rust
fn returns_summarizable() -> impl Summary {
    Tweet {
        username: String::from("kazakhstan_news"),
        content: String::from("Бүгінгі ауа-райы — жылы."),
        reply: false,
        retweet: false,
    }
}
```

Қайтару типі `impl Summary` — «`Summary`-ді бекітетін бір тип» дегенді
білдіреді. Шақырушы нақты қандай тип қайтарылғанын білмейді — тек
`Summary` интерфейсіне сүйеніп жұмыс істей алады.

Бірақ нюанс: `impl Summary` тек **бір** нақты тип қайтаруға болады. Егер
функция шартқа қарап `Tweet` немесе `NewsArticle` қайтаратын болса —
ол қателікке барады. Әр түрлі нақты типтерді бір функциядан қайтару
үшін динамикалық сипаттаулы жөндеулер (`Box<dyn Summary>`) керек —
17-тарауда қарастырамыз.

### Шектеулі әдістер

Трейт шектеуі арқылы әдісті тек қалаған типтерде ғана жариялауға болады:

```rust
struct Pair<T> {
    x: T,
    y: T,
}

impl<T> Pair<T> {
    fn new(x: T, y: T) -> Self {
        Self { x, y }
    }
}

impl<T: Display + PartialOrd> Pair<T> {
    fn cmp_display(&self) {
        if self.x >= self.y {
            println!("Үлкеннен: x = {}", self.x);
        } else {
            println!("Үлкеннен: y = {}", self.y);
        }
    }
}
```

`new` барлық `Pair<T>` түрінде бар. Бірақ `cmp_display` — тек `Display`
пен `PartialOrd` екеуін бекітетін `T`-да бар. Қалғандарында — жоқ.

Стандартты кітапхана осы үлгіні көп қолданады. Мысалы, `ToString`
трейті — кез келген `Display`-ді бекітетін типке автоматты түрде
бекітіледі:

```rust
impl<T: Display> ToString for T {
    // әдіс денесі...
}
```

Бұл — **жалпы іске асыру** (blanket implementation). `T: Display` бар
болғанда `T` автоматты түрде `ToString` бекітеді. Сондықтан кез келген
типте `.to_string()` шақыруға болады, егер ол `Display`-ді бекітетін
болса.

## 10.3. Тіршілік мерзімі арқылы сілтемелерді растау

Біз 4-тарауда бар сілтемелерден сөз қозғадық, бірақ оларда жасырын
бір сипат бар екенін айтпай қалдық — әр сілтеменің **тіршілік мерзімі**
бар. Тіршілік мерзімі — сілтеменің жарамды болып тұратын аумақ.

Көп жағдайда тіршілік мерзімі бөлектен туындайды — сілтеме типтері
сияқты. Бірақ кейде функция параметрі әртүрлі мерзімдегі сілтемелерді
қабылдай алатын кезде, біз компиляторға тіршілік мерзімдерінің байланысын
ашық айтуымыз керек.

### Жабайы сілтемелерді болдырмау

Тіршілік мерзімінің бар болуының басты мақсаты — жабайы сілтемелерді
болдырмау. 4-тарауда көрген мысалды еске сақтайық:

```rust
fn main() {
    let r;
    {
        let x = 5;
        r = &x;
    }
    println!("r: {}", r); // ҚАТЕ: x жоғалған, r — жабайы
}
```

`x` ішкі блокта жасалған — блок аяқталғанда `x` `drop`-қа барады. `r`
бірақ сыртта тұр — ол ішкі `x`-ке сілтейді, ал `x` жоғалған. Бұл — типтік
жабайы сілтеме. `Rust` мұны компиляцияда тоқтатады.

### Қарыз тексергіш

Компилятор бұған қалай көз жеткізеді? **Қарыз тексергіш** — мердікке
сілтеменің тіршілік мерзімдерін салыстырады. `r`-дің мерзімі сыртқы
блок (`'a`); `x`-тің мерзімі ішкі блок (`'b`). `'b` `'a`-дан қысқа
болғандықтан, `r = &x` сілтемесі қауіпсіз емес.

### Функцияда жалпылама тіршілік мерзімі

Кейде функция әртүрлі мерзімдегі сілтемелерді қабылдайды:

```rust
fn longest(x: &str, y: &str) -> &str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}
```

Бұл құрастырылмайды:

```text
error[E0106]: missing lifetime specifier
```

Компилятор: «нәтижеге сілтеме қайтарасыз, бірақ ол `x`-тен бе, `y`-тан ба
— білмеймін». Тіршілік мерзімін ашық белгілеу керек:

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}
```

`'a` — тіршілік мерзімінің параметрі. Ол: «`x`, `y` және нәтиже —
үшеуінің тіршілік мерзімі бір — `'a`» дегенді білдіреді. Шақырушы кодқа
бұл шектеу: екі сілтеме шакырылғанда олардың кесінді мерзімі — нәтиженің
жарамды болатын аумағы.

Тіршілік мерзімінің атаулары әдетте `'a`, `'b`, `'c` сияқты қысқа
жазылады.

### Тіршілік мерзімдерін ойлау

Әр функция қолтаңбасында өз ішкі тіршілік мерзімі болады. Ережелер:

- Әр кіріс параметр сілтеме болса — оның өз тіршілік мерзімі бар.
- Қайтару сілтеме — қандайда бір кіріс мерзіміне сай болуы керек.
- Тіршілік мерзімі параметрі қайдан туындайтынын — біз белгілеуіміз
  керек, әйтпесе компилятор шеше алмайды.

### Структта тіршілік мерзімі

Структ ішінде сілтеме болса, оған тіршілік мерзімі қажет:

```rust
struct ImportantExcerpt<'a> {
    part: &'a str,
}

fn main() {
    let novel = String::from("Алыста таулар көк болып көрінеді...");
    let first_sentence = novel.split('.').next().expect("сөйлем табылмады");
    let i = ImportantExcerpt {
        part: first_sentence,
    };
}
```

`ImportantExcerpt<'a>` — бұл структың `part` өрісі қандайда бір тіршілік
мерзіміне `'a`-ға сай. Мән осыдан соң `i` `novel` бар болғанда ғана
жарамды.

### Тіршілік мерзімін түсіріп жазу

Жоғарыдағы `longest` функциясынан байқағанымыздай — тіршілік мерзімдерін
жиі жазудың керегі болуы мүмкін. Бірақ кейбір қарапайым жағдайларда
компилятор оларды өзі шеше алады. Соған **тіршілік мерзімін түсіріп
жазу ережелері** (lifetime elision rules) бар:

1. **Бірінші ереже:** әр кіріс сілтеме параметрге өзіндік мерзімі
   беріледі.
2. **Екінші ереже:** егер тек бір кіріс мерзімі бар болса — қайтару
   сілтеменің мерзімі сондай болады.
3. **Үшінші ереже:** әдістерде, бірінші параметр `&self` немесе `&mut
   self` болса — қайтару сілтеменің мерзімі — `self`-тікі.

Қарапайым мысал:

```rust
fn first_word(s: &str) -> &str { /* ... */ }
```

Тіршілік мерзімі жазылмаған, бірақ компилятор шешеді: бір кіріс сілтеме —
қайтару сондай мерзімде. Сондықтан бұл код:

```rust
fn first_word<'a>(s: &'a str) -> &'a str { /* ... */ }
```

— бір нәрсе.

### Әдісте тіршілік мерзімі

`impl` блогында тіршілік мерзімі параметрі `<'a>` арқылы жарияланады:

```rust
impl<'a> ImportantExcerpt<'a> {
    fn level(&self) -> i32 {
        3
    }

    fn announce_and_return_part(&self, announcement: &str) -> &str {
        println!("Жариялау: {}", announcement);
        self.part
    }
}
```

Үшінші ережеге сай — `announce_and_return_part`-та қайтару `self.part`-тікі
мерзімінде. Сондықтан жазудың керегі жоқ.

### `'static` тіршілік мерзімі

`'static` — арнайы тіршілік мерзімі. Ол: «бағдарламаның бүкіл өмірлік
аумағы» дегенді білдіреді. Барлық жол литералдары — `'static`:

```rust
let s: &'static str = "Бұл — `'static` мерзімдегі жол.";
```

Жол литералы бағдарламаның бинарлы файлында тікелей жазылған, сондықтан
ол әрдайым жарамды.

`'static` мерзімін шектеу ретінде жиі қолдану қажет емес. Кейде
компилятор сізге `'static` ұсынады — бірақ оған бару алдында шынында
жабайы сілтемені жасап жатпағаныңызды тексеріңіз. Көбіне дұрыс шешім —
сілтеме емес, иелік типтерін қолдану.

### Жалпылама тип параметрлері, трейт шектеулері мен тіршілік мерзімі бір кодта

Үш ұғымды бір функцияда жинау — жиі кездеседі:

```rust
use std::fmt::Display;

fn longest_with_announcement<'a, T>(
    x: &'a str,
    y: &'a str,
    ann: T,
) -> &'a str
where
    T: Display,
{
    println!("Жариялау! {}", ann);
    if x.len() > y.len() { x } else { y }
}
```

Мұнда:

- `'a` — тіршілік мерзімі параметрі, `x` мен `y`-ке арналған.
- `T` — жалпылама тип, `ann` параметрі үшін.
- `T: Display` — трейт шектеуі (where клаузасы арқылы).

Үшеуі бірге жұмыс істейді. Бұл — `Rust`-тағы абстракцияның толық суреті:
типтер, мерзімдер, тәртіп.

## 10.4. Тарау қорытындысы

Жалпылама типтер арқылы біз кодты бір рет жазып, бірнеше нақты типке
қолдана аламыз. Трейттер арқылы біз ортақ тәртіпті бекітіп, әр түрлі
типтер — бір интерфейспен — бір функцияға беріле алатынын кепілдейміз.
Тіршілік мерзімі арқылы біз сілтемелердің жабайы болмауын бекітеміз.

Үш ұғым бір кодта араласқанда — `Rust` өзінің ең күшті жағын көрсетеді:
сіз абстрактты, жинақы код жазасыз; компилятор оны нақты, тиімді машиналық
кодқа айналдырады; орындалу кезінде ешбір қосымша баяулау жоқ; жадтың
қауіпсіздігі компиляция кезінде кепілделген.

Келесі тарау — тестілер туралы. Сіз `Rust`-та тестті қалай жазатынды,
қалай іске қосатыныңды, тестілер арқылы кодыңыздың дұрыстығын қалай
қамтамасыз ететіндігіңізді көресіз.
