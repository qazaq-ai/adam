# 5-тарау. Байланысты деректерді структ арқылы құру

Структ — өзара байланысты бірнеше мәнді бір аталған типке біріктіру
тәсілі. Объектілі-бағытталған тілдерден таныс болсаңыз, структ — оның
**мәлімет** бөлігіне (өрістер) ұқсас. Бұл тарауда `Rust`-та структты
қалай анықтайтынды, оны қалай дана етіп жасайтынды және оған әдістерді
қалай бекітетінді қарастырамыз.

Структ алдыңғы тарауда көрген кортежге біршама ұқсас. Кортеж сияқты,
структ те бірнеше мәнді бір топ ретінде сақтайды; әр өріс әртүрлі типке
жатуы мүмкін. Ал кортежден айырмашылығы: структтың әр өрісінде ат бар.
Бұл аттар коды оқырман үшін айқын етеді — нөмір емес, мағыналы белгі.

## 5.1. Структ анықтау және дана жасау

Структты жариялау үшін `struct` кілтсөзі мен типке атау керек. Содан
кейін фигуралы жақшалардың ішіне әр өрістің атауы мен типін көрсетеміз:

```rust
struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
```

Бұл `User` атты структты жариялайды; оның төрт өрісі бар. Енді осы
типтен **дана** (instance) жасайық:

```rust
fn main() {
    let user1 = User {
        active: true,
        username: String::from("ainur123"),
        email: String::from("ainur@example.com"),
        sign_in_count: 1,
    };
}
```

Дана жасау үшін структ атын жазып, фигуралы жақшалардың ішінде әр
өрістің мәнін көрсетеміз. Өрістердің реті анықтаудағы ретпен сай
болмауы мүмкін — `Rust` оларды атау бойынша табады.

### Өрістерге қол жеткізу

Дананың өрісіне нүкте арқылы қол жеткіземіз:

```rust
let user1 = /* ... */;
println!("{}", user1.email);
```

Егер дана өзгермелі болса, өрісті қайта тағайындай аламыз:

```rust
let mut user1 = User { /* ... */ };
user1.email = String::from("жаңа@example.com");
```

Назар аударыңыз: бүкіл дана өзгермелі. `Rust`-та тек жекелеген өрістерді
ғана өзгермелі ете алмайсыз — мәннің бүкіл иесі өзгермелі немесе тұрақты.

### Өрісті жаңа дананы қайтаратын функциядан құру

Жиі функция бір параметрді қабылдап, жаңа дана құрып, оны қайтарады:

```rust
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username: username,
        email: email,
        sign_in_count: 1,
    }
}
```

Параметр пен өрістің атаулары бірдей болғанда — оны қайталау артық.
`Rust` ұсынатын **өрісті қысқа жариялау** (field init shorthand)
синтаксисі мынадай:

```rust
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username,
        email,
        sign_in_count: 1,
    }
}
```

Тек ат жазсаңыз — `Rust` сол атты параметрден алады.

### Бар данадан жаңа дана: жаңарту синтаксисі

Жиі бар данадан жаңа дана жасап, тек кейбір өрістерді ауыстырамыз. Қолдан
бәрін көшіруге тура келмеу үшін **структты жаңарту синтаксисі** бар:

```rust
fn main() {
    let user1 = User {
        email: String::from("ainur@example.com"),
        username: String::from("ainur123"),
        active: true,
        sign_in_count: 1,
    };

    let user2 = User {
        email: String::from("aibek@example.com"),
        ..user1
    };
}
```

`..user1` сөйлемі: «қалған өрістерді `user1`-ден ал». Бұл жазылмаған
өрістерді (`active`, `username`, `sign_in_count`) `user1`-дегі мәндермен
толтырады.

Маңызды: бұл синтаксис де иелік ережелеріне бағынады. `username` —
`String`, ол `Copy` емес, сондықтан мән `user1`-ден `user2`-ге
**ауысады**. `user1.username` ары қарай жарамсыз. Бірақ `active` (bool)
және `sign_in_count` (u64) — `Copy`, олар көшіріледі. Сондықтан мысалда
`user1.active` пен `user1.sign_in_count` әлі де жарамды.

### Кортеж-структ

Кейде өрістерге арнайы атау берудің керегі жоқ — өрістердің реті жеткілікті
семантика береді. Соған арналған **кортеж-структ** бар:

```rust
struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

fn main() {
    let black = Color(0, 0, 0);
    let origin = Point(0, 0, 0);
}
```

`Color` мен `Point` екеуі де `(i32, i32, i32)` түрімен сай, бірақ
кортеж-структтер басқа типтерге жатады. Сондықтан `Color`-ды күтетін
функцияға `Point` беру мүмкін емес. Бұл — типтік қауіпсіздік: семантикасы
басқа болғанда бір кортежді басқасына шатастырмайсыз.

Кортеж-структтың өрістеріне нөмір арқылы қол жеткіземіз:

```rust
let black = Color(0, 0, 0);
let red = black.0;
```

### Бірлік структ

Кейде ешбір өрісі жоқ структ жасау керек — таза семантикалық белгі
ретінде. Ондай структты **бірлік структ** деп атайды:

```rust
struct AlwaysEqual;

fn main() {
    let subject = AlwaysEqual;
}
```

Бұл өріс жоқ дана. Кейін трейттер мен жалпылама типтер арқылы оған
тәртіп бекітуге болады. Бірлік структтер таза белгі — типтік жүйедегі
тег ретінде ғана өмір сүреді.

### Структ деректерінің иелігі

Жоғарыдағы `User` структындағы `username` мен `email` өрістері
`String` түрінде болды — `&str` емес. Бұл — саналы шешім: біз бұл
структтың әр өзінің деректеріне иелік ететінін, олар жасырын түрде басқа
жерден қарызға алынбайтынын қалаймыз. Структтың бүкіл аумағы бар болғанша,
оның деректері де бар болады.

Структтың өрісінде сілтеме сақтау мүмкін — бірақ ол үшін **тіршілік
мерзімі белгісі** қажет. Тіршілік мерзімі ол сілтеменің әрқашан жарамды
болатынына кепілдік береді. 10-тарауда тіршілік мерзімдерін толық
қарастырамыз. Қазірге дейін структ өрістерінде иелікті ұстайтын типтерді
(мысалы `String`) қолданатын боламыз.

## 5.2. Структты қолданатын мысал бағдарлама

Структтың пайдасын нақты бағдарламамен көрейік. Тіктөртбұрыштың ауданын
есептейтін кішкене бағдарлама жазайық. Алдымен қарапайым нұсқа — екі
бөлек айнымалы:

```rust
fn main() {
    let width1 = 30;
    let height1 = 50;

    println!(
        "Тіктөртбұрыштың ауданы: {} шаршы пиксел.",
        area(width1, height1)
    );
}

fn area(width: u32, height: u32) -> u32 {
    width * height
}
```

Жұмыс істейді, бірақ қате осы кодта анық емес. `width` пен `height`
өзара байланысты — олар бір тіктөртбұрыштың өлшемдері. Кодты оқып
шыққанда мұны түсінесіз, бірақ `Rust`-тың типтік жүйесі мұны қолдамайды:
`width` пен `height` бір-бірінен бөлек тұр.

### Кортежпен жетілдіру

Алдымен кортеж арқылы біріктірейік:

```rust
fn main() {
    let rect1 = (30, 50);

    println!(
        "Тіктөртбұрыштың ауданы: {} шаршы пиксел.",
        area(rect1)
    );
}

fn area(dimensions: (u32, u32)) -> u32 {
    dimensions.0 * dimensions.1
}
```

Бір қадам жақсы — қазір екі сан байланысты. Бірақ `dimensions.0` пен
`dimensions.1` — бұл *ені* пе, *биіктігі* ме? Бұл коды оқыған адам
тіктөртбұрышты тоқытайтын жерде болса, оны еске сақтап тұру керек. Кодтың
мағынасы атаулардан тыс жатыр.

### Структпен жетілдіру

Енді структ арқылы:

```rust
struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let rect1 = Rectangle {
        width: 30,
        height: 50,
    };

    println!(
        "Тіктөртбұрыштың ауданы: {} шаршы пиксел.",
        area(&rect1)
    );
}

fn area(rectangle: &Rectangle) -> u32 {
    rectangle.width * rectangle.height
}
```

Енді функция нақты не қабылдайтынын білдіреді: тіктөртбұрыштың сілтемесін
(`&Rectangle`). Шақырылғанда `&rect1` дейміз — бұл `rect1`-ге сілтеме,
иеліктен айырмайды. `main` функциясы `rect1`-дің иесі болып қала
береді.

### Алынған трейттермен пайдалы мүмкіндіктер

Жөндеу барысында структты экранда көрсету ыңғайлы. Мынадайды жасайық:

```rust
struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    println!("rect1: {}", rect1); // ҚАТЕ
}
```

Қате аламыз: `Rectangle` `Display` трейтін бекітпеген — `Rust`
қайдан білсін, оны қалай экранға шығаруды? Жасамшы әр құрылымның
жеке-жеке басу әдісін шешеді.

`println!("{:?}", rect1)` арқылы біз **Debug** форматын сұраймыз —
жөндеу үшін ұсынылған шығу пішімі. Бірақ оған да трейт бекіту керек.
Структтың үстіне `#[derive(Debug)]` атрибутын қосамыз:

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    println!("rect1: {:?}", rect1);
}
```

Енді бағдарлама шығарады:

```text
rect1: Rectangle { width: 30, height: 50 }
```

Үлкенірек құрылымдар үшін `{:#?}` (әдемі басып шығару) — әр өрісті жаңа
жолға қойып, шегініспен пішімдейді:

```rust
println!("rect1: {:#?}", rect1);
```

Шығаратыны:

```text
rect1: Rectangle {
    width: 30,
    height: 50,
}
```

`derive` атрибуты — `Rust`-тың күшті құралы. Стандартты кітапхана
ұсынатын `Debug`, `Clone`, `Copy`, `PartialEq`, `Eq`, `Hash`, `Default`
сияқты трейттерді бір қатармен типке бекітуге мүмкіндік береді. 10-тарау
мен Қосымшадан толығырақ ала аласыз.

Жөндеу үшін тағы бір пайдалы құрал — `dbg!` макросы. Ол өрнектің мәнін
экранға шығарып, мәнді қайтарады:

```rust
let scale = 2;
let rect1 = Rectangle {
    width: dbg!(30 * scale),
    height: 50,
};
dbg!(&rect1);
```

`dbg!` файл атауын мен жол нөмірін көрсетеді — қайдан шыққанын анық
білесіз.

## 5.3. Әдіс синтаксисі

**Әдіс** — функцияға ұқсас, бірақ ол бір типтің контекстінде анықталады
және оның бірінші параметрі әрқашан `self` (мәннің өзі) болады. `self`
параметрі арқылы әдіс типтің данасына қол жеткізеді.

### Әдістерді анықтау

Структтың әдістерін `impl` блогында жариялаймыз. Тіктөртбұрыштың ауданын
есептеуге `area` функциясын `Rectangle`-ге әдіске айналдырайық:

```rust
#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    println!(
        "Тіктөртбұрыштың ауданы: {} шаршы пиксел.",
        rect1.area()
    );
}
```

Бірінші параметр — `&self`. Бұл — `self: &Self` дегеннің қысқа жазылуы;
`Self` — әдіс жасалған типтің өзі (мұнда — `Rectangle`). `&self` дегеніміз
— тек оқу үшін иелікті алмайтын сілтеме. Біз дананың мазмұнын өзгерткіміз
келсе, `&mut self` жазамыз. Әдіс мәннің иелігін алмайтыны — әдеттегі
қалып.

Әдіс пен өрістің атаулары бірдей болуы мүмкін — `Rust` контексттен
ажыратады. `rect1.area` — өріс; `rect1.area()` — әдіс шақыруы.

`Rust` сонымен бірге **автоматты сілтемелеу** ұсынады. C/C++ тілдерінде
біз `obj.method()` пен `ptr->method()`-ті ажыратамыз. `Rust`-та сізге
ондай ажырату керек емес: `rect1.area()` шақырғанда, `Rust` `&rect1`,
`&mut rect1` немесе `rect1` ішіндегі қандай да формасына дұрыс
автоматты түрде өзгертеді — әдістің `self` параметрінің түріне сәйкес.

### Қосымша параметрлері бар әдістер

Әдіске әдеттегі параметрлерді қосуға болады. `can_hold` атты әдісті
жазайық — басқа тіктөртбұрышты ішінде сыйғыза ма, жоқ па?

```rust
impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    let rect2 = Rectangle { width: 10, height: 40 };
    let rect3 = Rectangle { width: 60, height: 45 };

    println!("rect1 rect2-ні сыйғыза ма? {}", rect1.can_hold(&rect2));
    println!("rect1 rect3-ні сыйғыза ма? {}", rect1.can_hold(&rect3));
}
```

`can_hold` екі параметрді қабылдайды — біріншісі `self` (мысал данасы),
екіншісі — басқа `Rectangle`-ге сілтеме.

### Байланысты функциялар

`impl` блогында `self` параметрсіз функцияны да жариялауға болады. Олар
**байланысты функциялар** деп аталады — типке тиесілі, бірақ нақты
данасына емес. Әдетте конструктор ретінде қолданылады:

```rust
impl Rectangle {
    fn square(size: u32) -> Self {
        Self {
            width: size,
            height: size,
        }
    }
}

fn main() {
    let sq = Rectangle::square(3);
}
```

`Self` — `impl` блогының түрі (мұнда `Rectangle`). Сондықтан
`Self { width: size, height: size }` — жаңа `Rectangle` дана. Байланысты
функцияны нүктемен емес, `::` арқылы шақырамыз: `Rectangle::square(3)`.
`String::from`, `Vec::new` сияқты — солар да байланысты функциялар.

### Бірнеше `impl` блогы

`Rust` бір типке бірнеше `impl` блогын жариялауға рұқсат береді:

```rust
impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
```

Бұл бір блокта жазумен пара-пар. Бірнеше блок әсіресе жалпылама типтер
мен трейттер тұсында пайдалы — оларды 10-тарауда қарастырамыз.

## 5.4. Тарау қорытындысы

Структ — өзара байланысты деректерді бір аталған типке біріктіру
тәсілі. Аталған өрістер кодтың мағынасын айқын етеді — `dimensions.0`
қатарына `rectangle.width` жазу әлдеқайда оқуға жеңіл. Структтың
үстінен `#[derive(...)]` атрибуты арқылы стандартты трейттерді бір
қатармен бекітуге болады; `impl` блогында әдістер мен байланысты
функцияларды жариялаймыз. Әдіс — алғашқы параметрі `self` болатын
функция; байланысты функция — `self` жоқ.

Кортеж-структ пен бірлік структ — структтың арнайы түрлері. Барлық
структтердің ортақ қасиеті: типтік жүйеде өзіндік орын алады, басқа
типтермен араластырылмайды.

Келесі тарауда структтың серіктес ұғымы — енам — туралы айтамыз. Онда
сіз `Option`, `Result` сияқты `Rust`-тың ең жиі қолданылатын типтерінің
ішкі құрылымын көресіз.
